# 여기어때 — 계정 및 동기화

여기어때는 파트너센터의 **동시 접속 제한**이 VCMS 연동 안정성에 직접적인 영향을 주며, 시즌/상품/시간 설정이 세분화되어 있어 초기 세팅과 유지 관리 모두에서 주의가 필요합니다.

---

## 동시 로그인 충돌

### 어떤 상황에서 많이 막히는지

- VCMS가 여기어때 파트너센터에 연동 중인 상태에서 **업장 직원이 파트너센터에 직접 로그인**하면 VCMS 연동이 끊김
- 반대로, VCMS 연동이 활성 상태면 직원이 파트너센터에 접속할 때 **기존 세션이 튕겨남**
- 가족 공동 운영(배우자, 부모님) 업장에서 여러 명이 번갈아 접속할 때 특히 빈번
- 교대 근무 시마다 로그인 충돌이 발생하여 업무에 지장

### 왜 이런 일이 생기는지

여기어때 파트너센터는 **동일 계정의 동시 세션을 허용하지 않습니다**. VCMS도 파트너센터에 로그인하여 데이터를 동기화하는 방식이므로, 한쪽이 로그인하면 다른 쪽이 자동으로 로그아웃됩니다.

추가 계정을 만들려면 여기어때에 전화로 직접 요청해야 하며, 이 과정도 번거롭습니다.

### 운영자가 직접 확인할 순서

<Steps>
  <Step title="현재 접속 상태 확인">
    업장 직원이 여기어때 파트너센터에 직접 로그인해 있는지 확인합니다.
  </Step>
  <Step title="VCMS 연동 상태 확인">
    VCMS에서 여기어때 채널의 연동 상태가 정상인지 확인합니다. 에러가 표시되면 동시 접속이 원인일 가능성이 높습니다.
  </Step>
  <Step title="파트너센터 직접 접속 최소화 안내">
    업장에 여기어때 파트너센터 직접 접속을 가능한 한 줄이고, VCMS를 통해 관리하도록 안내합니다.
  </Step>
  <Step title="직접 접속이 필요한 경우">
    파트너센터에 직접 접속해야 하는 경우, VCMS 동기화가 일시적으로 끊길 수 있음을 안내하고 작업 완료 후 VCMS 연동 상태를 다시 확인합니다.
  </Step>
</Steps>

### 자주 생기는 오해

<AccordionGroup>
  <Accordion title="여기어때 파트너센터도 같이 쓰면 안 되나요?">
    동시 접속이 불가능하므로, VCMS 연동 중에는 파트너센터 직접 접속을 피하는 것이 좋습니다. 꼭 필요하면 접속 후 빠르게 작업을 완료하고, VCMS 연동 상태를 다시 확인합니다.
  </Accordion>
  <Accordion title="직원마다 계정을 따로 만들 수 없나요?">
    여기어때는 추가 계정 생성을 위해 수동으로 전화를 해야 하며, 추가 계정이 만들어져도 동시 접속 제한은 동일하게 적용될 수 있습니다.
  </Accordion>
  <Accordion title="연동이 계속 끊기는데 여기어때 문제인가요 벤디트 문제인가요?">
    여기어때의 동시 접속 제한 정책으로 인한 문제가 대부분입니다. VCMS 측 장애가 아닌 파트너센터의 정책적 제한입니다.
  </Accordion>
</AccordionGroup>

## 시즌 요금 및 설정 활성화

### 어떤 상황에서 많이 막히는지

- 여기어때 파트너센터에서 시즌 요금 설정이 **'사용'으로 활성화되지 않은 상태**에서 VCMS가 요금을 전송하면 반영되지 않음
- 크리스마스 기간에 10만원 추가 요금을 설정했으나 여기어때에 적용되지 않아 기존 요금으로 예약이 들어온 사례
- 여기어때의 상품별 요금 및 시간 설정이 매우 다양하고 복잡하여, 초기 세팅 시 누락이 발생하기 쉬움
- 동기화 기간 설정이 30일 또는 180일로 제한되어, 설정 범위 밖의 날짜에는 요금이 전달되지 않음

### 왜 이런 일이 생기는지

여기어때 파트너센터는 시즌별 요금을 **명시적으로 '사용' 상태로 활성화**해야 작동합니다. VCMS에서 아무리 요금을 설정하고 전송해도, 파트너센터 측에서 해당 시즌이 비활성 상태면 반영되지 않습니다.

또한 동기화 기간이 제한적이므로, VCMS에서 6개월 뒤 요금을 설정해도 여기어때의 연동 기간(30일 또는 180일) 밖이면 전달되지 않습니다.

### 운영자가 직접 확인할 순서

<Steps>
  <Step title="시즌 요금 활성화 상태 확인">
    여기어때 파트너센터에서 현재 시즌 및 예정된 시즌의 요금 설정이 '사용'으로 활성화되어 있는지 확인합니다.
  </Step>
  <Step title="동기화 기간 확인">
    여기어때의 연동 기간이 30일인지 180일인지 확인하고, VCMS의 요금 설정 범위가 이 기간 내에 있는지 점검합니다.
  </Step>
  <Step title="특정 일자 요금 확인">
    공휴일, 성수기 등 특별 요금을 설정한 날짜가 여기어때에 정확히 반영되어 있는지 확인합니다.
  </Step>
  <Step title="VCMS 재동기화">
    설정이 누락되어 있었다면 파트너센터에서 활성화한 후, VCMS에서 재동기화를 진행합니다.
  </Step>
</Steps>

<Warning>
  시즌 요금이 비활성 상태인 것을 발견하면, 해당 기간에 들어온 예약이 잘못된 요금으로 처리되었을 가능성이 있습니다. 기존 예약의 요금도 확인해야 합니다.
</Warning>

## 매핑 오류 및 상품 불일치

### 빈출 사례

| 증상 | 원인 |
|------|------|
| VIP 투룸 예약이 VIP 원룸으로 올라옴 | 파트너센터 상품과 VCMS 객실 타입 간 매핑 오류 |
| 주니어 스위트룸이 파트너센터에서 보이지 않음 | 상품 등록 누락 또는 비활성화 |
| 금요일 입퇴실 시간만 다른 요일과 다르게 설정됨 | 슈퍼 디럭스 등 특정 객실의 시간 설정 불일치 |
| 여기어때 예약이 VCMS에 안 나타남 | 상품/요금 설정 미완료 상태에서 연동 시도 |
| 재고 수가 실제 예약 수와 다르게 표시 | 판매 관리 달력의 재고 표시 방식 차이 |

### 상품 매핑 시 주의사항

- 여기어때 파트너센터에 등록된 상품 목록과 VCMS의 객실 타입이 **1:1로 매핑**되어 있는지 반드시 확인
- 과거에 여기어때에서 설정한 상품 구조가 호실 단위와 타입 단위로 **혼재**되어 있는 경우, 정리 후 매핑 필요
- 추후 여기어때 객실 유형을 야놀자처럼 표준화하기로 결정할 경우, 모든 매핑 작업을 다시 해야 하는 부담 존재

<AccordionGroup>
  <Accordion title="여기어때에 상품이 있는데 VCMS에서 안 보여요">
    파트너센터의 상품이 VCMS에 매핑되지 않았거나 비활성 상태일 수 있습니다. 매핑 상태를 확인하고, 누락된 상품은 수동으로 매핑합니다.
  </Accordion>
  <Accordion title="여기어때와 야놀자 간 정보 형식이 달라요">
    두 채널에서 넘어오는 예약 정보의 형식이 다릅니다. 호실 정보가 없는 경우도 있으므로, VCMS의 자동 배정 방식에 의존해야 합니다.
  </Accordion>
</AccordionGroup>

## 입퇴실 시간 제한

### 호텔 카테고리의 제약

여기어때 **호텔 카테고리**는 입퇴실 시간 변경에 대한 제약이 있습니다:

- 일자별 입퇴실 시간 변경을 지원하지 않음
- 객실 설정에서 한 번 설정하면 전체에 일괄 적용
- VCMS에서 야놀자/여기어때의 입퇴실 시간은 캘린더로 제어 가능하지만, 네이버/아고다는 각 플랫폼에서 직접 설정해야 함

### 모텔 카테고리에서의 유연성

모텔 카테고리는 호텔 대비 입퇴실 시간 설정이 유연하지만, 여전히 **1시간 단위로만 변경 가능**한 제약이 있습니다. 30분 단위 등 세밀한 시간 조정은 불가합니다.

## 반영 지연 및 동기화 문제

### 확인된 지연 패턴

- VCMS에서 객실 마감 처리를 했는데 여기어때에 실시간으로 반영되지 않는 현상
- 벤디트 직원의 설정 오류, 업장의 인터넷 속도 문제 등이 복합적으로 작용하는 사례
- 전체 채널 동기화 시 기존 예약이 모두 삭제되어 수동으로 재입력해야 하는 상황
- CMS에서 연동 오류가 간헐적으로 발생하여 동기화 실패

### 2차 인증 관련 문제

여기어때 2차 인증 설정이 VCMS 연동에 문제를 일으키는 사례가 보고되었습니다.

- 2차 인증이 활성화되어 있으면 VCMS의 자동 로그인이 실패할 수 있음
- 벤디트 측에서는 **2차 인증 해제를 권장**
- 2차 인증을 유지해야 하는 보안 요구사항이 있다면, 인증 절차가 VCMS 연동에 미치는 영향을 사전에 테스트

## 기존 CMS에서의 전환 시 주의사항

### 여기어때와 기존 시스템의 관계

- 여기어때는 과거에 일부 CMS(떠나요 등)와 연동되어 있었으나, 현재는 관리가 더 어려워진 경우도 있음
- 기존 CMS에서 여기어때 파트너센터 연동 해제 시, 방막기를 일일이 수동으로 해야 하는 어려움 때문에 여기어때 판매를 아예 중단하는 업장도 있음
- 일부 채널 관리 시스템은 여기어때와의 연동을 일방적으로 종료하여 업장에 큰 혼란을 야기한 이력이 있음
- 기존에 무료 기간 후 의무 광고비를 청구하는 방식으로 계약을 진행했던 과거 사례도 있으므로, 업장의 기존 계약 상태를 확인할 필요가 있음

## 실무 체크리스트

- [ ] 여기어때 파트너센터 동시 접속 제한을 업장에 안내했는가
- [ ] 시즌 요금이 '사용' 상태로 활성화되어 있는가
- [ ] 연동 기간(30일/180일) 설정을 확인했는가
- [ ] 모든 상품이 VCMS에 정확히 매핑되어 있는가
- [ ] 호텔 카테고리 사용 시 입퇴실 시간 일괄 적용 제약을 안내했는가
- [ ] 2차 인증이 해제되어 있는가 (또는 연동에 영향 없음을 확인했는가)
- [ ] 크리스마스, 공휴일 등 특수 기간 요금이 정확히 반영되어 있는가
- [ ] 동기화 지연 가능성을 업장에 안내했는가
- [ ] 업장 직원이 파트너센터에 직접 접속하는 빈도를 최소화하도록 안내했는가
- [ ] 기존 CMS에서의 전환 시 여기어때 연동이 정상적으로 이관되었는지 확인했는가
