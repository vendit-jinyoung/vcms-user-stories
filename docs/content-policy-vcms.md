# VCMS 콘텐츠 정책 가이드

## 1. 제품 범위
- VCMS는 채널 매니저다.
- 재고, 요금, 채널 연동, 동기화, 채널 운영 포인트를 중심으로 쓴다.
- PMS, 객실 배정, 청소 상태, 키오스크, SMS, 부킹엔진을 VCMS의 기본 기능처럼 쓰지 않는다.

## 2. 채널 상태
### 작동 채널
- 야놀자
- 여기어때
- 네이버
- 아고다
- 트립닷컴
- 떠나요
- 온다 펜션플러스

### 준비 중 채널
- 부킹닷컴
- 익스피디아
- 에어비앤비

규칙:
- 준비 중 채널을 활성 연동 채널처럼 예시에 섞지 않는다.
- 필요하면 `준비 중`, `비연동 채널`, `별도 상태 확인 필요`처럼 쓴다.

## 3. 용어집 우선
- 가격 -> 요금
- 가격표 -> 요금표
- 최저가 -> 최저 요금
- 상품 -> Package
- 채널 상품 -> Channel Package
- 예약 -> Booking
- 동기화 -> Sync

## 4. 콘텐츠 톤
- 특정 OTA를 비난하지 않는다.
- `문제아`, `양아치`, `지뢰`, `지옥` 같은 표현 금지.
- `운영상 특성`, `주의점`, `반복 리스크`, `확인 포인트`처럼 중립적으로 쓴다.

## 5. 데이터/사실 노출
- 정확한 통화 수, 클레임 수, 내부 매출 수치 노출 금지.
- 내부 출처를 과도하게 드러내지 않는다.
- 특정 업장명, 개인명, 식별 가능한 사례는 제거한다.
- `실제 현장 사례`, `반복적으로 확인된 패턴` 정도로 완화한다.

## 6. 정산 관련 원칙
- 일반적인 정산/매출 리포트 설명은 VCMS 핵심 문서에서 과도하게 전면화하지 않는다.
- 채널 문서에서는 `입금가`, `수수료`, `요금 구조 확인 포인트` 수준으로만 다룬다.
- PMS/클라우드 성격의 정산 흐름은 VCMS 핵심 설명으로 확장하지 않는다.

## 7. 작성 프롬프트 초안
```text
VCMS 콘텐츠를 작성할 때는 다음을 지켜라.
- VCMS는 채널 매니저로 설명한다.
- 준비 중 채널(부킹닷컴, 익스피디아, 에어비앤비)을 활성 예시에 넣지 않는다.
- 용어는 VCMS 용어집 기준으로 통일한다. 특히 `가격` 대신 `요금`을 쓴다.
- 특정 채널을 비난하지 말고 운영상 특성과 주의점으로 표현한다.
- PMS, 키오스크, SMS, 부킹엔진 기능을 VCMS의 기본 기능처럼 쓰지 않는다.
- 정확한 내부 수치, 직접 매출 수치, 식별 가능한 사례는 제거한다.
- 통화 기반 문장은 `실제 현장 사례`, `반복적으로 확인된 패턴` 수준으로 완화한다.
```
