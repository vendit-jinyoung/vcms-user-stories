## 이런 상황이신가요?

여러 숙소나 지점을 운영하면서, 계정 관리와 권한 설정에 어려움을 겪고 계시죠? 실제 숙박업소에서 겪는 상황들이에요.

- **동시 접속하면 자꾸 튕겨요** — "벤디트 파트너 센터 계정이 여러 직원이 동시에 접속할 때마다 자주 튕겨서 정산 등 업무 처리에 어려움을 겪고 있어요."
- **퇴사 직원이 계정에 접근했어요** — "퇴사한 직원이 계정에 접근했던 경험이 있어서 보안에 대한 우려가 커요."
- **직원마다 볼 수 있는 걸 다르게 하고 싶어요** — "업장 직원들이 시스템을 함께 관리하기에는 어려움이 있어 계정 분리 및 개별 관리가 필요해요."
- **여러 직원이 재고를 바꿔서 충돌이 생겨요** — "여러 사용자가 동시에 시스템 동기화 설정을 변경하여 충돌이 발생하고, 시스템이 오작동하는 것으로 오해하고 있어요."
- **직원이 바뀔 때마다 세팅이 번거로워요** — "신규 직원이 프로그램 변경에 적응하는 과정에서 실수가 발생할까 봐 걱정이고, 사장님도 우려하고 있어요."
- **여러 업장의 ID가 꼬여 있어요** — "MPAY 연동이 안 되는 문제가 있는데, 슬로우오션과 히든포레스트의 ID가 분리되어 있고 벤디트 직원의 개인 ID와 얽혀있어요."
- **재고 불일치가 사람 때문이에요** — "여러 명의 사용자가 재고 값을 계속 바꾸면서 재고 불일치가 발생하고, 이는 시스템이 정상 작동해도 원치 않는 결과를 초래하는 주요 원인이에요."

## VCMS로 이렇게 해결할 수 있어요

VCMS는 **지점별 계정 분리**와 **사용자별 권한 설정**을 지원해요. 사장님은 전체 지점을 통합으로 보면서, 각 지점 직원에게는 필요한 권한만 부여할 수 있어요.

<Check>
  사용자별로 접근 가능한 메뉴와 기능을 제한할 수 있어요. 프론트 직원에게는 예약 조회와 체크인만, 관리자에게는 요금 변경과 정산까지 — 역할에 맞는 권한을 부여하면 실수와 보안 사고를 방지할 수 있어요.
</Check>

<Warning>
  여러 사용자가 동시에 같은 객실의 재고를 변경하면 충돌이 발생할 수 있어요. 재고 변경 권한은 가급적 소수의 관리자에게만 부여하고, 나머지 직원은 조회 권한만 갖도록 설정하세요. 퇴사 직원의 계정은 즉시 비활성화해야 해요.
</Warning>

## 단계별 가이드

<Steps>
  <Step title="1. 지점별 계정 구성 계획">
    운영 중인 지점 목록을 정리하고, 각 지점별로 필요한 사용자 수와 역할을 파악하세요. 사장님(전체 관리), 지점장(지점 관리), 프론트(예약 처리) 등으로 구분해요.
  </Step>
  <Step title="2. 마스터 계정 설정">
    사장님 또는 총 관리자의 마스터 계정을 설정하세요. 마스터 계정은 모든 지점의 데이터를 통합 조회하고, 사용자 관리 권한을 가져요.
  </Step>
  <Step title="3. 지점별 관리자 계정 생성">
    각 지점의 관리자 계정을 생성하세요. 지점 관리자는 해당 지점의 예약, 재고, 요금을 관리할 수 있지만, 다른 지점의 데이터에는 접근할 수 없도록 제한해요.
  </Step>
  <Step title="4. 직원 계정 생성 및 권한 부여">
    프론트 직원 등 일반 사용자의 계정을 생성하고, 역할에 맞는 권한만 부여하세요:
    - **프론트 직원**: 예약 조회, 체크인/체크아웃
    - **지점 관리자**: 예약 관리, 재고 변경, 요금 설정
    - **사장님**: 전체 관리, 정산, 사용자 관리
  </Step>
  <Step title="5. 동시 접속 설정 확인">
    계정을 분리하면 여러 직원이 동시에 접속해도 충돌이 발생하지 않아요. 하나의 계정을 여러 명이 공유하지 않도록 주의하세요.
  </Step>
  <Step title="6. 퇴사 직원 계정 비활성화">
    직원이 퇴사하면 즉시 해당 계정을 비활성화하세요. 비밀번호 변경만으로는 부족해요 — 계정 자체를 비활성화해야 재접근을 막을 수 있어요.
  </Step>
  <Step title="7. 재고 변경 권한 제한">
    재고를 변경할 수 있는 권한은 지점 관리자 이상에게만 부여하세요. 여러 직원이 동시에 재고를 바꾸면 데이터 충돌이 생기고, 의도치 않은 오버부킹으로 이어질 수 있어요.
  </Step>
  <Step title="8. 정기 계정 감사">
    분기마다 전체 사용자 계정 목록을 점검하세요. 사용하지 않는 계정이나 퇴사 직원 계정이 활성 상태로 남아있지 않은지 확인해요.
  </Step>
</Steps>

## 어떤 숙박업소에 적합한가요?

- **2개 이상의 지점**을 운영하는 숙박 사업자
- **직원 수가 3명 이상**이며 역할별 권한 분리가 필요한 곳
- **퇴사 직원 보안**에 민감한 숙박업소
- 여러 직원이 동시에 시스템에 접속하며 **충돌 문제**를 겪고 있는 곳
- 사장님이 현장에 없어도 **원격으로 전체 운영 현황**을 파악해야 하는 곳

---

## 기대 효과

<Check>
  - 사용자별 계정 분리로 **동시 접속 시 튕김 현상이 해소**돼요.
  - 퇴사 직원 계정 즉시 비활성화로 **보안 사고를 방지**해요.
  - 역할별 권한 설정으로 **실수로 인한 재고 변경이나 요금 오류**가 줄어요.
  - 마스터 계정에서 **전체 지점의 운영 현황을 한눈에** 파악할 수 있어요.
  - 재고 변경 권한 제한으로 **여러 사용자 간 충돌이 사라져요**.
</Check>

## 운영 팁

<AccordionGroup>
  <Accordion title="직원 온보딩/오프보딩 체크리스트">
    **신규 직원 입사 시:**
    1. 직원 전용 계정 생성 (공용 계정 공유 금지)
    2. 역할에 맞는 권한 부여 (최소 권한 원칙)
    3. 기본 조작 교육 실시
    4. 비밀번호 정책 안내 (정기 변경 권장)

    **직원 퇴사 시:**
    1. 당일 계정 비활성화
    2. 공용 비밀번호가 있었다면 즉시 변경
    3. 해당 직원이 마지막으로 변경한 설정 점검
  </Accordion>
  <Accordion title="동시 접속 충돌 방지 팁">
    여러 직원이 같은 시스템을 사용할 때 충돌을 방지하려면:
    - **각자 다른 계정**으로 접속하세요 (하나의 계정 공유 금지)
    - **재고 변경은 한 사람만** 담당하도록 정하세요
    - 같은 데이터를 동시에 수정하지 않도록 **업무 시간/구역을 분리**하세요
    - 변경 사항은 **저장 후 새로고침**하여 최신 데이터를 확인하세요
  </Accordion>
  <Accordion title="다지점 정산 관리 요령">
    여러 지점의 정산을 효율적으로 관리하려면:
    - 마스터 계정에서 지점별 매출을 비교 조회하세요
    - 지점별로 채널 수수료율이 다를 수 있으므로 개별 확인하세요
    - 월간 리포트에서 지점별 성과를 분석하여 운영 전략을 수립하세요
  </Accordion>
  <Accordion title="직원 권한 수준 가이드">
    역할별 권장 권한 수준:
    - **사장님/대표**: 전체 접근 + 사용자 관리 + 정산 + 설정 변경
    - **지점 관리자**: 해당 지점 예약 + 재고 + 요금 + 보고서
    - **프론트 직원**: 예약 조회 + 체크인/아웃 + 메모 입력
    - **청소/시설 직원**: 객실 상태 변경만 (해당 시)
    권한은 언제든 수정 가능하므로, 처음에는 최소 권한으로 시작하고 필요에 따라 추가하세요.
  </Accordion>
</AccordionGroup>
