채널 매니저 도입 첫날. 기대에 찬 사장님 앞에서 벤디트 담당자가 물었다. "야놀자 파트너센터 아이디하고 비밀번호 좀 알려주세요." 잠깐의 침묵. "야놀자는... 잠깐만요." 포스기 옆 메모장을 뒤진다. "여기어때는요?" "그건 전 직원이 알던 건데..." "네이버 스마트플레이스는요?" "네이버는 제 개인 계정이에요. 근데 비번이 뭐였더라..."

이 대화는 과장이 아니다. 온보딩 상담 기록에서 거의 판박이처럼 등장하는 첫날의 풍경이다. 요금 설정도, 재고 연동도, 예약 조회도 — 전부 뒤로 밀린다. 아이디와 비밀번호를 찾는 것부터가 전쟁이니까.

**일곱 개 채널, 일곱 개 비밀번호.** 야놀자 파트너센터, 여기어때 파트너센터, 네이버 스마트플레이스, 아고다 YCS, 트립닷컴 파트너허브, 온다 관리자, 떠나요닷컴 관리자. 여기에 VCMS 계정까지 합치면 여덟 개다. 아이디 형식도 제각각이다. 이메일로 로그인하는 곳, 사업자번호 기반인 곳, 전화번호를 쓰는 곳. 비밀번호 규칙도 다르다. 대문자 필수, 특수문자 필수, 8자 이상, 12자 이상, 90일마다 변경 강제. 모든 채널의 규칙을 외우는 것 자체가 하나의 기술이다.

이 계정들을 어디에 보관하는가. 포스기 옆 포스트잇, 카카오톡 '나에게 보내기', 엑셀 파일, 사장님 머릿속. 보관 방법이 다양한 만큼 분실 경로도 다양하다. 포스트잇이 떨어져서, 카톡 대화가 밀려서, 엑셀 파일을 저장한 PC가 바뀌어서, 사장님이 까먹어서. 공통점은 하나다. 한 사람의 기억에 의존한다는 것. 그리고 그 한 사람이 부재하면 시스템이 멈춘다는 것.

**야놀자 비밀번호가 바뀌면 벌어지는 일.** 플랫폼이 90일마다 비밀번호 변경을 요구한다. 사장님이 새 비밀번호를 만들고 로그인한다. 여기까진 문제없다. 그런데 채널 매니저에는 이전 비밀번호가 저장되어 있다. 사장님은 야놀자에는 로그인이 되지만, 채널 매니저의 야놀자 연동은 끊어진다. 재고 동기화가 멈춘다.

이 상태가 며칠 지속되면 어떻게 될까. 야놀자에서 예약이 들어와도 여기어때 재고가 안 줄어든다. 역도 마찬가지다. 채널 간 재고가 따로 놀기 시작하고, 결국 오버부킹이 터진다. 비밀번호를 바꾼 것과 오버부킹 사이에 인과관계가 있다는 걸 깨달았을 때는, 이미 손님 두 명이 같은 방을 예약한 뒤다. 비밀번호 변경 후 채널 매니저에 새 정보를 입력하는 것을 잊는 — 사소해 보이는 이 실수가 상담 기록에서 가장 빈번하게 등장하는 연동 끊김 원인 중 하나다.

**네이버의 2단계 인증은 다른 차원의 번거로움이다.** 네이버 스마트플레이스는 보안이 까다롭다. 새로운 기기나 IP에서 로그인하면 인증 코드를 요구한다. 채널 매니저가 요금을 동기화하려고 접속할 때 이 인증이 걸리면, 사장님 폰으로 인증번호를 받아 입력해야 한다. "자동화"의 의미가 반감되는 순간이다.

다지점 운영에서 이 문제는 증폭된다. 지점마다 네이버 계정이 따로 있는데, 본사에서 통합 관리하려면 각 지점의 인증 코드를 실시간으로 받아야 한다. 결국 카카오톡 단톡방에 "네이버 인증번호 보내주세요"라는 메시지가 매일 올라간다. 지점 직원이 바쁘면 답이 안 온다. 답이 안 오면 동기화가 안 된다. 보안을 위한 장치가 운영의 병목이 된 아이러니. 네이버 계정을 여러 지점에서 공유하기 때문에 비밀번호가 자주 변경되며, 2차 인증 관리가 어려워 보안 잠금 위험이 있다는 보고가 반복적으로 등장한다.

**여기어때의 동시접속 제한은 업계에서 유명하다.** 한 계정으로 두 사람이 동시에 접속하면 한쪽이 강제 로그아웃된다. 사장님이 정산을 확인하고 있는데 채널 매니저가 요금 동기화를 시도하면 — 사장님이 튕긴다. "또 튕겼다"와 "다시 로그인해야 한다"의 반복. 이 문제에 대한 상담 기록이 수십 건에 달한다.

해결책은 의외로 간단하다. 여기어때에 전화해서 VCMS 전용 관리자 계정을 별도로 만드는 것. 그런데 이 방법을 모르는 운영자가 놀라울 정도로 많다. 몇 달간 "튕기는 것"을 참으며 쓰다가, 상담 중에 이 해결책을 알게 되는 경우가 대부분이다.

**가장 치명적인 시나리오는 담당자 퇴사다.** 채널 관리를 전담하던 직원이 그만둔다. 인수인계는 "야놀자는 이 아이디로 들어가고, 여기어때는 저 아이디고"가 전부다. 메모에 적힌 정보가 맞는지 검증하는 사람은 없다. 새 직원이 로그인을 시도하면 비밀번호가 틀리다. 전 직원에게 연락해도 "기억이 안 나요"다. 아고다 YCS 비밀번호를 분실하면 아고다 본사에 이메일을 보내야 하고, 답변이 오기까지 이틀이 걸린다. 이틀 동안 아고다 요금은 변경 불가, 재고 조정 불가.

소규모 숙소에서 더 흔한 버전도 있다. 부부가 함께 운영하는 펜션에서 아내가 예약 관리를, 남편이 시설 관리를 담당한다. 아내만 시스템 비밀번호를 알고 있다. 아내가 출장 가거나 입원하면 온라인 예약 관리가 통째로 멈춘다. 시스템 지식의 단일 장애점(single point of failure). 신뢰가 높은 가족 경영이지만, 백업이 없다. 배우자가 귀국한 후 설명을 위해 재고·요금 연동을 보류했다는 기록은 이 구조의 취약성을 적나라하게 보여준다.

**이 모든 문제의 근본은 하나다.** 채널은 늘어나는데 인증 체계는 각자 따로 놀고, 그걸 한데 묶어줄 장치가 없다. 채널 매니저가 한 화면에서 모든 채널을 조작하게 해주지만, 그 전제 조건인 "각 채널에 로그인 가능한 상태를 유지하는 것" 자체가 이미 하나의 업무다. 채널이 세 개일 때는 관리할 만하다. 다섯 개가 넘으면 계정 관리 자체가 독립된 업무 영역이 된다. 일곱 개가 되면 그건 더 이상 부수적 작업이 아니라 전담 업무다.

온보딩 교육의 상당 시간이 로그인에 소비된다는 사실은, 비밀번호라는 가장 낮은 단계의 장벽이 얼마나 높은 비용을 발생시키는지를 보여준다. 요금 최적화, 재고 전략, 프로모션 운영 — 이런 고차원의 작업은 로그인이 된 다음에야 시작할 수 있다. 그런데 로그인 자체가 매일의 허들이라면, 나머지는 영원히 시작되지 못한다.

여기서 한 발짝 더 들어가면, 비밀번호 문제는 단순한 불편함을 넘어서 보안 위험으로 번진다. 비밀번호를 포스트잇에 적어 모니터에 붙여놓는 업장이 있다. 퇴사한 직원이 여전히 파트너센터에 접속할 수 있는 상태인 업장도 있다. 계정 정보가 카카오톡 대화방에 평문으로 떠돌아다니는 것은 흔한 일이다. 이건 편의성을 위해 보안을 포기한 것이지만, 현장에서는 그런 거 따질 여유가 없다. 지금 당장 야놀자에 로그인해서 요금을 바꿔야 하니까.

비밀번호 관리 도구를 쓰면 되지 않느냐는 질문이 나올 수 있다. 맞다, 해결책은 존재한다. 하지만 비밀번호 관리 도구를 도입하고 교육하고 유지하는 것도 또 하나의 프로젝트다. 채널 매니저 하나 도입하는 것도 버거운 업장에게, 거기에 비밀번호 관리 도구까지 도입하라는 건 현실적이지 않다. 결국 가장 현실적인 방법은 계정 정보를 문서화하고, 그 문서를 안전한 곳에 보관하고, 비밀번호가 변경될 때마다 채널 매니저에도 반영하는 습관을 들이는 것이다. 간단해 보이지만, 이 간단한 루틴이 현장에서 지켜지는 경우는 드물다.

채널 매니저의 가치를 논하기 전에, 먼저 로그인이 되는지부터 확인해야 한다. 비밀번호 대란은 거기서 시작된다. 그리고 거기서 끝나지 않는다.

## 채널별 인증 방식 비교

| 채널 | 인증 방식 | 주요 이슈 | VCMS 연동 영향 |
|------|---------|---------|-------------|
| 야놀자 | ID/PW | 비밀번호 변경 시 연동 끊김 | 재연동 필요 |
| 여기어때 | ID/PW | 동시 접속 시 계정 튕김 | 연동 불안정 |
| 네이버 | 2FA + 앱비밀번호 | 2FA 해제/변경 시 연동 끊김 | 앱비밀번호 재발급 |
| 아고다 | ID/PW (YCS) | 비밀번호 만료 정책 | 재연동 필요 |
| 트립닷컴 | ID/PW | 로그인 장애 빈발 | 수동 확인 필요 |

## 계정 관리 실무 가이드

### 연동 계정 분리 원칙

- VCMS 연동용 계정과 일상 관리용 계정을 **분리**하세요
- 연동용 계정의 비밀번호는 **변경하지 마세요** (변경 시 연동 끊김)
- 가능하면 연동 전용 네이버 계정을 별도로 생성하세요

### 비밀번호 관리 체크리스트

| 항목 | 권장 사항 |
|------|---------|
| 연동 계정 비밀번호 | **절대 변경 금지** (변경 시 연동 끊김) |
| 비밀번호 저장 | 안전한 곳에 기록 (매니저 인수인계 필수) |
| 2FA 수신 번호 | 상시 접근 가능한 번호로 설정 |
| 퇴사 직원 처리 | 당일 계정 비활성화 + 공유 비밀번호 변경 |
| 정기 점검 | 월 1회 연동 상태 확인 |

### 채널별 비밀번호 변경 시 대응

1. **야놀자**: 비밀번호 변경 후 VCMS에서 야놀자 연동 재설정
2. **여기어때**: 비밀번호 변경 후 VCMS 연동 상태 확인
3. **네이버**: 비밀번호 변경 시 앱비밀번호 무효화 → 2FA 유지 상태에서 앱비밀번호 재발급 → VCMS 재연동
4. **아고다**: YCS 비밀번호 만료 시 재설정 후 VCMS 연동 확인
5. **트립닷컴**: 비밀번호 변경 후 VCMS 연동 확인 + 트립닷컴 담당자에게 연동 활성화 확인

### 담당자 변경 시 인수인계 항목

- [ ] 모든 채널 로그인 정보 (ID, PW, 2FA 번호)
- [ ] VCMS 관리자 계정 정보
- [ ] 각 채널의 연동 상태 확인 방법
- [ ] 비밀번호 변경 시 연동 재설정 절차
- [ ] 비상 연락처 (벤디트 CS, 각 채널 담당자)
