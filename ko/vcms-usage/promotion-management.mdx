## 이런 상황 아닌가요?

> "아고다에서 내가 설정하지 않은 할인이 적용돼서 예상보다 싸게 팔렸어요. 야놀자에서는 쿠폰이 자동으로 걸리고요."

각 채널마다 프로모션 정책과 적용 방식이 달라서, 업장이 의도한 요금과 실제 판매 요금이 다르게 나오는 경우가 반복돼요.

- **아고다 자동 10% 할인**이 걸려서 내가 설정한 요금보다 낮게 팔리고 있어요. 동의한 적도 없어요.
- **야놀자에서 프로모션마다 새 상품이 자동 생성**돼요. 도보특가, 오픈특가, 핫딜 — VCMS에서 각각 매핑해야 해요.
- **전체 마감을 했는데 도보특가만 열려 있어서** 의도치 않은 저가 예약이 들어왔어요.
- **프로모션 기간에 요금을 일괄 변경**하고 싶은데, 채널마다 따로 설정해야 하는 게 번거로워요.
- **오픈 스페셜 상품이 2026년 12월까지 열려 있어요.** 판매 종료일을 안 넣었더니 의도치 않게 장기간 판매되고 있어요.
- **네이버 연동 시 공휴일이나 특가로 설정된 일정이 삭제**될 수 있어요.

## 채널별 프로모션 유형 비교

| 채널 | 프로모션 방식 | 업장 동의 | VCMS 대응 |
|------|------------|----------|----------|
| 야놀자 | **별도 상품 생성** (핫딜, 도보특가, 오픈특가) | 부분적 | 각 상품 VCMS 매핑 필수 |
| 아고다 | **자체 할인 적용** (10% 자동, 회원할인, 모바일할인) | 없음 (대부분 자동) | 입금가 마크업으로 대응 |
| 여기어때 | 쿠폰/할인 적용 | 부분적 | 채널 전송가 조정 |
| 네이버 | 오늘의 딜, 특가 노출 | 업장 직접 설정 | VCMS 연동 시 비활성화 주의 |
| 트립닷컴 | 자체 프로모션 + 환율 변동 | 없음 | 입금가 마크업으로 대응 |

## VCMS에서 이렇게 대응해요

<Check>
VCMS의 **채널 전송가(마크업)** 기능으로, 채널별 수수료와 프로모션 영향을 감안하여 요금을 차등 전송할 수 있어요. 아고다에는 +15%, 트립닷컴에는 +10% 식으로 설정하면 기준가 변경 시 자동 재계산돼요.
</Check>

<Check>
프로모션 기간에 **기간별 요금**을 미리 설정해두면 시즌 특가가 자동으로 적용돼요.
</Check>

<Warning>
야놀자의 프로모션 상품(도보특가, 오픈특가 등)은 VCMS에서 **별도로 매핑**해야 해요. 매핑하지 않으면 재고 차감이 안 되고 오버부킹 위험이 있어요.
</Warning>

<Warning>
아고다의 자동 할인은 업장이 끌 수 없는 경우가 많아요. 입금가(net rate) 설정 시 이 할인분을 감안해서 **마크업을 충분히** 설정하세요.
</Warning>

## 이렇게 관리하세요

<Steps>
  <Step title="채널별 프로모션 현황 파악">
    각 채널 파트너센터에서 현재 적용 중인 프로모션 목록을 확인하세요. 특히 아고다의 자동 할인, 야놀자의 활성 프로모션 상품을 점검하세요.
  </Step>

  <Step title="채널 전송가(마크업) 설정">
    VCMS 채널 전송가 설정에서 채널별 마크업을 조정하세요.

    | 채널 | 권장 마크업 | 이유 |
    |------|----------|------|
    | 야놀자 | +0~5% | 수수료 10% 감안 |
    | 여기어때 | +0~5% | 수수료 유사 |
    | 네이버 | -3~0% | 수수료 3%, 최저 요금 전략 |
    | 아고다 | +10~15% | 실질 수수료 16.5% + 자동할인 |
    | 트립닷컴 | +10~15% | 실질 수수료 최대 19% |

    <Tip>
    처음에는 보수적으로 마크업을 설정하고, 각 채널의 실제 판매가를 확인한 후 조정하세요.
    </Tip>
  </Step>

  <Step title="야놀자 프로모션 상품 매핑">
    야놀자 파트너센터에서 활성화된 프로모션 상품(도보특가, 오픈특가, 핫딜 등)이 VCMS에 매핑되어 있는지 확인하세요. 미지정 상태의 프로모션 상품이 있으면 즉시 매핑하세요.
  </Step>

  <Step title="판매 종료일 설정">
    시즌 프로모션이나 특가 상품에는 반드시 **판매 종료일**을 설정하세요. 종료일 없이 열어두면 의도치 않은 장기 판매가 발생해요.
  </Step>

  <Step title="실제 판매가 모니터링">
    각 채널 앱/사이트에서 실제 고객에게 노출되는 요금을 주기적으로 확인하세요. VCMS 설정가와 실제 판매가의 차이가 있다면 프로모션이나 자동 할인 적용 여부를 점검하세요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="아고다 자동 할인을 끌 수 있나요?">
    대부분의 아고다 자동 할인(10% 자동 차감, 회원 할인, 모바일 할인)은 업장이 직접 비활성화할 수 없어요. 아고다 담당자(AM)에게 요금 매칭 메일이나 프로모션 해제를 요청할 수는 있지만, 모든 프로모션을 끌 수 있는 것은 아니에요. 가장 현실적인 대응은 입금가 설정 시 마크업을 충분히 올리는 거예요.
  </Accordion>

  <Accordion title="야놀자 전체 마감했는데 도보특가만 열려 있어요">
    야놀자의 프로모션 상품(도보특가, 오픈특가)은 일반 상품과 별도의 마감 상태를 가져요. 전체 마감 시 프로모션 상품이 별도로 열려 있는지 반드시 확인하세요. 야놀자 파트너센터에서 모든 요금제의 마감 상태를 개별적으로 확인해야 해요.
  </Accordion>

  <Accordion title="프로모션 기간에만 요금을 변경하고 싶어요">
    VCMS의 기간별 요금 설정을 활용하세요. 프로모션 기간의 시작일/종료일과 해당 요금을 미리 설정해두면, 기간이 되면 자동으로 프로모션 요금이 적용되고 기간이 끝나면 원래 요금으로 돌아와요.
  </Accordion>

  <Accordion title="네이버 오늘의 딜이 VCMS 연동 후 사라졌어요">
    VCMS와 네이버를 동기화하면 네이버에서 직접 설정한 오늘의 딜, 공휴일 요금 등이 삭제될 수 있어요. 동기화 전에 네이버 설정을 캡처해두고, 연동 후 VCMS에서 재설정하세요.
  </Accordion>
</AccordionGroup>
