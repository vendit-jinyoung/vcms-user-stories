## 이런 상황 아닌가요?

> "예약이 들어올 때마다 직접 채널 하나하나 재고를 줄이고 있어요. 깜빡하면 오버부킹이 날 것 같아서 항상 긴장돼요."

- 채널마다 재고를 **수동으로 쪼개서** 올려두고 있는데 실수가 생겨요
- 워크인 손님을 위한 재고를 따로 빼두고 싶은데 방법을 모르겠어요
- 성수기에 예약이 한꺼번에 몰릴 때 자동으로 방을 막아주길 바라요
- 재고 연동 없이 예약만 받다가 오버부킹이 반복되고 있어요

---

## VCMS에서 이렇게 해결해요

<Check>최대 판매 재고 설정으로 채널에 노출되는 수량에 자동 상한선을 걸 수 있어요</Check>
<Check>재고 자동 보충 기능으로 예약 발생 후에도 채널에 일정 수량이 항상 표시돼요</Check>
<Check>자동 방막기 기능이 실제 재고 소진 시점에 채널 판매를 자동으로 중단해요</Check>
<Warning>재고 연동 없이 예약 수신만 설정된 경우, 오버부킹 방지 기능이 작동하지 않아요. 반드시 재고 연동을 활성화해야 해요</Warning>

---

## 이렇게 설정하세요

<Steps>
  <Step title="재고 연동 활성화 확인">
    **채널 관리 → 연동 채널 목록**에서 각 채널의 재고 연동 상태를 확인해요.

    예약 수신(무료)만 설정된 채널은 재고가 자동으로 차감되지 않아요. 오버부킹 방지가 필요하다면 재고+요금 연동을 활성화해야 해요.
  </Step>
  <Step title="최대 판매 재고 설정">
    **상품 관리 → 객실 타입 선택 → 최대 판매 재고** 항목에서 채널에 노출할 최대 수량을 설정해요.

    예를 들어 실제 객실이 10개라도 최대 판매 재고를 8로 설정하면, 2개는 항상 워크인이나 비상용으로 남겨둘 수 있어요.
  </Step>
  <Step title="재고 자동 보충 설정">
    **상품 관리 → 재고 자동 보충**에서 예약 발생 시 채널에 표시할 재고 수량을 설정해요.

    예: 3으로 설정 시, 예약이 발생하더라도 채널에는 항상 3개가 있는 것처럼 표시돼요. 실제 재고가 소진되면 자동으로 판매가 중단돼요.

    <Tip>재고 자동 보충은 수요가 높을 때 채널에서 "품절" 표시 없이 판매를 유지하면서도 오버부킹을 방지하는 데 효과적이에요</Tip>
  </Step>
  <Step title="자동 방막기 활성화">
    **재고 설정 → 자동 방막기** 옵션을 활성화하면, 실제 재고가 0이 될 때 연동된 모든 채널에 자동으로 판매 중단 신호를 보내요.

    성수기처럼 예약이 빠르게 쌓이는 시기에 수동으로 채널을 막을 필요가 없어져요.
  </Step>
  <Step title="특정 채널 운영 중지로 버퍼 확보">
    판매 채널이 많을 경우, 일부 채널을 **운영 중지** 상태로 전환해 여유 재고를 확보할 수 있어요.

    **채널 관리 → 해당 채널 → 운영 중지** 설정 후, 필요한 시점에 다시 활성화해요. 재고 압박이 심한 성수기 직전에 특히 유용해요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="재고 자동 보충 설정값을 너무 크게 잡으면 어떻게 되나요?">
    채널에 표시되는 수량이 실제 재고보다 많아 보여 예약이 과다하게 들어올 수 있어요. 보통 1~3개 수준으로 설정하는 게 안전해요. 실제 재고가 소진되면 시스템이 자동으로 판매를 중단하지만, 그 사이 타이밍 차이로 오버부킹이 발생할 수 있어요.
  </Accordion>
  <Accordion title="최대 판매 재고와 실제 재고는 어떻게 다른가요?">
    실제 재고는 물리적으로 존재하는 객실 수예요. 최대 판매 재고는 채널에 팔 수 있도록 허용하는 상한선이에요. 최대 판매 재고를 실제보다 낮게 설정하면 차이만큼 워크인이나 비상 배정용으로 확보돼요.
  </Accordion>
  <Accordion title="재고 연동 없이 예약 수신만 해도 자동 방막기가 작동하나요?">
    작동하지 않아요. 자동 방막기는 VCMS가 재고 수량을 직접 제어할 때만 동작해요. 예약 수신 전용 설정에서는 VCMS가 채널의 재고에 영향을 줄 수 없어요.
  </Accordion>
  <Accordion title="채널 운영 중지와 판매 마감의 차이는 무엇인가요?">
    판매 마감은 특정 날짜의 재고를 0으로 만드는 것이고, 채널 운영 중지는 해당 채널 전체의 연동을 일시적으로 끊는 것이에요. 채널을 중지하면 그 채널에서는 더 이상 예약이 들어오지 않아요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>**펜션 — 성수기 자동 방막기 적용**

여름 성수기마다 채널별 재고를 직접 확인하며 막아왔던 업장이 자동 방막기를 설정한 이후, 마지막 방이 예약되는 순간 모든 채널이 동시에 마감되도록 바뀌었어요. 수동으로 채널을 돌아다니며 막는 작업이 사라졌고, 오버부킹 불안 없이 운영할 수 있게 됐어요.</Note>

<Note>**모텔 — 워크인 버퍼 확보**

다수의 OTA를 운영하면서 워크인 손님을 위한 재고를 항상 확보하고 싶었던 업장이 최대 판매 재고 설정으로 채널에 노출되는 수량에 상한선을 걸었어요. OTA 예약이 아무리 들어와도 현장 손님을 위한 방이 남아 있게 됐어요.</Note>
