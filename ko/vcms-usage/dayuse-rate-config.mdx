## 이런 상황 아닌가요?

> "우리 업장은 숙박보다 대실 매출이 훨씬 커요. 대실은 시간 단위로 예약이 몰리고, 가격도 시시때때 바꿔야 하는데 VCMS에서 대실 관리가 잘 안 돼요."

모텔이나 호텔 중 대실 비중이 높은 업장에서 자주 나오는 고민이에요.

- **대실 체크인/체크아웃 시간이 채널마다 달라요.** 야놀자에는 12시~18시, 여기어때에는 10시~20시로 다르게 들어가 있어요.
- **대실 전용 '당일 특가'를 랜덤 객실로 배정**하고 싶은데 설정 방법을 모르겠어요.
- **아고다와 네이버는 대실 시간 정보를 충분히 제공하지 않아요.** 예약이 들어와도 정확한 대실 시간을 파악하기 어려워서 운영에 혼란이 생겨요.
- **캠핑존을 대실로만 판매하는데 VCMS에서 재고 조절이 안 돼요.** 수동 관리가 필요해요.

## VCMS의 대실 지원 범위 — 솔직하게

<Warning>
**VCMS는 대실의 "예약 연동"만 지원해요.** 대실의 재고 관리, 요금 설정, 재고 자동 차감은 VCMS에서 할 수 없어요. 대실 재고와 요금은 각 채널 파트너센터에서 직접 관리해야 해요.
</Warning>

<Warning>
**숙박 상품의 입퇴실 시간 변경도 모든 채널에서 되는 게 아니에요.** 아래 표에 해당하는 채널·카테고리만 VCMS에서 입력한 시간이 적용돼요.
</Warning>

### 입퇴실 시간 연동 가능 채널

| 채널 | 카테고리 | 입퇴실 시간 연동 |
|------|---------|----------------|
| 여기어때 사장님앱 | 모텔 | ✅ 지원 |
| 야놀자 | 모텔, 호텔, 게스트하우스 | ✅ 지원 |
| 꿀스테이 | 모텔, 호텔, 펜션 | ✅ 지원 |
| 네이버 | 전체 | ❌ 미지원 — 파트너센터 직접 관리 |
| 아고다 | 전체 | ❌ 미지원 — 파트너센터 직접 관리 |
| 트립닷컴 | 전체 | ❌ 미지원 — 파트너센터 직접 관리 |
| 여기어때 파트너센터 | 펜션/호텔/글램핑 | ❌ 미지원 |

<Info>
입퇴실 시간 설정을 지원하는 채널만 VCMS에서 입력한 시간이 적용됩니다. 미지원 채널은 각 파트너센터에서 직접 확인하고 관리해야 해요.
</Info>

### VCMS에서 할 수 있는 것

<Check>
대실 **예약이 들어오면 VCMS에서 확인**할 수 있어요. 어떤 채널에서 대실 예약이 들어왔는지 통합 조회가 가능해요.
</Check>

<Check>
지원 채널(여기어때 사장님앱/야놀자/꿀스테이)의 **숙박 상품 입퇴실 시간**은 VCMS에서 설정하면 채널에 반영돼요.
</Check>

### VCMS에서 할 수 없는 것

- ❌ 대실 **재고** 관리 (자동 차감, 자동 보충 불가)
- ❌ 대실 **요금** 설정 및 채널 전송
- ❌ 대실 예약에 따른 **숙박 재고 자동 차감**
- ❌ 시간대별 대실 요금 차등 설정
- ❌ 미지원 채널(네이버, 아고다, 트립닷컴, 여기어때 파트너센터)의 입퇴실 시간 연동

## 대실 운영 실무 가이드

대실 재고와 요금은 VCMS가 아닌 **각 채널 파트너센터에서 직접 관리**해야 해요.

<Steps>
  <Step title="채널별 대실 설정 분리 관리">
    각 채널의 파트너센터에서 대실 요금과 시간을 직접 설정하세요. VCMS에서는 관리가 안 되므로, 채널별로 별도 관리가 필수예요.

    | 채널 | 대실 설정 위치 |
    |------|-------------|
    | 야놀자 (모텔) | 파트너센터 > 대실 관리 |
    | 여기어때 사장님앱 | 앱 > 대실 요금 설정 |
    | 여기어때 파트너센터 | 파트너센터 > 대실 설정 |
  </Step>

  <Step title="대실/숙박 전환 버퍼 시간 확보">
    대실 퇴실과 숙박 체크인 사이에 **청소 시간 버퍼**를 확보하세요. 대실 퇴실을 17:00으로 설정하면 1시간 버퍼가 생겨요.
  </Step>

  <Step title="대실 예약 발생 시 숙박 재고 수동 확인">
    대실 예약이 들어오면 VCMS에서 해당 날짜의 숙박 재고를 수동으로 확인하세요. 대실 예약이 숙박 재고에 자동으로 영향을 주지 않으므로, 중복 판매 위험이 있어요.
  </Step>

  <Step title="야간 전용 상품 채널 설정">
    야놀자/여기어때에서 야간 요금제가 노출되려면 해당 채널의 파트너센터에서 **야간 요금제를 '사용'으로 활성화**해야 해요. VCMS 설정만으로는 채널에 노출되지 않아요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="VCMS에서 대실 요금을 설정할 수 있는 거 아닌가요?">
    아니요. VCMS는 대실의 **예약 연동**만 지원해요. 대실 요금은 각 채널 파트너센터에서 직접 설정해야 해요. VCMS에서 보이는 대실 정보는 예약이 들어온 후의 확인용이에요.
  </Accordion>

  <Accordion title="대실 재고를 따로 카운팅하고 싶어요">
    VCMS에서 대실 재고를 직접 관리할 수는 없어요. 대실 비중이 높은 업장이라면, 채널 파트너센터에서 대실 재고를 관리하고, VCMS에서는 대실 예약이 들어올 때 숙박 재고와의 충돌을 수동으로 확인하는 방식으로 운영하세요.
  </Accordion>

  <Accordion title="'당일 특가' 대실을 랜덤 배정하고 싶어요">
    당일 특가 대실은 채널 파트너센터에서 설정하세요. 야놀자의 도보특가, 여기어때의 당일특가 등 각 채널 고유의 특가 기능을 활용하면 돼요. VCMS에서는 해당 특가 예약이 들어온 후 확인만 가능해요.
  </Accordion>

  <Accordion title="대실 피크 시간대에 요금을 올리고 싶어요">
    시간대별 차등 요금은 각 채널 파트너센터에서 설정하세요. 야놀자 모텔의 경우 **오전 대실과 오후 대실을 별도 요금제**로 설정할 수 있어요. 여기어때도 시간대별 대실 요금 설정을 지원해요.
  </Accordion>

  <Accordion title="아고다/네이버에서 대실 예약 시간 정보가 안 와요">
    아고다와 네이버는 대실 예약 시 정확한 체크인/체크아웃 시간 정보를 VCMS로 전달하지 않는 경우가 있어요. 이 경우 예약 확인 후 고객에게 직접 연락해서 시간을 확인해야 해요.
  </Accordion>
</AccordionGroup>

## 이 방식으로 해결된 사례

<Note>
**도심 모텔 C — "대실은 채널에서, 숙박은 VCMS에서 — 역할을 나눴더니 깔끔해졌어요"**

대실 매출이 전체의 상당 부분을 차지하는 업장. 처음에는 VCMS에서 대실까지 통합 관리하려 했지만, VCMS가 대실 재고/요금을 지원하지 않는다는 걸 알고 역할을 분리했어요. 대실은 야놀자·여기어때 파트너센터에서 직접, 숙박은 VCMS에서 통합 관리. "처음엔 불편했는데, 역할이 명확해지니까 오히려 실수가 줄었어요. 대실 예약이 VCMS에 연동되니까 전체 현황 파악은 가능하고요."
</Note>
