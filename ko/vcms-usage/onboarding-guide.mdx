## 이런 상황 아닌가요?

> "VCMS를 처음 시작하는데 어디서부터 해야 할지 모르겠어요. 순서가 있나요?"

처음 시작할 때 순서를 모르면 설정이 꼬이거나, 나중에 처음부터 다시 해야 하는 상황이 생겨요.

- **리피터를 설치했는데 채널 연동이 안 돼요.** 리피터 설치 후 추가 설정이 필요한 경우예요.
- **기존 CMS에서 VCMS로 전환하는데**, 기존 예약 데이터와 상품 설정을 어떻게 옮겨야 하는지 모르겠어요.
- **초기 설정을 잘못해서 네이버에서 오버부킹이 났어요.** 초기 세팅 미흡으로 발생하는 가장 흔한 사고예요.
- **OTA별 요금 설정이 너무 복잡해요.** 인원 요금, 아동 요금, 채널별 차등 요금 — 처음에 다 설정하려면 막막해요.

## VCMS 초기 세팅 순서

<Steps>
  <Step title="1단계: 리피터 설치 및 PC 환경 확인">
    VCMS는 리피터(Repeater) 프로그램을 통해 채널과 통신해요. 리피터가 설치된 PC는 **항상 켜져 있어야** 해요.

    <Warning>
    리피터가 설치된 PC가 꺼지면 채널 동기화가 중단돼요. 전용 PC를 지정하거나, 서버 PC를 사용하는 것을 권장해요.
    </Warning>

    - 리피터 설치 파일을 다운로드하고 설치하세요
    - 설치 후 리피터가 정상 실행되는지 확인하세요
    - 방화벽이나 보안 프로그램이 리피터를 차단하지 않는지 확인하세요
  </Step>

  <Step title="2단계: 업장 기본 정보 등록">
    VCMS에 업장 정보를 등록하세요.

    - 업장명, 주소, 연락처
    - 체크인/체크아웃 기본 시간
    - 객실 타입 목록 (스탠다드, 디럭스, 스위트 등)
    - 각 객실 타입의 실제 운영 가능 수량

    <Tip>
    객실 타입은 나중에 추가할 수 있지만, 처음부터 정확하게 등록하면 채널 매핑 작업이 훨씬 쉬워요.
    </Tip>
  </Step>

  <Step title="3단계: 기본 요금 설정">
    채널 연동 전에 기본 요금을 먼저 설정하세요. 요금 없이 채널을 열면 0원으로 노출될 위험이 있어요.

    | 설정 항목 | 내용 |
    |----------|------|
    | 평일 기본 요금 | 객실 타입별 평일 요금 |
    | 주말 요금 | 금·토 기준 요금 |
    | 성수기 요금 | 시즌별 요금 (나중에 추가 가능) |
    | 채널 전송가 | 채널별 수수료 반영 요금 |

    <Warning>
    아고다는 입금가(net rate) 구조예요. 다른 채널과 다르게 설정해야 해요. 아고다 채널 특이사항 가이드를 먼저 확인하세요.
    </Warning>
  </Step>

  <Step title="4단계: 채널 연동 (우선순위 순서)">
    모든 채널을 한 번에 연동하려 하지 마세요. 하나씩 연동하고 안정화를 확인한 후 다음 채널로 넘어가세요.

    **권장 연동 순서:**
    1. 야놀자 (가장 많은 예약 볼륨, 먼저 안정화 필요)
    2. 여기어때
    3. 네이버 (2FA 설정 복잡, 별도 시간 확보)
    4. 아고다 (입금가 구조 이해 후 연동)
    5. 트립닷컴
    6. 떠나요닷컴
    7. 온다 플러스

    <Info>
    채널 연동 기본 절차는 별도 가이드를 참고하세요.
    </Info>
  </Step>

  <Step title="5단계: 재고 초기 설정">
    채널 연동 후 재고를 설정하세요.

    - 각 객실 타입의 전체 재고 수량 확인
    - 채널 노출 재고(최대 판매 재고) 설정
    - 판매 기간(동기화 기간) 설정 — 처음에는 60일 권장
    - 자동 보충 기능 활성화 여부 결정

    <Warning>
    재고 설정 전에 기존 예약 현황을 반드시 확인하세요. 기존 예약이 있는 날짜에 재고를 잘못 설정하면 오버부킹이 발생해요.
    </Warning>
  </Step>

  <Step title="6단계: 테스트 예약 및 동기화 확인">
    실제 운영 전에 테스트를 진행하세요.

    - 각 채널에서 실제 예약이 VCMS에 정상 수신되는지 확인
    - VCMS에서 재고 변경 시 모든 채널에 반영되는지 확인
    - 요금이 의도한 대로 각 채널에 노출되는지 확인
    - 취소 시 재고가 자동 복원되는지 확인
  </Step>
</Steps>

## 기존 CMS에서 전환할 때 주의사항

<AccordionGroup>
  <Accordion title="기존 예약 데이터는 어떻게 되나요?">
    기존 CMS에서 VCMS로 전환할 때, 기존 예약 데이터는 자동으로 이전되지 않아요. 전환 전에 기존 예약 목록을 별도로 저장해두고, VCMS에서 수동으로 등록하거나 해당 날짜의 재고를 수동으로 차감해야 해요. 특히 야놀자, 여기어때 연동을 끊으면 해당 채널에 등록된 상품 정보와 예약 데이터가 소실될 수 있으니 주의하세요.
  </Accordion>

  <Accordion title="기존 채널 설정(요금, 상품명 등)을 백업해야 하나요?">
    네. 전환 전에 각 채널 파트너센터에서 현재 상품 설정(상품명, 요금, 입퇴실 시간, 인원 설정 등)을 스크린샷으로 저장해두세요. VCMS에서 채널 상품을 재매핑할 때 참고 자료가 돼요.
  </Accordion>

  <Accordion title="전환 중에 예약이 들어오면 어떻게 하나요?">
    전환 작업 중에는 채널을 일시적으로 마감하는 것이 안전해요. 특히 재고 초기 설정 중에 예약이 들어오면 재고 불일치가 생길 수 있어요. 예약이 적은 시간대(새벽)에 전환 작업을 진행하는 것을 권장해요.
  </Accordion>

  <Accordion title="초기 설정 후 오버부킹이 났어요">
    초기 설정 미흡으로 오버부킹이 발생하는 가장 흔한 원인은:
    1. 기존 예약을 반영하지 않고 재고를 설정한 경우
    2. 채널 연동 전에 해당 채널에서 수동으로 열어놓은 재고가 남아있는 경우
    3. 동기화 기간 밖의 날짜에 이전 요금이 남아있는 경우

    오버부킹 발생 시 즉시 해당 채널을 마감하고, 재고 현황을 처음부터 다시 확인하세요.
  </Accordion>

  <Accordion title="설정이 너무 복잡해서 어디서부터 해야 할지 모르겠어요">
    처음에는 기본 설정만 하고 운영하면서 점진적으로 고도화하는 것을 권장해요. 처음부터 모든 기능을 완벽하게 설정하려 하면 오히려 실수가 생겨요. 기본 요금 → 기본 재고 → 1~2개 채널 연동 → 안정화 확인 → 추가 채널 연동 순서로 진행하세요.
  </Accordion>
</AccordionGroup>

## 초기 세팅 체크리스트

<Note>
**이 항목들을 순서대로 완료하면 기본 운영이 가능해요:**

- [ ] 리피터 설치 및 PC 상시 가동 확인
- [ ] 업장 기본 정보 등록
- [ ] 객실 타입 및 수량 등록
- [ ] 기본 요금 설정 (평일/주말)
- [ ] 채널 전송가 설정 (채널별 수수료 반영)
- [ ] 1번 채널 연동 및 동기화 확인
- [ ] 재고 초기 설정
- [ ] 판매 기간 설정 (60일 권장)
- [ ] 테스트 예약 수신 확인
- [ ] 추가 채널 순차 연동
</Note>
