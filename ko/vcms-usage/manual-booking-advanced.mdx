## 이런 상황이신가요?

전화 예약을 받고 나서, 채널마다 들어가서 재고를 직접 닫고 계신가요?

- 전화로 장박 예약을 받았는데 **날짜별로 하나씩 재고를 막아야** 해서 번거로워요.
- 단체 예약이 들어오면 여러 객실을 **각각 개별로 등록**해야 해서 시간이 걸려요.
- 수기로 등록한 예약이 취소됐는데 **연동 채널에 재고가 자동으로 안 풀려요**.
- OTA 예약은 자동으로 들어오는데 전화·워크인 예약은 **별도로 관리해야 해서 현황 파악이 어려워요**.
- 지인 예약이나 특정 고객 요청으로 **객실을 직접 배정하고 싶은데** 자동 배정과 충돌해요.
- 전화 문의가 왔을 때 날짜별 요금이 바로 안 보여서 **수동으로 계산해 안내**하고 있어요.

---

## VCMS로 이렇게 해결할 수 있어요

VCLOUD의 수기 예약 등록은 단순 기록이 아니에요. **VCLOUD에서 수기 예약을 등록하면 연동 채널의 재고가 자동으로 차감**되고, 취소 시에는 재고가 자동으로 복원돼요. OTA 예약과 동일한 방식으로 통합 관리할 수 있어요.

<Check>
  수기 예약도 OTA 예약처럼 전체 재고에 즉시 반영돼요. 채널별로 따로 닫을 필요가 없어요.
</Check>

<Warning>
  에어비앤비·부킹닷컴 등 비연동 채널은 VCMS에서 자동 차감이 되지 않아요. 수기 예약 등록 후 해당 채널에서 직접 재고를 조정해야 해요.
</Warning>

---

## 이렇게 설정하세요

### 전화·워크인 예약 등록

<Steps>
  <Step title="1. 예약 추가 화면 이동">
    VCLOUD에서 **수기 예약 등록**을 클릭해요.
  </Step>
  <Step title="2. 예약 채널 선택">
    '전화 예약', '워크인', '직접 예약' 중 해당 유형을 선택해요. 채널 선택이 정확해야 나중에 채널별 예약 현황 파악이 쉬워요.
  </Step>
  <Step title="3. 객실 타입 및 날짜 입력">
    객실 타입을 정확히 선택해요. VIP 객실 예약인데 스탠다드로 잘못 등록하면 체크인 시 혼란이 생겨요.
  </Step>
  <Step title="4. 예약 확정 및 재고 자동 차감 확인">
    예약을 저장하면 연동 채널에서 해당 날짜의 재고가 자동으로 차감돼요.
  </Step>
</Steps>

### 장박 예약 등록

<Steps>
  <Step title="1. 체크인·체크아웃 날짜를 장박 기간으로 입력">
    1박짜리 예약을 여러 개 만들 필요 없이, **체크인 날짜와 체크아웃 날짜를 장박 기간 전체로** 설정해서 한 번에 등록해요.
  </Step>
  <Step title="2. 장박 요금 적용">
    장박 할인이 있다면 요금 입력 시 협의된 요금을 직접 입력해요. 날짜별 기본 요금과 다르게 적용할 수 있어요.
  </Step>
  <Step title="3. 장박 예약 아이콘 확인">
    등록 후 예약 현황 화면에서 장박 예약은 별도 아이콘으로 표시돼요. 일반 숙박 예약과 시각적으로 구분할 수 있어요.
  </Step>
</Steps>

### 단체 예약 등록

<Steps>
  <Step title="1. 객실별 개별 예약 생성">
    단체 예약은 객실 타입별로 **예약을 각각 생성**해요. 동일한 예약자명으로 여러 건을 등록할 수 있어요.
  </Step>
  <Step title="2. 메모란 활용">
    단체명, 담당자 연락처, 특이사항을 메모란에 기재해 두면 체크인 시 빠르게 파악할 수 있어요.
  </Step>
  <Step title="3. 전체 재고 반영 확인">
    모든 객실 등록 후 재고 현황에서 해당 날짜의 잔여 재고가 정확히 줄었는지 확인해요.
  </Step>
</Steps>

### 전략적 채널 재고 분배

<Steps>
  <Step title="1. OTA 노출 재고 조정">
    전화·직접 예약을 위한 여유 재고를 확보하고 싶다면, VCMS에서 OTA 채널에 노출되는 재고를 실제 보유 수보다 적게 설정해요.
  </Step>
  <Step title="2. 직접 예약용 버퍼 유지">
    예를 들어 스탠다드 객실이 8개라면 OTA에는 5개만 노출하고, 나머지 3개는 전화·워크인용으로 남겨 두어요.
  </Step>
  <Step title="3. 수수료 절감 효과 확인">
    직접 예약 비중이 높아지면 OTA 수수료 지출이 줄어요. 채널별 예약 현황 통계로 효과를 확인해요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="수기 예약을 취소했는데 채널 재고가 안 풀렸어요">
    연동 채널의 재고는 VCLOUD에서 수기 예약을 취소하면 자동으로 복원돼요. 단, 비연동 채널(에어비앤비, 부킹닷컴)은 자동 복원이 안 되므로 해당 채널에서 직접 재고를 오픈해야 해요.
  </Accordion>
  <Accordion title="전화 예약을 수기 등록했는데 왜 재고가 안 줄어들까요?">
    예약 저장 후 채널 반영까지 최대 2~5분이 소요돼요. 즉시 확인이 필요하면 판매 관리 화면에서 강제 동기화를 실행해요. 그래도 반영이 안 된다면 연동 상태를 확인해요.
  </Accordion>
  <Accordion title="객실 배정을 직접 하고 싶은데 자동 배정이 덮어써요">
    수기 예약 등록 시 특정 호실을 지정하면 자동 배정보다 우선해요. 배정 화면에서 원하는 호실을 직접 선택해 저장하면 해당 호실로 고정돼요.
  </Accordion>
  <Accordion title="전화 문의 시 날짜별 요금을 빠르게 안내하려면?">
    VCMS 요금 관리 화면에서 해당 날짜를 선택하면 기본가, 연휴가, 주말가 등 날짜별 요금이 표시돼요. 다박 예약이면 총 요금 합산도 화면에서 바로 확인할 수 있어요.
  </Accordion>
  <Accordion title="단체 예약인데 각각 예약 만들면 너무 번거롭지 않나요?">
    현재 VCLOUD 수기 예약은 객실 타입별로 개별 등록이 기본이에요. 동일 예약자명·연락처로 여러 건을 빠르게 입력하고, 메모란에 단체명을 기재하는 방식으로 운영하면 이후 관리가 편해요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
  **단체 직접 예약 운영 호텔 — "OTA를 거치지 않아도 재고가 자동으로 정리돼요"**

  기업 단체 예약을 전화로 받아 VCloud에 수기 등록하는 업장에서, 수기 예약 후 연동 채널 재고가 자동 차감되는 방식으로 전환했어요. OTA 수수료 없이도 전체 재고 현황을 한 곳에서 파악할 수 있게 됐어요.
</Note>

<Note>
  **전화 예약 비중 높은 펜션 — "전화받으면서 바로 요금 안내가 가능해졌어요"**

  날짜마다 요금이 다른 펜션에서, 전화 문의 시 요금 화면을 띄워 날짜별 금액을 즉시 확인하고 안내하는 방식으로 바꿨어요. 수동 계산 오류도 사라졌어요.
</Note>

<Note>
  **소규모 호텔 — "지인 예약도 시스템에 넣으니까 오버부킹 걱정이 없어요"**

  지인 예약이나 협약 업체 예약을 별도 메모로만 관리하다 오버부킹을 겪은 후, 모든 수기 예약을 VCLOUD에 등록하는 루틴으로 바꿨어요. 전체 예약 현황이 한 화면에 보여 운영이 편해졌어요.
</Note>
