## 이런 상황 아닌가요?

> "다음 주 수요일만 공사라서 막고 싶은데, 야놀자·여기어때·네이버를 다 들어가서 하나씩 막아야 해요."

방막기 하나에 채널 수만큼 반복 작업. 그리고 더 무서운 건, **막아놨는데 안 막혀 있는 사고**예요.

- VCMS에서 마감했는데 **야놀자에서 여전히 열려 있어서** 예약이 들어왔어요.
- 마감 처리한 객실이 **동기화 때 다시 열려서** 의도치 않은 예약이 생겼어요.
- VCMS 기본 동기화 기간이 180일인데, **한 달 단위로 오픈하고 싶어요.** 나머지 기간을 수동으로 막아야 해서 번거로워요.
- **가족(배우자)에게 방막기만 맡기고 싶은데** 시스템이 복잡해서 실수할까 걱정돼요.
- 사전 고지 없이 **전체 재고가 막힌 적**이 있어요. 큰 혼란이었어요.

## VCMS에서 이렇게 해결돼요

<Check>
VCMS에서 방막기를 하면 **연동된 모든 채널에 10초 이내로 반영**돼요. 야놀자, 여기어때, 네이버, 아고다에 따로 들어갈 필요 없어요.
</Check>

<Check>
**특정 날짜의 특정 객실 타입만 선택적으로 마감**할 수 있어요. "다음 주 수요일 스탠다드만 마감, 디럭스는 열어두기" 같은 세밀한 제어가 돼요.
</Check>

<Check>
VCMS에서 마감하면 **VCMS에서 오픈할 때까지 유지**돼요. 동기화 때 자동으로 풀리지 않아요.
</Check>

<Warning>
**VCMS에서 마감하지 않고 OTA에서 직접 마감하면**, 5분 주기 동기화 때 VCMS 값으로 되돌아가서 **다시 열릴 수 있어요.** 반드시 VCMS에서 마감하세요.
</Warning>

<Info>
VCMS에는 특정 채널에서 예약을 불러올 수 없을 때 오버부킹을 막기 위해 해당 채널의 판매를 일시적으로 자동 중지하는 기능이 있어요. 연동이 정상화되면 판매 상태도 자동으로 복구돼요. 이 기능은 수동으로 활성화하거나 비활성화할 수 있고, 예약 연동 여부와 상관없이 판매를 원하면 판매 활성화 버튼을 직접 눌러 진행할 수 있어요.
</Info>

<Note>
실제 사례로, 특정 OTA 채널이 보안 업데이트를 진행하면서 여러 업장의 예약 연동이 동시에 차단된 적이 있어요. 이럴 때는 해당 채널만 VCMS 판매관리에서 일시적으로 판매 중지하고, 연동이 복구된 뒤 다시 여는 방식으로 대응해요.
</Note>

## 이렇게 설정하세요

<Steps>
  <Step title="마감 대상 확정">
    어떤 날짜의 어떤 객실을 막을지 정하세요. 공사, 수리, 자가 사용, 전략적 마감 등 사유를 정리해두면 나중에 "왜 이 날이 막혀있지?" 하는 혼란을 방지해요.
  </Step>

  <Step title="판매 관리 캘린더에서 마감">
    판매 관리 캘린더에서 마감할 날짜와 객실 타입을 선택하고 **판매 중지**를 설정하세요. 전 객실 한 번에 마감도, 특정 타입만 마감도 가능해요.
  </Step>

  <Step title="채널별 반영 확인">
    마감 설정 후 야놀자, 여기어때, 네이버에서 실제로 마감 처리가 됐는지 확인하세요. 채널별로 표시 방식이 달라서 눈으로 직접 확인하는 게 안전해요.
  </Step>

  <Step title="재오픈이 필요하면 같은 경로로">
    마감 해제도 VCMS 판매 관리에서 해당 날짜의 판매 중지를 해제하면 돼요. 10초 이내에 전 채널에 반영돼요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="야놀자 방막기가 안 풀려요">
    야놀자 펜션은 **다중 재고 기능을 지원하지 않고**, 방 막기/열기만 가능해요. VCMS에서 오픈으로 바꿨는데 야놀자 파트너센터에서는 여전히 마감으로 보이는 버그가 보고되고 있어요. 이 경우 벤디트 지원팀에 야놀자 채널 동기화 강제 갱신을 요청하세요. 또한 야놀자가 과도한 API 요청으로 IP를 차단하는 경우도 있어서, 짧은 시간에 대량 변경은 피하세요.
  </Accordion>

  <Accordion title="특정 채널만 막고 다른 채널은 열어놓을 수 있나요?">
    VCMS에서 방막기를 하면 기본적으로 모든 연동 채널에 반영돼요. 하지만 특정 채널에만 노출되는 **별도 상품을 구성**하면 선택적 마감이 가능해요. 예를 들어 "아고다 전용 스탠다드"라는 상품을 만들고, 그 상품만 마감하면 아고다에서만 안 보여요.
  </Accordion>

  <Accordion title="방막기가 갑자기 풀려버렸어요">
    네이버 채널에 과거 수동 조작 이력이 남아 있으면 VCMS의 판매 중지가 제대로 작동하지 않을 수 있어요. 벤디트 지원팀에 **네이버 수동 설정 초기화**를 요청하세요. 또한 VCMS가 아닌 OTA에서 직접 마감한 경우, 5분 동기화 때 VCMS 값(열림)으로 되돌아가요.
  </Accordion>

  <Accordion title="한 달 단위로 오픈하고 싶어요">
    VCMS 기본 동기화 기간이 30~180일이에요. 당월+다음 달만 오픈하고 나머지를 마감해두는 전략을 쓰세요. 매월 초에 다음 달 분을 오픈하면 관리가 편해요. 현재 VCMS에서는 특정 요일을 무기한으로 마감하는 기능은 제공하지 않아요. 예를 들어 "매주 토요일 특정 OTA 채널 마감"처럼 운영하려면, 일정 기간을 정한 뒤 일괄 변경으로 처리해야 해요. 보통 1년 단위로 잡아 한 번에 반영해요. 판매 기간 설정 가이드에서 자세한 방법을 확인하세요.
  </Accordion>

  <Accordion title="가족에게 방막기만 맡기려면?">
    판매 관리 화면에서 "마감" 버튼만 사용하도록 안내하세요. 요금이나 설정을 건드리지 않고 마감/오픈만 하면 돼요. 실수해도 다시 오픈하면 바로 복원되니 부담 없이 사용할 수 있어요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**가족 운영 펜션 D — "와이프가 방막기를 직접 하게 됐어요"**

5실 규모의 펜션. 이전에는 사장님이 직접 야놀자, 네이버, 여기어때에 들어가서 하나씩 막았어요. 시간이 없으면 전화로 부탁해야 했고, 실수로 한 채널을 빠뜨려서 오버부킹이 2번 발생했어요. VCMS 도입 후 배우자에게 "여기서 마감 버튼만 누르면 다 막혀"라고 알려줬더니, 이제 사장님 없이도 방막기가 가능해졌어요. "오버부킹 스트레스에서 해방됐어요. 캘린더 보면서 클릭 한 번이면 끝."
</Note>
