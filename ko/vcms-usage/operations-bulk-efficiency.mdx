## 이런 상황이신가요?

성수기 전환 때마다 객실 타입별로 요금을 하나씩 바꾸고 계신가요?

- 비수기에서 성수기로 바뀔 때 **객실 타입마다 요금을 따로따로** 수정해야 해요.
- 프로모션 기간 설정을 5개 채널에 **각각 접속해서 동일한 작업을 반복**해요.
- 특정 기간의 재고를 막아야 할 때 날짜를 **하루하루 클릭해서 설정**하고 있어요.
- 카라반이나 다양한 상품 타입이 있는 곳에서 **타입별로 개별 처리**하느라 시간이 오래 걸려요.
- 실시간으로 요금을 바꿔야 할 때 채널 반영이 늦어져 **기회를 놓치는 경우**가 생겨요.

---

## VCMS로 이렇게 해결할 수 있어요

VCMS의 일괄 변경 기능은 **객실 타입·날짜 범위·채널을 한 번에 선택해서 동시에 적용**해요. 반복 작업 없이 하나의 화면에서 성수기 전환, 프로모션 적용, 기간 마감을 처리할 수 있어요.

<Check>
  일괄 변경 한 번으로 전체 객실 타입과 연동 채널에 동시에 반영돼요.
</Check>

<Warning>
  일괄 변경 전에는 변경 대상 범위를 반드시 확인해요. 잘못된 날짜 범위나 객실 타입을 선택하면 의도치 않은 요금이 전체 채널에 반영될 수 있어요.
</Warning>

---

## 이렇게 설정하세요

### 기간·객실 타입 일괄 요금 변경

<Steps>
  <Step title="1. 요금 관리 화면 이동">
    VCMS **요금 관리** 화면에서 일괄 변경 옵션을 선택해요.
  </Step>
  <Step title="2. 적용 범위 선택">
    변경할 **객실 타입(복수 선택 가능)**, **날짜 범위**, **요일 조건**을 설정해요. 예를 들어 8월 전체 주말만 선택하거나, 전 객실 타입에 동일하게 적용할 수 있어요.
  </Step>
  <Step title="3. 변경 요금 입력">
    새 요금을 입력하거나, 기존 요금 대비 **정률/정액 변경** 방식을 선택해요.
  </Step>
  <Step title="4. 채널 적용 확인 후 저장">
    저장하면 선택한 범위 전체에 동시에 적용되고, 연동 채널에 자동으로 반영돼요.
  </Step>
</Steps>

### 기간 일괄 재고 마감

<Steps>
  <Step title="1. 재고 관리 화면 이동">
    VCMS **재고 관리** 화면에서 마감할 기간을 범위로 드래그해서 선택해요.
  </Step>
  <Step title="2. 전체 또는 특정 타입 선택">
    전체 객실 타입을 닫거나, 특정 타입만 선택해서 마감할 수 있어요.
  </Step>
  <Step title="3. 재고 0 설정 후 저장">
    재고를 0으로 설정하면 모든 연동 채널에서 해당 기간이 판매 중지돼요.
  </Step>
</Steps>

### 재고 자동 보충으로 채널 노출 관리

<Steps>
  <Step title="1. 재고 자동 보충 설정 화면 이동">
    실제 보유 객실보다 적은 수를 채널에 노출하고 싶을 때 유용한 기능이에요.
  </Step>
  <Step title="2. 노출 재고 수량 설정">
    예를 들어 실제 객실이 10개여도 채널에는 **3개만 표시**되도록 설정할 수 있어요. 예약이 발생하면 자동으로 보충돼서 항상 설정한 수량만큼 보여요.
  </Step>
  <Step title="3. 희소성 효과 활용">
    남은 객실이 적어 보이면 예약 전환율이 높아지는 효과가 있어요. 성수기 프로모션 시 함께 활용해요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="일괄 변경 후 특정 채널만 안 바뀐 것 같아요">
    채널 동기화에는 최대 2~5분의 딜레이가 있어요. 일괄 변경 직후 채널 앱을 바로 확인하면 아직 반영 전일 수 있어요. 즉시 반영이 필요하면 판매 관리 화면에서 저장 후 해제로 강제 동기화를 실행해요.
  </Accordion>
  <Accordion title="요일 조건을 잘못 설정해서 평일 요금이 바뀌었어요">
    일괄 변경 시 '주말만', '평일만', '전체' 옵션을 꼭 확인해요. 변경 전 미리보기 화면에서 적용 범위가 맞는지 체크하는 습관을 들여요.
  </Accordion>
  <Accordion title="재고 자동 보충이 실제 재고보다 많게 설정되면 어떻게 되나요?">
    자동 보충 수량은 실제 보유 재고 이하로만 의미가 있어요. 실제 재고보다 높게 설정하면 오버부킹 위험이 생겨요. 항상 실제 가용 객실 수 이내로 설정해요.
  </Accordion>
  <Accordion title="프로모션 기간이 끝나면 요금이 자동으로 원복되나요?">
    일괄 변경으로 적용된 요금은 자동으로 원복되지 않아요. 프로모션 종료 시점에 맞춰 정상 요금으로 다시 일괄 변경하거나, 날짜 범위를 정확히 설정해서 기간이 끝나면 기존 요금이 유지되도록 미리 구성해 두어요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
  **카라반 펜션 — "비수기·성수기 전환 때마다 반나절 걸리던 설정이 10분으로 줄었어요"**

  다양한 카라반 타입을 운영하며 성수기마다 타입별로 개별 요금을 수정하던 업장에서, 일괄 변경 기능으로 전환 후 설정 시간이 크게 줄었어요. 오타로 인한 요금 오류도 사라졌어요.
</Note>

<Note>
  **도심 호텔 — "채널별로 따로 접속할 필요 없이 한 번에 다 됐어요"**

  여러 OTA를 동시에 운영하면서 프로모션 기간마다 채널별로 접속해 요금을 바꾸던 업장에서, VCMS 일괄 변경 한 번으로 모든 채널에 동시 반영되는 방식으로 전환했어요.
</Note>
