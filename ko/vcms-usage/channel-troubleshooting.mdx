## 이런 상황 아닌가요?

> "야놀자 재고를 VCMS에서 바꿨는데 야놀자 앱에서 변경이 안 됐어요. 내가 설정을 잘못한 건지, 야놀자가 이상한 건지 모르겠어요."

채널 연동 문제는 원인이 네 가지로 나뉘어요. 원인을 먼저 구분하면 대응이 빨라져요:

- **VCMS 설정 오류** — 상품 매핑 누락, 재고 설정 오류, 판매 기간 미설정
- **채널 계정 문제** — 파트너센터 비밀번호 변경, 2단계 인증 만료, 계정 잠금
- **채널 측 장애** — OTA 서버 이슈, 채널 정기 점검, API 응답 지연
- **연동 설정 충돌** — 이전 채널 매니저와 이중 연동, 트립닷컴 단독 연결 규칙 위반

이 네 가지 중 어디에 해당하는지 먼저 파악하는 것이 가장 중요해요.

## VCMS에서 이렇게 해결해요

<Check>
VCMS에서 동기화 오류가 발생하면 **알림과 함께 어느 채널에서 오류가 생겼는지** 표시돼요. 알림 내용을 보고 채널 계정 문제인지 설정 오류인지 1차 구분할 수 있어요.
</Check>

<Check>
채널 장애 중에도 **VCMS에 수기 예약을 입력**해서 운영 공백을 최소화할 수 있어요. 수기 입력된 예약은 채널 연동이 복구된 뒤 중복 여부를 꼭 확인하세요.
</Check>

<Warning>
채널 장애로 의심될 때 **동일한 설정을 반복해서 저장하거나 연동을 껐다 켰다** 하면 오히려 혼선이 생길 수 있어요. 상태 변화를 먼저 지켜보고, 변경은 원인이 확인된 뒤에 하세요.
</Warning>

<Warning>
네이버 채널 연동은 **2단계 인증 앱 비밀번호**를 VCMS에 입력해야 유지돼요. 비밀번호를 바꾸거나 인증 앱을 재설치하면 연동이 끊겨요. 변경 시 VCMS에서 재설정 필요해요.
</Warning>

## 원인별 빠른 진단

<Steps>
  <Step title="어느 채널에서 오류가 발생했는지 확인">
    VCMS 알림 또는 채널 관리 화면에서 오류 채널을 확인해요. 특정 채널 하나에서만 오류가 나면 해당 채널 설정 문제일 가능성이 높아요. 여러 채널에서 동시에 오류가 나면 VCMS 자체 이슈나 네트워크 문제를 먼저 의심해요.
  </Step>

  <Step title="VCMS 설정 오류 여부 확인">
    아래 항목을 순서대로 확인해요:

    - 해당 채널의 상품 매핑이 '미지정' 또는 '연결 없음' 상태인지
    - 판매 기간이 오늘 날짜를 포함하고 있는지
    - 재고가 0으로 설정되어 있지 않은지
    - 해당 채널 연동이 '활성' 상태인지

    이 중 하나라도 문제가 있으면 설정을 수정하고 재동기화를 시도해요.
  </Step>

  <Step title="채널 계정 상태 확인">
    해당 채널 파트너센터에 직접 로그인해서 계정 상태를 확인해요.

    - 로그인이 정상적으로 되는지
    - 비밀번호 변경 요청이나 보안 경고가 있는지
    - 네이버의 경우 2단계 인증 앱 비밀번호가 만료되지 않았는지

    계정 접근에 문제가 있으면 채널 파트너센터에서 먼저 해결한 뒤 VCMS에서 계정 정보를 재입력해요.
  </Step>

  <Step title="채널 측 장애 여부 확인">
    같은 시간대에 해당 채널 관련 이슈가 보고되고 있는지 확인해요. 채널 측 장애는 보통 수 시간 내에 자동 해소돼요.

    채널 장애 중에는:
    - VCMS 내부 설정을 반복 변경하지 않기
    - 예약 누락 대비해 수기 입력으로 임시 운영
    - 채널 정상화 후 최근 예약·재고·요금 반영 상태 확인
  </Step>

  <Step title="연동 충돌 여부 확인">
    과거에 다른 채널 매니저를 사용했거나, OTA 파트너센터에서 직접 API 연동을 설정한 적이 있는지 확인해요. 특히 트립닷컴은 **하나의 채널 매니저에만 연결**할 수 있어요. 이전 연동이 남아있으면 트립닷컴 담당자에게 기존 연동 해제를 요청하세요.
  </Step>

  <Step title="복구 후 상태 점검">
    연동이 복구된 뒤 반드시 아래를 확인해요:

    - 최근 24시간 예약이 VCMS에 모두 반영됐는지
    - 재고가 각 채널에 정확히 표시되는지
    - 수기 입력한 예약이 중복 처리되지 않았는지
  </Step>
</Steps>

## 채널별 자주 발생하는 오류

<AccordionGroup>
  <Accordion title="야놀자 — VCMS에서 재고를 바꿔도 야놀자 앱에 반영이 안 돼요">
    야놀자는 동기화 주기가 있어요. 변경 후 몇 분 내에 반영되지 않으면 야놀자 파트너센터에서 직접 확인해요. 야놀자 파트너센터에서 해당 상품이 활성 상태인지, 판매 가능 날짜 범위가 맞는지 확인하세요. 야놀자 측 장애인 경우 야놀자 고객센터에 문의해야 해요.
  </Accordion>

  <Accordion title="네이버 — 연동이 갑자기 끊겼어요">
    네이버 연동은 2단계 인증 앱 비밀번호를 주기적으로 갱신해야 해요. 비밀번호 변경, 인증 앱 재설치, 기기 변경 시 기존 비밀번호가 무효화돼요. VCMS 채널 관리에서 네이버 계정 정보를 재입력하세요. 재입력 후에도 연동이 안 되면 네이버 파트너센터에서 VCMS 연동 허용 상태를 확인하세요.
  </Accordion>

  <Accordion title="아고다 — 특정 날짜만 재고가 안 맞아요">
    아고다는 인원별 요금 설정이 있을 때, 인원 설정 값과 재고 설정이 맞지 않으면 특정 날짜의 재고가 정상적으로 동기화되지 않을 수 있어요. 아고다 YCS에서 해당 날짜의 객실 타입 설정을 확인하고, 인원별 요금이 모두 정상 설정되어 있는지 검토하세요.
  </Accordion>

  <Accordion title="여기어때 — 마감 처리를 했는데 여기어때에서 계속 열려 있어요">
    여기어때 파트너센터에는 상품별 판매 상태가 별도로 있어요. VCMS에서 재고를 0으로 줄여도 여기어때 파트너센터에서 해당 상품이 '판매 중' 상태면 계속 노출될 수 있어요. VCMS와 여기어때 파트너센터 양쪽을 모두 확인하세요.
  </Accordion>

  <Accordion title="트립닷컴 — '이미 다른 곳에 연결된 객실 타입'이라고 나와요">
    트립닷컴 객실 타입은 동시에 하나의 채널 매니저에만 연결할 수 있어요. 이전 채널 매니저와의 연동이 완전히 해제되지 않으면 이 오류가 나요. 트립닷컴 담당자에게 이전 연동 해제를 요청하고, 해제 완료 후 VCMS에서 재연결하세요.
  </Accordion>

  <Accordion title="채널 오류가 해결됐는데 예약이 누락된 것 같아요">
    장애 구간에 들어온 예약은 VCMS에 자동으로 소급 반영되지 않을 수 있어요. 장애 발생 시간대에 각 채널 파트너센터에서 직접 예약 목록을 확인하고, VCMS에 없는 예약이 있으면 수기 입력하세요. 수기 입력 후 채널 측 예약과 VCMS 예약이 중복되지 않도록 확인하세요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**모텔 — "네이버 비밀번호 바꾼 게 연동 끊긴 원인이었어요"**

네이버 예약이 갑자기 VCMS에 반영이 안 된다는 걸 발견했는데, 원인을 몰라서 VCMS 재고 설정만 반복해서 확인했어요. 알고 보니 2주 전에 네이버 로그인 비밀번호를 바꾸면서 2단계 인증 앱 비밀번호도 초기화됐던 거였어요. VCMS에서 앱 비밀번호를 재입력하니 바로 연동이 복구됐어요.
</Note>

<Note>
**펜션 — "트립닷컴 이전 채널 매니저 연동을 해제하니 오류가 사라졌어요"**

트립닷컴을 VCMS에 연결하려는데 계속 '이미 연결된 객실 타입' 오류가 났어요. 1년 전에 다른 채널 매니저를 썼을 때 연동을 해제하지 않은 상태였어요. 트립닷컴 담당자에게 기존 연동 해제를 요청하고 3일 뒤 VCMS에서 재연결하니 정상 작동했어요.
</Note>
