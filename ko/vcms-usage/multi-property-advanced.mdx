## 이런 상황 아닌가요?

> "9개 지점을 하나의 관제실에서 관리해요. 직원들이 각자 다른 지점을 담당하는데, 계정이 꼬이지 않게 운영하는 게 제일 어려워요."

다지점 운영에서 권한 분리는 한 번 설정해 두면 되지만, **본사-지점 간 요금 정책과 채널 판매 구조**는 계속 관리해야 해요.

- **지점마다 채널을 따로 열고 닫아야 해요.** 1호점은 성수기 마감, 2호점은 열어두기 — 지점별로 판매 전략이 달라요.
- **계정을 두 개 만들어서 더 많은 상품을 팔고 싶어요.** 단일 계정으로는 상품 수 제한이 있어요.
- **본점과 지점의 요금 정책이 달라요.** 본점은 프리미엄 포지셔닝, 지점은 가격 경쟁력으로 가야 하는데 설정이 분리가 안 돼요.
- **에어비앤비 두 숙소를 하나의 시스템에서 보고 싶어요.** 공동 호스트 계정으로 운영 중인데 예약이 섞여요.
- **중앙에서 모니터링하고 싶은데** 직원들이 실수로 서로 지점 데이터를 건드려요.

## VCMS에서 이렇게 해결해요

<Check>
**지점별 독립 계정 구조**로 각 지점의 재고·요금·채널 설정을 완전히 분리해요. 마스터 계정에서는 전체 지점을 통합 조회할 수 있어요.
</Check>

<Check>
**채널별 판매 상품 매핑**을 지점별로 다르게 설정할 수 있어요. 본점은 일반 요금 + 프리미엄 상품, 지점은 할인 상품 중심으로 운영해요.
</Check>

<Check>
**작업 이력 기능**으로 어떤 직원이 어떤 지점에서 무엇을 변경했는지 추적할 수 있어요. 중앙 관제 환경에서 특히 유용해요.
</Check>

<Warning>
에어비앤비는 VCMS 연동이 준비 중이에요. 에어비앤비 다중 숙소는 현재 에어비앤비 자체 관리 화면에서 운영하세요.
</Warning>

## 이렇게 설정하세요

<Steps>
  <Step title="지점별 계정 목록 정리">
    운영 중인 모든 지점의 계정을 하나의 문서에 정리하세요.

    | 지점 | 계정 ID | 역할 | 담당자 |
    |------|---------|------|--------|
    | 본점 | account_main | 마스터 | 대표 |
    | 2호점 | account_2 | 지점 관리자 | 담당 매니저 |
    | 3호점 | account_3 | 지점 관리자 | 담당 매니저 |

    계정이 뒤섞여 있다면 지금이 정리할 때예요. 불필요한 계정은 비활성화하세요.
  </Step>

  <Step title="본사/지점 요금 정책 분리">
    지점마다 요금 정책이 다르다면, 각 지점 계정에서 **요금 설정을 독립적으로** 관리하세요.

    - 본점: 주말 프리미엄 요금, 성수기 요금 별도 설정
    - 지점: 비수기 할인 상품 운영, 도보특가 상시 유지

    마스터 계정에서는 전체 지점 요금을 비교 조회할 수 있어요.
  </Step>

  <Step title="중앙 관제실 직원 계정 배분">
    관제실 직원 수에 맞게 개별 계정을 만드세요. 직원 한 명당 하나의 계정이 원칙이에요.

    - 담당 지점이 정해진 직원: 해당 지점 계정만 접근 가능하도록 제한
    - 여러 지점을 순환하는 직원: 필요한 지점 계정을 개별 부여
    - 관제실장: 전체 지점 조회 권한 (요금 변경 없이)
  </Step>

  <Step title="판매 채널 지점별 독립 운영">
    같은 채널(야놀자, 여기어때 등)을 여러 지점에서 사용하더라도, **각 지점 계정에서 독립적으로** 연동 설정하세요.

    - 지점 A: 야놀자 연동 ON, 네이버 연동 ON
    - 지점 B: 야놀자 연동 ON, 네이버 연동 OFF (직접 예약 위주)

    재고와 요금은 각 지점 계정에서 별도로 관리돼요.
  </Step>

  <Step title="작업 이력 주기적 점검">
    VCMS 작업 이력 기능으로 **누가 언제 무엇을 바꿨는지** 확인하세요. 특히 요금 변경 이력을 주 1회 점검하면 실수나 무단 변경을 빠르게 발견할 수 있어요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="마스터 계정으로만 모든 지점을 설정하면 안 되나요?">
    마스터 계정에서 전체 조회는 가능하지만, **설정 변경은 해당 지점 계정에서 하는 게 안전해요.** 마스터 계정으로 모든 걸 하면 어떤 지점 데이터를 변경하는지 혼동이 생겨요. 지점별 계정으로 접속해서 변경하면 실수가 줄어요.
  </Accordion>

  <Accordion title="계정을 두 개 만들면 채널도 두 배로 연동할 수 있나요?">
    두 계정이 완전히 독립된 업장으로 설정되어 있으면, 각 계정에서 채널을 별도로 연동할 수 있어요. 다만 **같은 물리 객실을 두 계정에서 동시에 팔면 오버부킹이 발생해요.** 재고를 어떻게 분배할지 먼저 설계하세요.
  </Accordion>

  <Accordion title="여러 직원이 동시에 같은 지점을 관리하면 어떻게 되나요?">
    같은 객실의 재고를 동시에 변경하면 마지막에 저장한 값이 적용돼요. 충돌을 막으려면 **재고·요금 변경 권한을 지점당 한 명**에게만 부여하고, 나머지는 조회 권한만 주세요. 작업 이력에서 충돌 여부를 확인할 수 있어요.
  </Accordion>

  <Accordion title="모바일에서 여러 지점을 전환하며 볼 수 있나요?">
    모바일 웹(vcms.io)에서 각 지점 계정으로 로그인하면 해당 지점을 볼 수 있어요. 여러 지점을 동시에 보려면 브라우저 탭을 여러 개 열거나, 시크릿 모드를 활용하세요. 한 계정에 여러 지점이 묶여 있으면 지점 전환 기능으로 전환해요.
  </Accordion>

  <Accordion title="직원이 설날 같은 특정 기간에 객실을 수동으로 막고 싶어요">
    직원 예약이나 특별 기간 차단은 **재고를 0으로 설정**하거나 **방막기 기능**으로 처리할 수 있어요. 단, 장기 차단(3~6개월)은 해당 지점 관리자 권한이 필요해요. 일반 직원 계정은 조회만 가능하도록 제한하면 실수 차단이 가능해요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**9지점 운영 숙박 체인 — "중앙 관제실에서 9개 지점을 한눈에"**

송도 중앙 관제실에서 9개 지점을 운영하고 있어요. 처음에는 관제실 직원들이 같은 계정을 돌려 쓰다가 데이터 충돌이 잦았어요. 지점별 계정을 분리하고 담당 지점만 접근 가능하도록 권한을 설정했어요.

관제실장은 전체 조회 전용 계정, 각 담당 직원은 담당 지점 관리 계정. 작업 이력 기능으로 "누가 어느 지점의 재고를 건드렸는지" 추적이 가능해졌어요. 혼선이 크게 줄었어요.
</Note>

<Note>
**2계정으로 판매 채널 확장한 펜션 — "상품 수 제한을 계정 분리로 해결했어요"**

기존 계정으로는 판매 상품 수에 한계가 있었어요. 두 번째 계정을 만들어서 프리미엄 상품과 할인 상품을 각각 다른 계정에서 관리했어요. 재고는 물리 객실 수에 맞게 두 계정에 나눠 배분했어요. 오버부킹 없이 판매 채널을 효과적으로 확장했어요.
</Note>
