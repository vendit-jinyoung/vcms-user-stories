## 이런 상황 아닌가요?

> "VCMS에는 '스탠다드'가 하나인데, 야놀자에서는 4개 상품이 이 타입에 연결돼 있어요. 어떤 상품이 재고를 얼마나 쓰는지 감이 안 잡혀요."

채널마다 객실 상품 구조가 달라서 매핑이 꼬이는 상황들:

- 야놀자는 '도보특가', '오픈특가', '연박 2일' 등 **프로모션마다 상품이 추가로 생성**돼 하나의 객실 타입에 상품이 10개 넘게 붙는 경우가 있어요
- 아고다는 'Room Only', 'Breakfast Included' 처럼 **식사 포함 여부로 상품이 분리**돼 있어요. VCMS 객실 타입 구조와 맞추기 어려워요
- 트립닷컴에서 '스탠다드 더블'과 '스탠다드 트윈'을 **같은 VCMS 객실 타입에 연결하니 재고 차감이 이상하게** 돼요
- 내부에서 '흡연 가능' 객실을 추가했는데, 채널에는 별도 상품이 없어서 **어떤 상품에 붙여야 할지 모르겠어요**
- 야놀자 여기어때 동시에 '온돌 디럭스'를 팔고 있는데, 채널별로 재고 묶음 방식이 달라서 **한 채널에서 팔리면 다른 채널 재고가 안 줄어요**
- 채널 담당자 없이 **직접 OTA 파트너센터에서 상품을 생성**했다가 VCMS와 연결이 안 되는 상태가 됐어요

## VCMS에서 이렇게 해결해요

<Check>
VCMS 객실 타입 하나에 여러 채널의 여러 상품을 동시에 연결할 수 있어요. 이름이 달라도, 채널이 달라도 같은 물리 객실에 해당하면 같은 타입에 묶어요. 하나의 타입에서 재고가 차감되면 **모든 연결된 채널 상품이 동시에 업데이트**돼요.
</Check>

<Check>
채널에서 새 상품이 자동 생성되면 **VCMS에 '미지정' 알림**이 와요. 알림을 받는 즉시 올바른 객실 타입에 매핑하면 재고 차감 공백이 생기지 않아요.
</Check>

<Warning>
흡연 가능/금연 객실처럼 운영상 별도 관리가 필요한 경우 VCMS에도 **별도 객실 타입으로 분리**하는 것이 좋아요. 하나의 타입에 합쳐놓으면 재고 구분이 안 돼 실무에서 혼란이 생겨요.
</Warning>

<Warning>
트립닷컴의 '스탠다드 더블'과 '스탠다드 트윈'처럼 **물리적으로 다른 방은 VCMS에서도 별도 타입**으로 분리해야 해요. 같은 타입에 묶으면 재고가 합산 차감돼 오버부킹 위험이 생겨요.
</Warning>

## 이렇게 설정하세요

<Steps>
  <Step title="채널별 상품 목록 전체 수집">
    각 채널 파트너센터에서 현재 활성화된 상품 전체를 목록으로 뽑아요. 야놀자 파트너센터, 여기어때 파트너센터, 아고다 YCS, 트립닷컴 파트너센터 순서로 확인해요.

    체크 포인트: 상품명, 객실 타입 코드, 현재 연동 상태(연결됨/미지정)
  </Step>

  <Step title="물리 객실 기준으로 VCMS 타입 분류">
    같은 침대 타입, 같은 인테리어인 방끼리 VCMS 객실 타입을 하나로 묶어요. 운영상 구분이 필요한 경우(흡연/금연, 뷰 차이 등)는 별도 타입으로 분리해요.

    | 물리 객실 구분 | VCMS 타입명(예시) | 비고 |
    |--------------|----------------|------|
    | 스탠다드 온돌 금연 | 스탠다드 온돌 | 기준 타입 |
    | 스탠다드 침대 흡연 | 스탠다드 침대 흡연 | 별도 분리 |
    | 디럭스 (더블/트윈 겸용) | 디럭스 | 같은 타입 가능 |
  </Step>

  <Step title="채널별 상품을 VCMS 타입에 연결">
    VCMS 채널 상품 관리에서 각 채널 상품을 해당 VCMS 객실 타입에 매핑해요. 한 VCMS 타입에 여러 채널 상품이 연결되는 것은 정상이에요.

    - 야놀자 '일반실' + 야놀자 '도보특가' + 야놀자 '오픈특가' → 모두 '스탠다드 온돌' 에 연결
    - 아고다 'Standard Room Only' + 아고다 'Standard Breakfast' → 둘 다 '스탠다드 온돌' 에 연결 (요금은 상품별로 다르게 설정)
  </Step>

  <Step title="미지정 상품 즉시 처리">
    VCMS에서 '미지정' 알림이 오면 24시간 안에 처리해요. 해당 채널 파트너센터에서 상품 내용을 확인한 뒤 맞는 VCMS 타입에 연결해요.

    미지정 상태로 예약이 들어오면 재고가 차감되지 않아 오버부킹 위험이 바로 생겨요.
  </Step>

  <Step title="재고 동기화 테스트">
    매핑이 완료되면 테스트 날짜를 선택해 재고를 0으로 변경하고, 모든 연결된 채널에서 해당 날짜가 품절 상태로 뜨는지 확인해요. 특히 야놀자처럼 프로모션 상품이 많은 채널은 하위 상품까지 전부 확인해요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="야놀자 프로모션 상품이 자꾸 새로 생겨요">
    야놀자는 도보특가, 오픈특가, 얼리버드, 시즌 프로모션 등 이벤트가 생길 때마다 새 상품이 자동 생성돼요. VCMS에서 '미지정'으로 표시되는 이유예요. 생성될 때마다 매핑해야 하는 건 아니고, **새 상품이 생기면 알림이 오니 그때마다 처리**하면 돼요. 매핑을 빠뜨리면 그 상품으로 들어온 예약은 재고 차감이 안 돼요.
  </Accordion>

  <Accordion title="아고다 'Room Only'와 'Breakfast Included'를 같은 타입에 연결해도 되나요?">
    네, 가능해요. 물리적으로 같은 방이면 같은 VCMS 타입에 연결해도 돼요. 재고는 타입 기준으로 차감되고, **요금은 각 채널 상품마다 다르게 설정**할 수 있어요. 아고다에서 조식 포함 예약이 들어오면 VCMS에서는 해당 타입 재고 1개가 줄어요.
  </Accordion>

  <Accordion title="여기어때 '타입 묶음' 기능이 VCMS 매핑과 다른가요?">
    여기어때 파트너센터에는 '객실 타입 묶음' 기능이 있어요. 이 기능은 여기어때 내부에서 묶음 판매를 위한 설정이고, VCMS 매핑과는 별개예요. VCMS에서는 여기어때 묶음 안에 있는 **개별 상품마다 VCMS 타입에 매핑**해야 해요.
  </Accordion>

  <Accordion title="트립닷컴에서 '이미 연결된 객실 타입'이라고 오류가 나요">
    트립닷컴 상품은 하나의 채널 매니저에만 연결될 수 있어요. 이전에 다른 채널 매니저나 직접 연동 이력이 있으면 충돌이 생겨요. 트립닷컴 담당자에게 기존 연동 해제를 요청한 뒤 VCMS에서 다시 연결하세요.
  </Accordion>

  <Accordion title="상위 상품을 닫았는데 하위 채널 상품이 여전히 열려 있어요">
    일부 채널에서는 상위 상품 판매 상태 변경이 하위 프로모션 상품에 자동으로 적용되지 않아요. 특히 야놀자 프로모션 상품은 따로 확인해서 개별 마감 처리해야 해요.
  </Accordion>

  <Accordion title="흡연/금연 구분을 같은 타입에 묶어도 되나요?">
    운영 여건에 따라 다르지만, 흡연 가능 객실과 금연 객실이 **재고를 별도로 관리해야 한다면 VCMS에서도 별도 타입으로 분리**하는 것을 권장해요. 같은 타입에 묶으면 흡연 객실만 품절이어도 금연 객실 재고까지 함께 차감될 수 있어요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**모텔 운영업소 — "야놀자 미지정 알림을 무시하다가 오버부킹이 났어요"**

스탠다드 객실을 야놀자, 여기어때, 네이버에서 동시 판매 중이었어요. 야놀자에서 도보특가 상품이 생겼는데 알림을 놓치고 2주 동안 미지정 상태로 뒀다가, 해당 상품으로 예약 3건이 들어왔는데 VCMS 재고는 차감되지 않아 오버부킹이 발생했어요. 그 뒤로 미지정 알림이 오면 당일 처리하는 규칙을 세웠어요.
</Note>

<Note>
**펜션 — "흡연 가능 객실을 타입 하나로 관리하다가 원하지 않는 배정이 생겼어요"**

흡연 가능 객실 2개와 금연 객실 6개를 같은 '스탠다드' 타입으로 관리하고 있었어요. 재고가 8개인 타입 하나로 관리하니 흡연 객실에 금연 예약이 배정되는 일이 반복됐어요. 별도 타입으로 분리하고 각각 채널 상품을 재매핑하니 배정 혼선이 사라졌어요.
</Note>
