## 이런 상황 아닌가요?

> "오버부킹이 났어요. 지금 당장 어떻게 해야 하나요?"

오버부킹은 발생 즉시 빠르게 대응해야 피해를 최소화할 수 있어요.

- **재고가 0인데 예약이 들어왔어요.** 채널 동기화 지연이나 비연동 채널 예약이 원인일 수 있어요.
- **수기 예약을 VCMS에 등록하지 않아서** 채널에서 그 방이 계속 열려 있었어요.
- **청소 직원이 장기 숙박 객실을 공실 처리해서** 재고가 잘못 열렸어요.
- **시스템에서 -1로 표시**되는 오버부킹이 발생했어요.

<Warning>
이 가이드는 VCMS 채널 매니저 관점의 실무 대응 가이드예요. 오버부킹의 심리적·운영적 패턴에 대한 분석은 숙박 매거진의 "오버부킹, 왜 반복될까?" 아티클을 참고하세요.
</Warning>

## 오버부킹 발생 시 즉각 대응

<Steps>
  <Step title="즉시: 해당 채널 전체 마감">
    오버부킹이 확인되면 즉시 VCMS에서 해당 날짜의 모든 채널을 마감하세요. 추가 예약이 들어오는 것을 막는 게 최우선이에요.
  </Step>

  <Step title="5분 이내: 예약 현황 파악">
    어떤 예약이 충돌하는지 파악하세요.
    - VCMS에서 해당 날짜의 예약 목록 확인
    - 각 채널 파트너센터에서도 예약 확인
    - 수기 예약이 있다면 별도 확인
  </Step>

  <Step title="30분 이내: 고객 연락">
    나중에 들어온 예약의 고객에게 먼저 연락하세요. 빠른 연락이 리뷰 피해를 줄여요.

    | 대응 옵션 | 내용 |
    |----------|------|
    | 업그레이드 제공 | 더 좋은 객실로 무료 업그레이드 |
    | 인근 숙소 연결 | 동급 이상 숙소 예약 대행 |
    | 전액 환불 | 취소 처리 + 전액 환불 |
    | 보상 제공 | 다음 방문 할인 쿠폰 등 |
  </Step>

  <Step title="원인 파악 및 재고 정리">
    오버부킹 원인을 파악하고 재고를 정확하게 재설정하세요.
  </Step>
</Steps>

## 원인별 재발 방지 설정

<AccordionGroup>
  <Accordion title="원인 1: 수기 예약 미등록">
    **증상**: 전화/워크인 예약을 받았는데 VCMS에 등록하지 않아 채널에서 계속 열려 있었음

    **재발 방지**:
    - 전화 예약 즉시 VCMS 등록 루틴 수립
    - 프론트 직원 전원 교육
    - 또는 채널 노출 재고를 실제 재고보다 적게 설정하여 버퍼 확보
  </Accordion>

  <Accordion title="원인 2: 채널 동기화 지연">
    **증상**: VCMS에서 마감 처리했는데 채널에서 동기화가 늦어 추가 예약이 들어옴

    **재발 방지**:
    - 마감 처리 후 채널 파트너센터에서 실제 반영 상태 확인
    - 당일 예약이 활발한 시간대에는 VCMS 마감 + 채널 직접 마감 병행
    - 야놀자는 동기화 지연이 특히 빈번하므로 주의
  </Accordion>

  <Accordion title="원인 3: 비연동 채널 예약 미반영">
    **증상**: 에어비앤비 등 비연동 채널 예약이 VCMS에 반영되지 않아 재고 불일치 발생

    **재발 방지**:
    - 비연동 채널 예약 즉시 VCMS 수기 등록
    - 또는 비연동 채널 운영 수량만큼 VCMS 노출 재고에서 미리 차감
  </Accordion>

  <Accordion title="원인 4: 재고 설정 오류">
    **증상**: 재고를 잘못 설정하거나, 객실 타입 매핑이 틀려서 실제보다 많은 재고가 열림

    **재발 방지**:
    - 객실 타입별 실제 운영 가능 수량 재확인
    - 채널별 상품 매핑 정확성 점검
    - 정기적으로 VCMS 재고와 실제 재고 대조
  </Accordion>

  <Accordion title="원인 5: 직원 실수로 재고 변경">
    **증상**: 청소 직원이나 야간 아르바이트가 실수로 재고를 변경하거나 마감을 해제함

    **재발 방지**:
    - 재고 변경 권한을 관리자 이상으로 제한
    - 직원별 계정 분리 및 최소 권한 부여
    - 다지점 권한 관리 가이드 참고
  </Accordion>
</AccordionGroup>

## 오버부킹 예방 체크리스트

<Note>
**주간 점검 항목:**
- [ ] VCMS 재고와 실제 운영 가능 객실 수 일치 여부
- [ ] 비연동 채널 예약이 VCMS에 모두 등록되었는지
- [ ] 채널별 동기화 상태 정상 여부
- [ ] 수기 예약 등록 누락 여부

**월간 점검 항목:**
- [ ] 객실 타입 매핑 정확성
- [ ] 채널 노출 재고 설정 적정성
- [ ] 직원 계정 및 권한 현황
</Note>
