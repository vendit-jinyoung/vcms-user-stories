## 이런 상황 아닌가요?

> "재고를 0으로 설정했는데 자꾸 1로 돌아가요. 자동 보충이 의도와 다르게 동작하는 것 같아요."

재고 자동 보충은 단순히 "취소되면 복원"이 아니에요. 동작 방식을 정확히 이해하지 않으면 의도치 않은 재고 변동이 생겨요.

- 재고를 0으로 설정했는데 **계속 설정값으로 돌아가요.** 자동 보충이 활성화된 상태에서 잔여 수량이 설정값보다 낮으면 자동으로 채워줘요.
- **채널마다 재고 수치가 달라요.** 같은 객실 타입에 여러 채널 상품을 연결했을 때 설정값이 서로 다르면 수량이 계속 바뀌어요.
- **아고다에 상품을 2개 연결했는데 재고가 불규칙하게 변해요.** 같은 객실 타입에 아고다 상품이 여러 개면 수량을 동일하게 맞춰야 해요.
- **1개씩만 노출하면서 팔고 싶어요.** 최대 판매 재고 설정과 재고 자동 보충을 함께 써야 해요.
- **180일치 재고가 자동으로 열려있어요.** 동기화 기간 설정과 구분해서 이해해야 해요.

## 재고 자동 보충의 실제 동작

<Info>
재고 자동 보충은 "연결된 모든 채널 상품의 잔여 수량을 보충해 재고를 일정하게 유지"하는 기능이에요. 취소나 체크아웃과 직접 연동되는 게 아니라, **잔여 수량이 설정값보다 낮아진 경우 설정값으로 채워주는 구조**예요.
</Info>

핵심은 두 가지예요:

1. **잔여 수량 → 설정값으로 보충**: 어떤 이유로든 잔여 수량이 설정값 아래로 내려가면 자동으로 설정값까지 채워줘요.
2. **설정값 = 실제 재고이면 판매 시 함께 차감**: 설정된 수량이 실제 재고와 같으면, 예약이 들어올 때 해당 수량이 차감돼요.

<Warning>
재고를 의도적으로 0으로 유지하고 싶다면 자동 보충을 끄거나, **판매 중지(방 막기)**를 사용하세요. 자동 보충이 켜진 상태에서는 잔여 수량이 설정값으로 계속 복구돼요.
</Warning>

## 최대 판매 재고와의 관계

재고 자동 보충과 함께 **최대 판매 재고** 설정을 쓰면 노출 수량을 제어할 수 있어요.

- **최대 판매 재고**: 판매 가능한 수량을 제한해요. 객실 타입의 실제 객실 수까지 설정할 수 있어요.
- **재고 자동 보충**: 잔여 수량을 설정값으로 일정하게 유지해요.

예를 들어, 실제 객실이 5개인데 채널에 2개씩만 노출하고 싶다면 최대 판매 재고를 2로 설정하면 돼요. 2개가 팔리면 잔여 수량이 0이 되고, 자동 보충이 다시 2개로 채워주는 흐름이에요.

## 설정 방법

<Steps>
  <Step title="재고 자동 보충 활성화">
    VCMS 상품 설정에서 재고 자동 보충을 켜세요. 상품별로 설정하는 항목이에요. 객실 타입 단위가 아니라 **각 채널 상품마다** 개별 설정이에요.
  </Step>

  <Step title="보충 수량 설정">
    잔여 수량을 유지할 목표 수량을 입력하세요. 이 값이 실제 재고와 같으면 예약 시 함께 차감돼요. 실제 객실 수를 초과해서 설정하지 않도록 주의하세요.
  </Step>

  <Step title="같은 객실 타입에 연결된 상품 수량 확인">
    같은 객실 타입에 여러 채널 상품이 연결된 경우, 각 상품의 재고 설정값을 동일하게 맞추세요. 수량이 다르면 잔여 재고가 설정값에 따라 계속 변동돼요.
  </Step>

  <Step title="의도적 마감은 판매 중지로 처리">
    특정 날짜나 기간을 판매하지 않으려면 자동 보충을 끄는 것보다 **판매 중지(방 막기)**를 사용하세요. 이 경우 자동 보충이 해당 날짜에 재고를 채우지 않아요.
  </Step>
</Steps>

## 아고다 다중 상품 연결 시 필수 주의사항

<Warning>
같은 객실 타입에 아고다 상품을 2개 이상 연결한 경우, **각 상품의 재고 수량을 반드시 동일하게 설정**해야 해요.

아고다는 같은 객실 타입에 연결된 상품들의 잔여 수량을 객실 타입 기준으로 조절해요. VCMS에서 각 상품의 설정값이 다르면, 잔여 수량이 서로 다른 설정값으로 계속 변경되어 재고 불일치가 발생해요.

재고 자동 보충을 사용하는 경우, 아고다 상품이 여러 개라면 **모든 상품의 수량을 동일하게** 맞추세요.
</Warning>

## 자주 묻는 것들

<AccordionGroup>
  <Accordion title="재고가 0인데 자꾸 설정값으로 돌아가요">
    재고 자동 보충이 켜져 있으면 잔여 수량이 설정값보다 낮아질 때마다 자동으로 채워줘요. 의도적으로 0을 유지하려면 해당 날짜를 **판매 중지(방 막기)**로 설정하거나, 자동 보충을 비활성화하세요.
  </Accordion>

  <Accordion title="야놀자 펜션에서 자동 보충이 적용되나요?">
    야놀자 펜션 카테고리는 숫자 기반 재고가 아니라 방 열기/닫기만 지원해요. VCMS의 재고 자동 보충이 적용되지 않아요. 야놀자 호텔/모텔 카테고리는 정상 지원돼요.
  </Accordion>

  <Accordion title="객실 타입마다 다른 수량으로 설정할 수 있나요?">
    네. 재고 자동 보충은 상품별로 설정하기 때문에 객실 타입마다, 상품마다 다르게 설정할 수 있어요. 단, 같은 객실 타입에 여러 상품이 연결된 경우 수량이 일치해야 재고 불일치가 생기지 않아요.
  </Accordion>

  <Accordion title="180일치 재고가 자동으로 열려있어요">
    이건 재고 자동 보충과 별개로 VCMS의 동기화 기간 설정이에요. 동기화 기간을 줄이면 그 기간 밖의 날짜는 자동으로 열리지 않아요. 판매 기간 설정 가이드를 참고하세요.
  </Accordion>

  <Accordion title="전화·수기 예약은 재고에 자동 반영되나요?">
    전화 예약은 VCMS에서 수동으로 예약을 등록해야 재고가 차감돼요. 등록하지 않으면 채널 재고와 실제 객실 점유가 맞지 않아 오버부킹 위험이 생겨요.
  </Accordion>
</AccordionGroup>
