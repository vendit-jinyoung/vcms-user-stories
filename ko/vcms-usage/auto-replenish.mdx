## 이런 상황 아닌가요?

> "예약이 취소됐는데, 채널에서 자동으로 다시 열리지 않아서 빈 방인데 못 팔았어요."

취소가 생기면 해당 객실을 다시 열어줘야 하는데, 이걸 수동으로 하면 반응이 늦어져요. 특히 당일 취소 같은 경우에요.

- 예약 취소 후 **야놀자에서는 다시 열렸는데 네이버에서는 안 열렸어요.** 채널별로 반영 시점이 다르니까 수동 확인이 필수였어요.
- 재고를 0으로 설정했는데 **1로 자꾸 돌아가요.** 자동 보충이 의도와 달리 동작하는 것 같아요.
- **2~3시간 단기 대실 취소 후** 자동으로 숙박 재고가 복원되면 좋겠어요.
- 1개씩 열어서 **마감 임박 효과를 주면서 판매**하고 싶어요.
- **180일치 재고가 자동으로 열려버렸어요.** VCMS 기본 설정이 180일치를 자동 오픈하는데, 한 달 단위로 관리하던 업장에서는 불필요한 수동 마감 작업이 생겨요.
- **디럭스는 2개, 스탠다드는 1개씩 자동 보충**하고 싶어요. 객실 타입마다 보충 수량을 다르게 설정할 수 있나요?

## VCMS의 재고 자동 보충 기능

<Check>
**체크아웃이 되면 자동으로 재고가 복원**돼요. 별도 조작 없이 빈 객실이 다시 판매 가능 상태로 돌아가요.
</Check>

<Check>
예약이 **취소되면 해당 채널뿐 아니라 모든 연동 채널의 재고가 동시에 복원**돼요. "야놀자는 열렸는데 네이버는 안 열림" 같은 불일치가 사라져요.
</Check>

<Check>
VCMS에서 **수동으로 마감한 객실은 자동 보충 대상에서 제외**돼요. 의도적으로 막아놓은 방이 취소건 때문에 열리는 일은 없어요.
</Check>

<Warning>
**야놀자 펜션 카테고리**에는 재고 자동 보충 개념이 없어요. 야놀자 펜션은 "방 열기/닫기"만 가능하고, 숫자 기반 재고 관리가 안 돼요. 이 경우 취소 시 수동으로 야놀자를 열어줘야 할 수 있어요.
</Warning>

## 이렇게 설정하세요

<Steps>
  <Step title="재고 자동 보충 활성화">
    VCMS 설정에서 재고 자동 보충 기능을 켜세요. 체크아웃·취소 시 자동으로 재고가 1개 복원되는 구조예요.
  </Step>

  <Step title="객실 타입별 전체 재고 확인">
    자동 보충이 정상 작동하려면 각 객실 타입의 **전체 재고 수량이 정확하게 설정**되어 있어야 해요. 공사 중이거나 사용하지 않는 객실은 빼고 설정하세요.
  </Step>

  <Step title="마감 객실과 자동 보충의 관계 확인">
    수동으로 마감한 객실은 자동 보충에서 제외돼요. 특정 날짜를 막아놓으면 취소가 나도 그 날짜는 열리지 않아요.
  </Step>

  <Step title="1개씩 리필 판매 패턴 설정 (선택)">
    실제 재고보다 적은 수량을 채널에 노출하면 "마감 임박" 효과를 줄 수 있어요. 5개 객실 중 1~2개만 노출하고, 팔리면 자동 보충으로 다음 1개가 열리는 패턴이에요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="재고가 0인데 자꾸 1로 돌아가요">
    자동 보충 기능이 활성화되어 있으면, 재고가 0이 된 후 체크아웃·취소가 발생하면 다시 1로 복원돼요. 의도적으로 재고를 0으로 유지하고 싶다면 **해당 날짜를 판매 중지(방막기)**로 설정하세요. 방막기한 객실은 자동 보충에서 제외돼요.
  </Accordion>

  <Accordion title="야놀자 펜션에서 자동 보충이 안 돼요">
    야놀자 펜션 카테고리는 숫자 기반 재고가 아니라 방 열기/닫기만 지원해요. VCMS의 재고 자동 보충이 적용되지 않아요. 야놀자 펜션의 경우 취소 발생 시 VCMS에서 해당 날짜를 수동으로 열어줘야 해요. 야놀자 호텔/모텔 카테고리는 정상 지원돼요.
  </Accordion>

  <Accordion title="대실 취소 후 숙박 재고도 복원되나요?">
    대실과 숙박의 재고가 연결되어 있는 경우, 숙박 재고가 모두 팔려서 대실도 닫혔다가 숙박 취소가 나면 대실도 다시 열릴 수 있어요. 반대로 대실 취소는 숙박 재고에 직접 영향을 주지 않아요.
  </Accordion>

  <Accordion title="홈페이지/전화 예약은 재고에 반영되나요?">
    홈페이지를 통한 예약이 VCMS에 연동되어 있지 않으면 재고로 인식되지 않아요. 전화 예약은 VCMS에서 수동으로 예약을 등록해야 재고가 차감돼요. 등록하지 않으면 오버부킹 위험이 있어요.
  </Accordion>

  <Accordion title="채널별 자동 보충 지원 현황">
    | 채널 | 자동 보충 지원 | 비고 |
    |------|-------------|------|
    | 야놀자 (호텔/모텔) | ✅ 지원 | 정상 작동 |
    | 야놀자 (펜션) | ❌ 미지원 | 수동 오픈 필요 |
    | 여기어때 | ✅ 지원 | 정상 작동 |
    | 네이버 | ✅ 지원 | 정상 작동 |
    | 아고다 | ✅ 지원 | 정상 작동 |
    | 트립닷컴 | ✅ 지원 | 정상 작동 |
    | 떠나요닷컴 | ✅ 지원 | 정상 작동 |
  </Accordion>

  <Accordion title="객실 타입마다 다른 수량으로 자동 보충할 수 있나요?">
    네. 객실 타입별로 자동 보충 수량을 다르게 설정할 수 있어요. 예를 들어 디럭스는 2개, 스탠다드는 1개로 설정하면, 취소가 날 때마다 해당 수량만큼 복원돼요. 단, 전체 재고 수량을 초과하는 보충은 되지 않아요.
  </Accordion>

  <Accordion title="180일치가 자동으로 열려서 수동 마감이 너무 많아요">
    VCMS 기본 설정은 180일치 재고를 자동으로 오픈해요. 한 달 단위로 관리하는 업장이라면 이 설정이 맞지 않을 수 있어요. 판매 기간 설정에서 동기화 기간을 30~60일로 줄이면, 그 기간 밖의 날짜는 자동으로 열리지 않아요. 판매 기간 설정 가이드를 참고하세요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**도심 호텔 E — "취소 후 5분 안에 전 채널에서 다시 팔리기 시작해요"**

20실 규모 호텔. 이전에는 취소가 나면 각 채널에 들어가서 재고를 수동으로 열어줬어요. 야간이나 주말에 취소가 나면 다음 날 아침까지 빈 방인데 판매가 안 되는 경우가 많았어요. VCMS 자동 보충 설정 후, 취소가 발생하면 모든 채널에서 자동으로 재고가 복원되니까 판매 기회를 놓치지 않게 됐어요. 특히 당일 취소가 많은 성수기에 효과가 컸어요.
</Note>
