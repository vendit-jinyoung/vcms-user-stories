## 이런 상황 아닌가요?

> "처음에는 마감용으로만 쓰다가, 익숙해지면 본격적으로 쓸 거예요. 근데 뭐부터 어떻게 시작해야 하는지 모르겠어요."

VCMS를 처음 도입하면 **기능이 너무 많아서 무엇을 먼저 해야 할지** 막막한 경우가 많아요.

- **일단 마감용으로만 써도 되나요?** 모든 채널 연동을 한꺼번에 하기가 부담돼요.
- **요금 설정을 처음부터 완벽하게 해야 하나요?** 일단 대충 넣고 나중에 고치려 했는데 채널에 잘못된 요금이 전송됐어요.
- **모바일에서도 쓸 수 있나요?** 앱이 따로 없어서 어떻게 접속해야 하는지 모르겠어요.
- **재고 자동 보충 설정을 켜야 하나요?** 자동으로 되면 편할 것 같은데 잘못 설정하면 오버부킹이 날까봐 무서워요.
- **미래 예약 가능 기간을 얼마나 열어야 하나요?** 채널별로 다르게 설정해야 하나요?
- **원격 지원을 받아서 초기 설정은 했는데**, 그 다음에 스스로 어떻게 관리해야 할지 모르겠어요.

## VCMS에서 이렇게 해결해요

<Check>
처음에는 **마감용으로만 사용하는 것도 정상적인 온보딩 방식**이에요. 채널 연동을 하나씩 추가하면서 익숙해지는 게 실수를 줄여요.
</Check>

<Check>
**모바일 웹(vcms.io)** 으로 스마트폰에서도 VCMS에 접속할 수 있어요. 별도 앱 설치 없이 크롬이나 사파리에서 바로 사용해요. 모바일에서 요금 변경도 가능하지만, **저장 버튼을 반드시 눌러야** 적용돼요.
</Check>

<Warning>
요금 설정은 초기에 **반드시 정확하게** 해야 해요. 잘못된 요금이 채널에 전송되면, 예약이 들어온 뒤에는 취소 수수료가 발생하거나 고객 분쟁이 생겨요. 요금을 입력하기 전에 기존 채널 요금과 비교하세요.
</Warning>

## 이렇게 설정하세요

<Steps>
  <Step title="1주차 — 기초 확인 (마감 전용 운영)">
    첫 주에는 **모든 채널을 연동하지 말고**, VCMS로 기존 예약을 확인하고 마감하는 연습을 하세요.

    **1주차 체크리스트:**
    - [ ] 마스터 계정으로 모든 지점 접속 확인
    - [ ] 모바일 웹(vcms.io) 접속 및 기본 화면 확인
    - [ ] 기존 예약 데이터가 정상적으로 보이는지 확인
    - [ ] 객실 타입, 요금 구조가 실제 운영과 일치하는지 점검
    - [ ] 방막기(재고 0) 기능 테스트
  </Step>

  <Step title="2주차 — 첫 채널 연동">
    가장 익숙한 채널 **하나만** 먼저 연동하세요. 야놀자나 여기어때처럼 메인 채널부터 시작하는 게 좋아요.

    **채널 연동 체크리스트:**
    - [ ] 채널 계정 정보 확인 (ID, API 키 등)
    - [ ] 객실 타입 매핑 설정 (VCMS 객실 ↔ 채널 상품)
    - [ ] 요금 전송 테스트 — 채널에서 실제 요금 반영 확인
    - [ ] 재고 동기화 테스트 — 방막기 후 채널에서 마감 확인
    - [ ] 예약 수신 테스트 — 테스트 예약이 VCMS에 들어오는지 확인

    **채널 연동 권장 순서:**
    1. 주력 채널 1개 (야놀자 또는 여기어때)
    2. 2번째 채널 (네이버)
    3. 나머지 채널 (아고다, 트립닷컴, 떠나요 등)
  </Step>

  <Step title="3주차 — 요금 정책 완성">
    채널 연동이 안정되면 **요금 정책을 체계적으로 설정**하세요.

    **요금 설정 체크리스트:**
    - [ ] 주중/주말 요금 분리 설정
    - [ ] 성수기/비수기 요금 기간 설정
    - [ ] 공휴일 추가 요금 설정 (설날, 크리스마스 등)
    - [ ] 채널별 요금 마크업 설정 (필요 시)
    - [ ] 최소 판매 기간 설정

    요금을 변경하면 연동된 채널에 자동으로 전송돼요. **대형 요금 변경 전에는 채널 화면에서 즉시 반영 여부 확인**을 습관화하세요.
  </Step>

  <Step title="4주차 — 재고 자동화 설정">
    수동 관리가 안정됐다면 **재고 자동화 설정**을 추가하세요.

    - [ ] 재고 자동 보충 방식 결정 (판매 시 자동 보충 or 고정 수량)
    - [ ] 채널별 최대 재고 상한선 설정
    - [ ] 오버부킹 방지 버퍼 설정 (전체 재고에서 1~2개 예비)
    - [ ] 자동 보충 설정 후 3일간 모니터링

    자동 설정을 켜고 나서 바로 방심하지 마세요. 3~5일은 수동 모니터링과 병행하세요.
  </Step>

  <Step title="첫 달 안정화 루틴 수립">
    첫 달이 지나면 **주간 점검 루틴**을 만드세요.

    **매일:**
    - 예약 현황 확인 (입실/퇴실 예정)
    - 채널 동기화 상태 확인

    **매주:**
    - 채널별 재고 상태 확인
    - 요금 정확성 점검 (채널 화면과 VCMS 비교)
    - 자동 메시지 발송 현황 확인

    **매월:**
    - 다음 달 성수기/이벤트 요금 미리 설정
    - 채널 수수료 설정 재확인
    - 사용자 계정 목록 점검
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="모든 채널을 한꺼번에 연동해도 되나요?">
    기술적으로는 가능하지만, **실수 가능성이 높아져요.** 채널 하나씩 연동하면서 재고/요금 동기화를 확인하는 게 안전해요. 한꺼번에 연동하면 어느 채널에서 문제가 생겼는지 찾기 어려워요. 2~3주에 걸쳐 하나씩 추가하세요.
  </Accordion>

  <Accordion title="모바일에서 요금을 바꿨는데 반영이 안 돼요">
    모바일 웹에서 변경 후 **저장 버튼을 눌러야** 적용돼요. 저장 없이 화면을 닫으면 변경이 사라져요. 모바일 환경에서는 저장 확인 팝업을 놓치기 쉬우니 주의하세요.
  </Accordion>

  <Accordion title="미래 예약 가능 기간을 얼마나 열어야 하나요?">
    일반적으로 **3~6개월**이 적당해요. 에어비앤비는 현재 VCMS 연동 준비 중이에요. 각 채널의 기본 설정이 다를 수 있으니, 채널 파트너 화면에서 현재 열려 있는 기간을 확인하고 VCMS 설정과 맞추세요. 너무 길게 열면 먼 미래 요금을 미리 설정해야 하는 부담이 생겨요.
  </Accordion>

  <Accordion title="재고 자동 보충을 켜면 오버부킹이 날 수 있나요?">
    설정을 정확하게 하면 오버부킹이 나지 않아요. 핵심은 **각 채널의 최대 재고 상한선**을 실제 물리 객실 수 이하로 설정하는 거예요. 전체 재고보다 1~2개 적게 설정해서 버퍼를 두는 게 안전해요. 설정 후 며칠간 모니터링하면서 이상이 없으면 안심하고 쓸 수 있어요.
  </Accordion>

  <Accordion title="원격 지원으로 초기 설정했는데, 혼자 관리하기가 무서워요">
    처음에는 **마감 기능과 예약 확인만** 하는 것부터 시작하세요. 요금 변경이나 채널 설정은 익숙해진 뒤에 하세요. VCMS 고객지원에 전화하면 설정 방법을 안내받을 수 있어요. 잘못된 설정을 되돌리는 방법도 있으니 너무 겁먹지 마세요. 모르면 변경하지 말고 먼저 문의하는 게 제일 안전해요.
  </Accordion>

  <Accordion title="객실 이름을 채널에 맞게 변경하고 싶어요">
    채널에 표시되는 객실 이름은 VCMS에서 직접 변경하거나, 변경 요청을 통해 처리할 수 있어요. 채널별로 노출되는 이름이 다를 수 있으니, 주력 채널 기준으로 이름을 통일하는 게 고객 혼선을 줄여요. 특히 야놀자, 여기어때, 네이버에서 보이는 이름을 일관성 있게 맞추세요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**독채 펜션 — "처음 2주는 마감만 했어요, 그 다음에 채널 연동했어요"**

VCMS를 처음 사용할 때 바로 모든 채널을 연동하지 않고, 처음 2주는 기존 예약 확인과 방막기 기능만 사용했어요. 시스템이 익숙해진 뒤에 야놀자부터 연동하고, 한 달 후에 여기어때를 추가했어요.

급하게 하지 않으니 실수가 없었어요. 지금은 5개 채널을 문제없이 운영하고 있고, 모바일로도 수시로 확인해요.
</Note>

<Note>
**소형 호텔 — "공휴일 요금을 미리 설정해뒀더니 설날에도 걱정이 없었어요"**

첫 달 온보딩 마무리 시점에 설날, 추석, 크리스마스 등 주요 공휴일 추가 요금을 미리 설정해뒀어요. 예전에는 공휴일 직전에 급하게 요금을 바꾸다가 잊어버리는 경우가 있었는데, 연간 일정을 한 번에 넣어두니 훨씬 안심이 됐어요. 성수기 요금도 자동으로 적용되니 수동 작업이 크게 줄었어요.
</Note>
