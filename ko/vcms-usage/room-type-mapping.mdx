## 이런 상황 아닌가요?

> "PMS에는 '스탠다드 온돌'인데 아고다에서는 'Standard Korean Room'이고 야놀자에서는 '일반실'이에요. 어떤 게 어떤 건지 모르겠어요."

객실 매핑이 틀리면 잘못된 방이 배정되거나 오버부킹이 나요. 가장 흔한 상황들:

- **아고다에서 호실별로 등록**했는데, VCMS 연동하려면 상품별(객실 타입별)로 바꿔야 해요. 기존 예약은 어떻게 되는 거죠?
- 매핑이 잘못돼서 **고객이 디럭스를 예약했는데 스탠다드가 배정**됐어요.
- 야놀자에서 프로모션마다 새 상품이 자동 생성돼서 **'미지정' 상품이 자꾸 생겨요.** 이 상태로 예약이 들어오면 재고 차감이 안 돼요.
- **아고다 인원수가 잘못 설정**돼 있어서 4인실인데 2인 기준으로 되어 있어요. 추가 인원 요금을 받을 수 없어요.
- **트립닷컴 연동 시 "이미 다른 곳에 연결된 객실 타입"이라고 오류**가 나요. 기존 연동이 남아있어서 충돌이 생기는 경우예요.
- **상위 상품 판매 상태를 바꿨는데 하위 채널 상품에 안 먹혀요.** 일부 채널은 상위 변경이 하위에 자동 반영되지 않아요.

## VCMS에서 이렇게 해결해요

<Check>
VCMS 상품에 각 채널의 상품을 매핑하면, **하나의 재고 풀에서 모든 채널의 재고가 통합 관리**돼요. 이름이 달라도 같은 객실 타입이면 같은 VCMS 상품에 연결하면 돼요.
</Check>

<Check>
**미지정 상품 알림**이 와서 빠르게 매핑할 수 있어요. 매핑하지 않은 상품으로 예약이 들어오면 재고 차감이 안 되니 즉시 처리하세요.
</Check>

<Warning>
아고다(YCS)에서 호실별로 등록한 경우, **VCMS 연동 전에 반드시 상품별(객실 타입별)로 전환**해야 해요. 호실별 등록 상태에서는 재고 동기화가 정상 작동하지 않아요.
</Warning>

## 이렇게 설정하세요

<Steps>
  <Step title="VCMS 객실 타입 정리">
    실제 운영 중인 객실 타입을 먼저 VCMS에 등록하세요. 스탠다드, 디럭스, 스위트 등. 흡연/금연, 온돌/침대 구분이 필요하면 별도 타입으로 분리하세요.
  </Step>

  <Step title="채널별 상품명 대조표 만들기">
    각 OTA 파트너센터에서 등록된 상품명을 확인하고, VCMS 객실 타입과 대응시키세요.

    | VCMS 타입 | 야놀자 | 여기어때 | 아고다 | 네이버 |
    |----------|--------|---------|--------|--------|
    | 스탠다드 온돌 | 일반실 | 스탠다드 | Standard Korean | 온돌 일반 |
    | 디럭스 트윈 | 디럭스 | 디럭스 | Deluxe Twin | 디럭스 (트윈) |
  </Step>

  <Step title="아고다 호실별→상품별 전환 (해당 시)">
    아고다 담당자(AM)에게 '상품별(Room Type) 등록으로 전환'을 요청하세요. 보통 1~2영업일 소요. 기존 예약은 유지돼요. **성수기를 피해서 진행**하세요.
  </Step>

  <Step title="VCMS에서 채널별 상품 매핑">
    채널 상품 관리에서 각 채널의 상품을 VCMS 상품에 연결하세요. 이름이 달라도 같은 객실 타입이면 같은 VCMS 상품에 매핑해요.
  </Step>

  <Step title="인원 설정 확인">
    각 채널의 기준 인원과 최대 인원이 VCMS와 일치하는지 확인하세요. 특히 **아고다는 인원 설정이 추가 요금에 직접 영향**을 줘요. 4인실이 2인 기준으로 되어 있으면 추가 요금을 못 받아요.
  </Step>

  <Step title="재고 동기화 테스트">
    하나의 상품 재고를 변경한 후, 모든 채널에서 동일하게 반영되는지 테스트하세요.
  </Step>

  <Step title="미지정 상품 즉시 처리">
    야놀자 프로모션 등으로 새 상품이 자동 생성되면 '미지정' 상태로 뜨어요. 알림이 오면 **최대한 빠르게 적절한 VCMS 상품에 매핑**하세요. 미지정 상태로 예약이 들어오면 재고 차감이 안 돼요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="아고다 호실별→상품별 전환, 기존 예약은 안전한가요?">
    아고다에서 전환하면 기존 예약은 유지돼요. 다만 전환 과정에서 일시적으로 채널이 닫힐 수 있으니, 예약이 적은 날에 진행하세요. 전환 완료 후 VCMS에서 아고다 상품을 재매핑해야 해요.
  </Accordion>

  <Accordion title="미지정 상품이 자꾸 생기는 이유">
    야놀자는 **프로모션마다 새 상품이 자동 생성**돼요. 도보특가, 오픈특가, 시즌 프로모션 등에서 기존에 없던 상품이 생기면 VCMS에서 '미지정'으로 뜨어요. 이 상태로 예약이 들어오면 재고 차감이 안 되니 **오버부킹 위험**이 생겨요. 미지정 알림을 놓치지 마세요.
  </Accordion>

  <Accordion title="상위 상품 판매 상태를 바꿨는데 하위에 안 먹혀요">
    일부 채널에서는 상위 상품의 판매 상태 변경이 하위 상품에 자동 반영되지 않아요. 하위 채널 상품도 개별 확인해서 수동으로 변경해야 하는 경우가 있어요.
  </Accordion>

  <Accordion title="아고다 인원수가 잘못 설정돼 있어요">
    아고다는 기준 인원과 최대 인원에 따라 추가 요금이 자동 계산돼요. 인원수가 잘못 설정되면 매출 손실이에요. 아고다 YCS에서 객실 타입별 기준 인원을 확인하고, 필요하면 아고다 담당자에게 수정 요청하세요.
  </Accordion>

  <Accordion title="트립닷컴에서 '이미 연결된 객실 타입' 오류가 나요">
    트립닷컴은 하나의 객실 타입이 하나의 연동에만 연결될 수 있어요. 이전에 다른 채널 매니저나 직접 연동으로 연결된 이력이 있으면 충돌이 생겨요. 트립닷컴 담당자에게 기존 연동 해제를 요청한 후 VCMS에서 재연동하세요.
  </Accordion>

  <Accordion title="상위 상품 판매 상태를 바꿨는데 하위 채널 상품에 반영이 안 돼요">
    채널에 따라 상위 상품의 판매 상태 변경이 하위 상품에 자동 반영되지 않는 경우가 있어요. 특히 야놀자에서 프로모션 상품이 별도로 생성된 경우, 상위 상품을 닫아도 프로모션 상품은 열려 있을 수 있어요. 각 채널 파트너센터에서 하위 상품 상태를 개별 확인하세요.
  </Accordion>

  <Accordion title="채널에서 예약된 객실 타입을 VCMS에서 변경할 수 있나요?">
    VCMS에서 객실 타입을 변경해도 이미 확정된 예약의 객실 타입은 바뀌지 않아요. 채널에서 들어온 예약의 객실 타입 변경은 해당 채널 파트너센터에서 직접 처리해야 해요. VCMS에서 변경하면 채널에 반영되지 않아 이중 배정 위험이 있어요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**호텔 G — "아고다 호실별 등록을 타입별로 바꾸니 동기화가 정상 작동했어요"**

15실 규모 호텔. 아고다에 101호, 102호, 201호… 식으로 호실별로 등록하고 있었는데, VCMS 연동하니 재고 동기화가 전혀 안 됐어요. 아고다 담당자에게 상품별 전환을 요청하고 1영업일 만에 완료. VCMS에서 재매핑한 뒤부터는 야놀자에서 예약이 들어오면 아고다 재고도 자동으로 줄어들게 됐어요. "가장 복잡하고 시간 걸린 건 아고다 전환이었는데, 그걸 넘으니 전부 해결됐어요."
</Note>
