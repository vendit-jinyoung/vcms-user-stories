## 이런 상황 아닌가요?

> "체크인을 3시로 바꾸고 싶은데, 야놀자·여기어때·네이버를 다 들어가서 바꿔야 하나요?"

입퇴실 시간 하나 바꾸려고 채널마다 접속하는 건 너무 비효율적이죠.

- **얼리체크인 상품(13시 체크인)과 일반 상품(15시 체크인)을 동시에 판매**하고 있는데, 채널마다 따로 시간을 관리해야 해요.
- VCMS에서 시간을 바꾸면 **5분마다 동기화되면서 채널에서 수동으로 바꾼 시간도 다시 원래대로** 돌아가요.
- **채널마다 입퇴실 시간 표기 방식이 달라요.** 아고다는 14:00/11:00, 야놀자는 오후 2시/오전 11시 — 고객이 혼란스러워해요.
- **레이트체크아웃을 별도 상품으로 판매**하고 싶은데 어떻게 설정하는지 모르겠어요.
- **늦은 입실 요금제를 비활성화했는데 시스템이 자동으로 다시 활성화**돼요. 연동 요금제 설정이 충돌하는 경우예요.
- **네이버 예약은 입퇴실 시간 설정이 안 돼요.** 네이버 스마트플레이스 특성상 입퇴실 시간이 고정되어 수동 관리가 필요한 경우가 있어요.

## VCMS에서 이렇게 해결해요

<Check>
VCMS에서 입퇴실 시간을 한 번 변경하면 **지원 채널에 자동 반영**돼요. 단, 모든 채널이 아니라 아래 채널만 지원해요.
</Check>

<Info>
**입퇴실 시간 연동 지원 채널:**
- 여기어때 사장님앱 (모텔)
- 야놀자 (모텔, 호텔, 게스트하우스)
- 꿀스테이 (모텔, 호텔, 펜션)

**미지원 채널 (파트너센터에서 직접 관리):**
- 네이버, 아고다, 트립닷컴, 떠나요닷컴, 온다 플러스, 여기어때 파트너센터(펜션/호텔/글램핑)
</Info>

<Check>
**상품별로 다른 입퇴실 시간**을 설정할 수 있어요. 일반 상품은 15:00 체크인, 얼리체크인 상품은 13:00 체크인으로 각각 설정 가능해요.
</Check>

<Warning>
입퇴실 시간 변경은 **변경 이후 새 예약부터 적용**돼요. 이미 확정된 예약의 시간은 바뀌지 않아요.
</Warning>

<Warning>
VCMS 동기화가 활성화된 상태에서 **OTA에서 직접 시간을 바꿔도 5분 주기로 VCMS 값으로 다시 덮어씌워져요.** 시간 변경은 반드시 VCMS에서 하세요.
</Warning>

## 이렇게 설정하세요

<Steps>
  <Step title="객실 타입별 기본 시간 확인">
    현재 운영 중인 입퇴실 시간을 정리하세요.

    | 구분 | 체크인 | 체크아웃 |
    |------|--------|---------|
    | 숙박 일반 | 15:00 | 11:00 |
    | 얼리 체크인 | 13:00 | 11:00 |
    | 레이트 체크아웃 | 15:00 | 14:00 |
    | 대실 | 12:00 | 18:00 |
  </Step>

  <Step title="객실 관리에서 기본 시간 설정">
    VCMS 객실 관리에서 각 객실 타입의 기본 체크인/체크아웃 시간을 입력하세요.
  </Step>

  <Step title="얼리체크인/레이트체크아웃 별도 상품 구성">
    시간이 다른 상품은 **별도 상품으로 생성**하세요. "디럭스 얼리체크인(13시)"처럼 상품명에 시간을 명시하면 고객 혼란이 줄어요. 각 상품에 해당 시간을 설정하세요.
  </Step>

  <Step title="채널별 반영 확인">
    설정 후 각 채널에서 시간이 정확하게 표시되는지 확인하세요. 채널마다 시간 표기 형식(24시/12시)이 달라요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="채널마다 시간이 다르게 보여요">
    VCMS에서 동일하게 보냈어도 채널별로 표기 방식이 달라요. 아고다는 24시간제, 야놀자는 12시간제(오전/오후)를 쓰는 경우가 있어요. 또한 일부 채널은 **입퇴실 시간이 고정**돼서 VCMS 설정이 반영되지 않을 수도 있어요. 반영 안 되는 채널은 파트너센터에서 직접 확인하세요.
  </Accordion>

  <Accordion title="레이트체크아웃을 점유율에 따라 다르게 팔고 싶어요">
    예를 들어 "공실이 많으면 스탠다드 +3만원, 적으면 디럭스 +3만원"식으로 동적으로 운영하고 싶다면, **별도 상품을 만들어서 수동으로 판매 가능/불가를 전환**하는 방법밖에 없어요. 자동화는 현재 지원되지 않아요.
  </Accordion>

  <Accordion title="시즌별로 입퇴실 시간을 다르게 하고 싶어요">
    VCMS에서 기간별로 다른 시간을 설정할 수 있어요. 성수기에는 체크인을 14시로 앞당기고, 비수기에는 15시로 늦추는 식으로요. 시즌 전환 시점에 미리 설정해두면 자동으로 바뀌어요.
  </Accordion>

  <Accordion title="대실과 숙박 체크인 시간이 겹쳐요">
    대실 퇴실(18:00)과 숙박 체크인(15:00) 사이에 **청소 시간 버퍼**를 확보하세요. 대실 퇴실을 17시로 설정하면 1시간 버퍼가 생겨요. 대실 시간 설정은 대실 요금 설정 가이드에서 자세히 확인하세요.
  </Accordion>

  <Accordion title="늦은 입실 요금제가 비활성화해도 자꾸 다시 켜져요">
    연동 요금제가 활성화된 상태에서 늦은 입실 요금제를 비활성화하면, 동기화 시 다시 활성화될 수 있어요. 연동 요금제를 먼저 해제한 후 늦은 입실 요금제를 비활성화하세요. 순서가 중요해요.
  </Accordion>

  <Accordion title="네이버 예약의 입퇴실 시간을 변경할 수 없어요">
    네이버 스마트플레이스는 입퇴실 시간 설정이 제한적이에요. VCMS에서 설정한 시간이 네이버에 반영되지 않는 경우, 네이버 파트너센터에서 직접 확인하고 수동으로 관리해야 해요. 네이버는 채널 특성상 일부 설정이 VCMS 동기화 범위 밖에 있어요.
  </Accordion>

  <Accordion title="얼리체크인 설정을 닫으면 '오픈 판매 중지' 느낌표가 떠요">
    얼리체크인 상품을 닫으면 해당 상품의 판매가 중지되면서 VCMS에서 경고 표시가 뜰 수 있어요. 이건 오류가 아니라 판매 중지 상태를 알려주는 정상 알림이에요. 얼리체크인을 다시 열면 사라져요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**도심 호텔 I — "얼리체크인 상품을 별도로 만들어서 객단가를 올렸어요"**

기존에는 단순히 체크인 15시 고정이었는데, "13시에 들어갈 수 없냐"는 문의가 자주 왔어요. VCMS에서 "얼리체크인 13시" 상품을 +2만원으로 만들어서 야놀자·여기어때에 올렸더니, 주말 기준 20% 예약이 얼리체크인으로 들어와요. "체크인 시간 하나로 객단가를 올릴 수 있다는 걸 몰랐어요. 설정도 5분이면 끝나요."
</Note>
