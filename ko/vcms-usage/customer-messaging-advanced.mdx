## 이런 상황 아닌가요?

> "입실 하루 전에 체크인 안내 문자를 항상 보내야 하는데, 매일 수동으로 찾아서 보내고 있어요. 예약이 많은 날에는 빠뜨리기도 해요."

자동 메시지 기능이 있다는 건 알지만, **어떤 시점에 무슨 내용을 보내야 효과적인지** 헷갈리는 경우가 많아요.

- **예약이 생기면 즉시 확인 문자**가 가야 하는데, 수동으로 보내고 있어요.
- **체크인 당일 안내 문자**를 빠뜨리면 고객이 비밀번호를 몰라서 전화해요. 밤에 전화 받기 싫어요.
- **퇴실 안내(분리수거, 수건 정리)** 를 미리 보내면 퇴실 처리가 훨씬 편한데, 매번 기억해서 보내기가 어려워요.
- 채널에 따라 **고객 연락처가 없어서** 문자를 못 보내는 경우가 있어요.
- 기존 PMS에서 자동 문자를 쓰고 있었는데, VCMS로 넘어오면서 이 기능을 어떻게 설정해야 할지 모르겠어요.

## VCMS에서 이렇게 해결해요

<Check>
**예약 생성 시**, **입실 N시간/일 전**, **퇴실 예정 시간 전** 등 특정 시점에 자동으로 문자가 발송돼요. 한 번 설정해 두면 매일 수동으로 보낼 필요가 없어요.
</Check>

<Check>
메시지 템플릿에 **예약자명, 객실명, 체크인 날짜, 비밀번호** 등 예약 정보를 자동으로 삽입할 수 있어요. 개인화된 메시지를 자동으로 발송해요.
</Check>

<Warning>
자동 문자는 **고객 연락처가 VCMS에 저장된 경우에만 발송**돼요. 일부 채널은 고객 전화번호를 제공하지 않거나 채널 자체 메시징 시스템을 사용해요. 채널별 연락처 수신 여부를 먼저 확인하세요.
</Warning>

<Warning>
VCMS 자동 메시지는 SMS 발송 기반이에요. **카카오 알림톡은 VCMS 기본 기능 범위 밖**이에요. 카카오 알림톡이 필요하다면 별도 서비스 연동을 검토하세요.
</Warning>

## 이렇게 설정하세요

<Steps>
  <Step title="발송 시점별 메시지 계획 수립">
    어떤 시점에 어떤 내용을 보낼지 먼저 정리하세요.

    | 발송 시점 | 주요 내용 | 권장 여부 |
    |----------|----------|---------|
    | 예약 생성 직후 | 예약 확인, 체크인 정보 안내 | 필수 |
    | 입실 1~2일 전 | 체크인 시간, 위치, 주차, 비밀번호 | 권장 |
    | 입실 당일 | 비밀번호/키박스 안내, 체크인 절차 | 권장 |
    | 퇴실 전날 | 퇴실 시간, 분리수거, 정리 안내 | 선택 |
    | 퇴실 후 | 리뷰 요청 (채널 정책 확인 필요) | 선택 |
  </Step>

  <Step title="메시지 템플릿 작성">
    각 시점별 템플릿을 작성하세요. 너무 길면 읽지 않아요 — **핵심 정보만 간결하게** 담는 게 중요해요.

    **체크인 안내 템플릿 예시:**
    ```
    [숙소명] 체크인 안내
    안녕하세요, {예약자명}님.
    내일 체크인 날이에요.
    - 체크인: {체크인시간}
    - 객실: {객실명}
    - 비밀번호: {비밀번호}
    문의: {연락처}
    ```

    변수로 처리되는 항목({예약자명} 등)은 VCMS에서 자동으로 채워져요.
  </Step>

  <Step title="VCMS 자동 메시지 설정">
    VCMS 설정에서 **자동 메시지 발송** 항목을 찾아 각 시점별 템플릿을 등록하세요.

    - 발송 조건(예약 생성, 입실 N일 전 등) 선택
    - 메시지 템플릿 입력
    - 발송 대상 채널 선택 (전체 또는 특정 채널)
    - 테스트 발송으로 실제 수신 확인
  </Step>

  <Step title="채널별 연락처 수신 여부 확인">
    자동 메시지가 실제로 전달되려면 고객 연락처가 있어야 해요.

    - **야놀자/여기어때:** 예약 시 고객 연락처 포함 — 자동 발송 가능
    - **네이버:** 대부분 연락처 포함
    - **아고다/트립닷컴:** 연락처 정책이 다를 수 있어요 — 채널 파트너 정책 확인
    - **에어비앤비:** 에어비앤비 자체 메시지 시스템 사용, VCMS SMS 별도 불가 (현재 에어비앤비 VCMS 연동 준비 중)
  </Step>

  <Step title="발송 현황 주기적 점검">
    자동 메시지가 설정한 대로 발송되고 있는지 주 1회 확인하세요. 특히 연락처가 없어서 발송 실패한 예약은 수동으로 따로 연락해야 해요.
  </Step>
</Steps>

## 자주 틀리는 부분

<AccordionGroup>
  <Accordion title="자동 메시지를 설정했는데 발송이 안 돼요">
    가장 흔한 원인은 **고객 연락처 미수신**이에요. 해당 채널이 예약 시 연락처를 제공하는지 확인하세요. 또 VCMS에 발신번호가 등록되어 있는지, SMS 잔량이 충분한지도 확인하세요.
  </Accordion>

  <Accordion title="같은 고객에게 문자가 중복 발송돼요">
    예약 수정이 발생하면 새 예약으로 처리되면서 메시지가 다시 발송될 수 있어요. 예약 수정 시 자동 발송 조건을 재확인하고, 중복 발송이 잦은 시점의 트리거를 조정하세요.
  </Accordion>

  <Accordion title="채널에서 이미 자동 메시지를 보내는데 VCMS에서도 보내면 고객이 두 번 받아요">
    맞아요. 야놀자, 여기어때 등 채널에서도 예약 확인 메시지를 보내요. VCMS 자동 메시지는 채널 메시지와 **내용이 다를 때만** 설정하는 게 좋아요 — 예를 들어 채널 메시지에 없는 비밀번호/키박스 안내, 숙소 자체 안내사항 등이에요.
  </Accordion>

  <Accordion title="퇴실 후 리뷰 요청 문자를 보내도 되나요?">
    가능하지만 **채널 정책을 먼저 확인**하세요. 일부 채널은 고객을 채널 외부로 유도하는 행위를 제한해요. 리뷰 요청은 "채널에서 리뷰를 남겨주시면 감사해요" 형태로 채널 내 리뷰를 유도하는 방식이 안전해요.
  </Accordion>

  <Accordion title="입실 당일에 보내는 문자가 너무 이르거나 늦게 와요">
    발송 시간을 조정하세요. 입실 당일 오전 10시에 발송하는 게 일반적으로 적당해요 — 너무 이른 새벽은 고객을 깨우고, 입실 1시간 전에 발송하면 못 본 경우에 전화 문의가 늘어요.
  </Accordion>
</AccordionGroup>

## 이 설정으로 해결된 사례

<Note>
**무인 운영 펜션 — "비밀번호 안내 자동화로 야간 전화가 사라졌어요"**

직접 체크인 없이 무인으로 운영하는 펜션이에요. 입실 당일 오전에 비밀번호와 주차 안내가 자동으로 발송되도록 설정했어요. 예전에는 체크인 시간에 맞춰 수동으로 문자를 보냈는데, 잊어버리면 고객에게 야간 전화가 오곤 했어요.

자동 발송으로 전환한 뒤 "비밀번호 알려주세요" 전화가 거의 없어졌어요. 주로 VCMS를 입실 전날 자동 문자 발송 용도로 활용하고 있어요.
</Note>

<Note>
**도심 호텔 — "퇴실 안내 문자로 분리수거 미이행이 줄었어요"**

퇴실 전날 저녁에 퇴실 시간, 분리수거 방법, 수건 정리 안내를 자동 발송하도록 설정했어요. 안내 전에는 퇴실 후 정리 상태가 들쭉날쭉했는데, 문자를 받은 고객들이 훨씬 협조적으로 퇴실하게 됐어요. 퇴실 점검 시간도 줄어들었어요.
</Note>
