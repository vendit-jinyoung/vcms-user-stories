# 통화에서 발견한 진짜 이야기

이 페이지의 콘텐츠는 실제 현장 상담과 운영 사례에서 출발합니다.

벤디트 팀이 현장에서 반복해서 마주한 운영 이슈, 온보딩 과정의 막힘, 채널별 문제, 가격 사고, 오버부킹 사례를 한데 모아 패턴으로 다시 정리한 것이 이 '활용 사례' 탭입니다.

여기에 있는 글들은 **기능 매뉴얼이 아닙니다.** "이 버튼을 누르세요"가 아니라, "왜 그 버튼을 눌러야 하는 상황이 생기는지", "그 상황에서 운영자들은 어떤 선택지를 고민하는지", "VCMS가 풀 수 있는 것과 풀 수 없는 것은 무엇인지"를 다룹니다.

단계별 설정 가이드가 필요하시면, 각 스토리 하단의 관련 가이드 링크를 따라가시면 됩니다.

## 이 데이터는 어떻게 만들어졌나

1. **통화 녹음 수집** — 벤디트 고객지원팀의 1,413건 통화 녹음 (개인정보 제거)
2. **STT + NLP 처리** — 음성을 텍스트로 변환하고, 개별 클레임(주장·요구·패턴)을 추출
3. **중복 제거** — 6,504개 원시 클레임에서 85개 중복을 제거하여 6,419개 고유 클레임 확보
4. **카테고리 분류** — 16개 주제 카테고리로 클레임을 분류
5. **패턴 분석** — 카테고리별 반복 패턴을 식별하고, 스토리 형태로 구성
6. **사실 검증** — VCMS 실제 기능과 대조하여 허위 기능 주장을 제거

모든 스토리는 실제 통화 데이터에 근거하며, VCMS에 존재하지 않는 기능을 있는 것처럼 서술하지 않습니다.
