# 야놀자 — 계정 및 카테고리

야놀자는 **호텔, 모텔, 펜션, 게스트하우스** 등 카테고리에 따라 시스템 동작 방식이 상당히 다릅니다. 카테고리를 잘못 인지하고 있으면 재고 관리, 가격 설정, 연동 방식 전반에서 문제가 발생합니다.

---

## 호텔/모텔 카테고리 차이

### 어떤 상황에서 많이 막히는지

- 업장이 야놀자에서 **호텔과 모텔 두 가지 카테고리를 동시에 운영**하는 경우, VCMS에서 이 구분이 되지 않음
- 야놀자 호텔과 모텔의 수수료율이 다른데, VCMS에서 예약이 '야놀자'로만 통합되어 들어와 수기로 구분해야 함
- 호텔 카테고리의 재고/요금 정보가 VCMS와 전혀 연동되지 않아, 가격 변경이나 재고 업데이트가 반영되지 않는 사례
- 호텔과 모텔 카테고리를 분리하여 가격과 입실 시간을 다르게 설정하려 하지만, VCMS에서 이를 구분 관리하기 어려움

### 왜 이런 일이 생기는지

야놀자는 내부적으로 호텔과 모텔을 **완전히 별도의 상품 체계**로 관리합니다. 수수료, UI, 상품 구조, 판매 정책이 모두 다릅니다. 그러나 VCMS는 야놀자 채널을 하나로 취급하므로:

- 호텔/모텔 예약이 혼재되어 정산 시 수기 구분이 필요
- 호텔 카테고리는 모텔 대비 연동 지원 범위가 제한적인 경우가 있음
- 업장이 야놀자 호텔 가격을 모텔보다 5% 높게 설정하고 싶어도, VCMS의 채널 전송가 비율 조정 기능이 카테고리별로 분리되지 않을 수 있음

<Warning>
  야놀자 호텔 카테고리를 사용하는 업장은 연동 전에 반드시 호텔 카테고리의 재고/요금 연동 가능 범위를 확인해야 합니다. 모텔 카테고리와 동일한 수준의 연동을 기대하면 문제가 발생할 수 있습니다.
</Warning>

### 운영자가 직접 확인할 순서

<Steps>
  <Step title="파트너센터 카테고리 확인">
    야놀자 파트너센터에서 업장이 어떤 카테고리(호텔/모텔/펜션)로 등록되어 있는지 확인합니다.
  </Step>
  <Step title="다중 카테고리 여부 파악">
    호텔과 모텔을 동시에 운영하는지, 아니면 하나의 카테고리만 사용하는지 확인합니다. 모텔 카테고리만 연동을 원하는 경우 이를 명시적으로 설정합니다.
  </Step>
  <Step title="수수료율 차이 안내">
    호텔/모텔 카테고리별 수수료율이 다르다는 점을 업장에 안내하고, VCMS에서의 정산 시 수기 구분이 필요할 수 있음을 설명합니다.
  </Step>
  <Step title="연동 범위 설정">
    업장의 의도에 맞게 연동할 카테고리를 명확히 설정합니다. 불필요한 카테고리까지 연동하면 관리 복잡도만 증가합니다.
  </Step>
</Steps>

## 펜션 카테고리 특수성

야놀자 펜션(야펜) 카테고리는 호텔/모텔과 **근본적으로 다른 구조**입니다:

- 객실 수 표시가 안 되며, 요금 설정 방식이 타입별/호실별로 일관되지 않음
- 객실 상품을 하나의 재고로 묶어 관리하는 것을 허용하지 않음
- 호실 단위로 객실을 나누어 관리해야 하므로, VCMS의 타입 단위 관리와 충돌할 수 있음
- 단일 객실 타입에 대한 다중 재고 관리 기능이 펜션 카테고리에서 제공되지 않음

<AccordionGroup>
  <Accordion title="펜션인데 VCMS에서 타입 단위로 나오는데요?">
    야놀자 펜션은 호실 단위 관리가 기본인데, VCMS는 타입 단위로 관리하는 경우가 있습니다. 업장의 야놀자 파트너센터에서 실제 객실 구성과 VCMS의 객실 타입이 일치하는지 확인이 필요합니다.
  </Accordion>
  <Accordion title="숙소가 펜션 카테고리로 잘못 등록되어 있어요">
    펜션 카테고리로 잘못 등록된 경우 재고 관리가 호실 단위가 아닌 타입 단위로 처리되어 운영에 문제가 생깁니다. 야놀자 측에 카테고리 변경을 요청해야 합니다.
  </Accordion>
</AccordionGroup>

## 프로모션 및 특수 상품 매핑

### 어떤 상황에서 많이 막히는지

- 야놀자 프로모션(핫딜, 오픈특가, 당일 할인) 생성 시 **VCMS에서 매핑을 수동으로 다시 수행**해야 함
- 프로모션으로 인해 의도한 것보다 훨씬 낮은 가격으로 객실이 판매되는 사례
- 야놀자에서 설정한 상품명('늦은 퇴실')이 VCMS에서 '늦은 입실'로 잘못 인식되는 현상
- 호텔스토리 연동 시 예약 메시지가 VCMS로 넘어오지 않아 수동 입력 필요
- 야놀자가 아고다와 채널링을 통해 자체 상품으로 판매하여, 아고다 예약이 야놀자로 표시되는 혼란

### 왜 이런 일이 생기는지

야놀자의 프로모션 시스템은 **새 상품을 생성하는 방식**으로 동작합니다. 기존 상품에 할인을 적용하는 것이 아니라, 별도의 프로모션 상품이 만들어지므로:

- 프로모션 생성마다 VCMS에서 새 상품을 인식하고 매핑해야 함
- 프로모션 상품의 가격이 VCMS 기준가와 다르게 적용될 수 있음
- 프로모션이 종료되어도 매핑 정리가 자동으로 이루어지지 않음

### 실무 대응

<Steps>
  <Step title="프로모션 사용 여부 사전 확인">
    연동 전에 업장이 야놀자에서 사용 중인 프로모션 목록을 확인합니다.
  </Step>
  <Step title="프로모션 상품 매핑">
    활성화된 프로모션 상품을 VCMS 객실 타입에 매핑합니다.
  </Step>
  <Step title="가격 검증">
    프로모션 적용 후 실제 판매 가격이 업장의 의도와 일치하는지 야놀자 앱에서 확인합니다.
  </Step>
  <Step title="프로모션 변경 시 재매핑 안내">
    업장에 프로모션을 새로 만들거나 수정하면 VCMS 매핑도 다시 해야 한다는 점을 안내합니다.
  </Step>
</Steps>

## 야놀자의 연동 차단 이력

### 확인된 차단 패턴

야놀자는 벤디트와 같은 외부 CMS의 연동을 **정책적으로 제한하거나 차단**한 이력이 있습니다. 확인된 패턴:

- 벤디트 서버 IP를 차단하여 연동을 중단시킨 사례
- 모텔에 대한 정식 계약 부재를 이유로 벤디트 시스템을 차단
- 요금/재고 조절은 가능하지만 **예약 연동만 선택적으로 차단**하는 방식
- 특정 요일(화·수·목)에만 차단이 발생하는 패턴
- 점주의 PC에서 직접 접속하는 것까지 차단하는 강력한 조치
- 오류 발생 시 해당 업장의 모든 방을 일괄 마감 처리

<Warning>
  야놀자 연동 차단이 발생하면 이전에 없던 오버부킹이 처음으로 발생하는 사례가 보고되었습니다. 연동 차단과 동시에 예약이 들어오면 재고 차감이 이루어지지 않기 때문입니다.
</Warning>

### 차단 감지 방법

- VCMS에서 야놀자 채널 상태가 비정상(빨간색 표시)으로 표시되는지 확인
- 야놀자 예약이 일정 시간 이상 들어오지 않는 경우 의심
- 야놀자 파트너센터에서 객실이 마감 처리되어 있는지 확인
- 특정 시간대에만 예약이 끊기는 패턴이 있는지 모니터링

## 상품 매핑 오류

### 빈출 사례

| 증상 | 원인 |
|------|------|
| 예약이 VCMS에 반영되지 않음 | 야놀자 상품이 잘못된 카테고리에 연결되어 있거나 비활성화 |
| 특정 객실(206호)이 판매 캘린더에 없음 | 다른 객실과 중복 연결되어 있거나 누락 |
| '늦은 퇴실' 상품이 '늦은 입실'로 표시 | 상품명 인식 오류 |
| 야놀자 예약이 들어왔는데 알림이 안 옴 | 문자 알림 목록에 누락 |
| 아고다 예약인데 야놀자로 표시 | 야놀자-아고다 채널링으로 인한 정상 동작 |

### 컴퓨터 상시 가동 요구사항

VCMS의 야놀자 연동 기능은 **컴퓨터 상시 가동을 요구**합니다. 개인 컴퓨터를 사용하거나 전용 컴퓨터를 두어야 하며, PC 재시작 후 연동 프로그램이 자동 실행되는지 확인해야 합니다.

<Warning>
  야놀자 연동을 위해 별도 프로그램 설치가 필요한 경우가 있습니다. PC를 끄면 연동이 중단되므로, 업장에 이 점을 반드시 안내해야 합니다.
</Warning>

## 실무 체크리스트

- [ ] 업장의 야놀자 카테고리(호텔/모텔/펜션)를 정확히 파악했는가
- [ ] 호텔/모텔 동시 운영 시 VCMS에서의 구분 관리 방법을 안내했는가
- [ ] 활성 프로모션 상품이 VCMS에 모두 매핑되어 있는가
- [ ] 프로모션 변경 시 재매핑 필요성을 업장에 안내했는가
- [ ] 야놀자 연동 차단 가능성과 감지 방법을 업장에 안내했는가
- [ ] 야놀자의 아고다 채널링으로 인한 예약 출처 혼란을 설명했는가
- [ ] 연동 프로그램이 PC 상시 가동을 요구함을 안내했는가
- [ ] 기존에 사용하던 다른 CMS(사나윙스 등)의 설정이 야놀자에서 비활성화되었는지 확인했는가
