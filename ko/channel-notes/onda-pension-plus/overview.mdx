# 온다 펜션플러스 채널 운영 노트

온다 펜션플러스는 일반적인 채널 매니저 서비스와는 다른 **판매 대행** 구조를 가진 채널이다. 직접 판매 상품과 대행 판매 상품의 구분, 독특한 재고 관리 방식, 가격 반영의 비대칭성 등으로 인해 VCMS 연동 시 별도의 이해가 필요하다.

---

## 채널 특성

| 항목 | 내용 |
|------|------|
| 채널 유형 | **판매 대행** — 일반적인 OTA와 구조가 다름 |
| 판매 구분 | 직접 판매 상품 + 대행 판매 상품이 혼재 |
| 재고 관리 | 채널 자체의 묶음 방식 — VCMS 기준과 다를 수 있음 |
| 가격 반영 | 실시간 예약창에만 적용되고, 온라인 판매 대행 채널에는 별도 반영 필요 |
| 당일특가 | 온다 펜션플러스 고유 기능 — 다른 채널에는 없음 |
| 연동 상태 | VCMS 연동에 어려움이 있는 채널로 분류됨 |

<Warning>
온다 펜션플러스는 일반적인 채널 매니저 서비스와 다른 독특한 판매 대행 및 재고 관리 방식을 가지고 있어 VCMS 연동에 구조적 어려움이 있다. 직접 판매 상품과 대행 판매 상품의 구분, 가격 설정 방식에 특히 주의가 필요하다.
</Warning>

## 많이 막히는 지점

### 1. 판매 대행 구조의 특이성

온다 펜션플러스는 단순한 OTA가 아니라 판매 대행 서비스를 겸한다. 이 구조적 차이가 여러 혼란의 근본 원인이다.

<AccordionGroup>
<Accordion title="직접 판매 vs 대행 판매">
- **직접 판매**: 온다 펜션플러스 자체 사이트에서 직접 판매되는 상품
- **대행 판매**: 온다가 다른 OTA(부킹닷컴, 호텔스닷컴 등)에 대행하여 판매하는 상품
- 이 두 가지가 하나의 시스템에서 혼재되어 관리됨
- VCMS에서 설정한 가격이 직접 판매에는 적용되지만, 대행 판매 채널에는 별도로 반영되어야 할 수 있음
</Accordion>

<Accordion title="부킹닷컴 리뷰 연관성">
온다를 통해 부킹닷컴에 판매 대행을 할 경우, 기존에 쌓아둔 부킹닷컴 리뷰가 사라질 수 있다는 우려가 보고됨. 이는 업장의 가장 큰 우려 사항 중 하나로, 대행 전환 시 리뷰 보존 여부를 사전에 확인해야 한다.
</Accordion>
</AccordionGroup>

### 2. 재고 묶음 방식의 차이

| 구분 | VCMS 기준 | 온다 펜션플러스 |
|------|-----------|----------------|
| 재고 단위 | 객실 타입별 수량 관리 | 자체 묶음 방식 — 직접 판매와 대행 판매가 다를 수 있음 |
| 재고 조절 | 세부 수량 조절 가능 | 특정 제약이 있을 수 있음 |
| 동기화 | VCMS에서 일괄 관리 | 채널 자체 로직으로 재고 처리 |

### 3. 온다에서만 이전 가격으로 예약이 들어오는 문제

이 문제는 반복적으로 보고되며, 온다 펜션플러스 채널의 핵심 페인 포인트 중 하나다.

<Warning>
가격 인상 후에도 온다 채널에서만 이전(낮은) 가격으로 예약이 들어와 판매 전략에 혼란과 손실이 발생하는 사례가 확인되었다. 가격 변경 시 반드시 온다 펜션플러스에서의 반영 여부를 별도로 확인해야 한다.
</Warning>

<AccordionGroup>
<Accordion title="문제 발생 구조">
1. 업장이 VCMS에서 가격을 인상
2. 다른 채널(야놀자, 여기어때 등)에는 정상 반영
3. 온다 펜션플러스에서는 이전 가격이 그대로 유지
4. 고객이 온다를 통해 이전(낮은) 가격으로 예약
5. 업장은 의도하지 않은 저가 판매로 손실 발생

**원인 추정**: 온다 펜션플러스에서 가격 변경 시 실시간 예약창에만 적용되고, 온라인 판매 대행 채널에는 별도 처리가 필요한 구조적 문제
</Accordion>

<Accordion title="고객 오해 사례">
온다 펜션플러스에서 가격 변경 시 실시간 예약창에 반영되면 모든 채널에 적용된다고 오해하는 경우가 많다. 실제로는 온라인 판매 대행 채널(OTA)에는 별도로 반영되지 않을 수 있다.
</Accordion>
</AccordionGroup>

### 4. 계정 정보 및 서버 불안정

| 이슈 | 상세 |
|------|------|
| 가격 0원 표시 | 특정 채널에서 가격이 0원으로 표시되어 판매될 위험 |
| 서버 불안정 | 접속 지연, 간헐적 오류 등 서버 안정성 문제 |
| 계정 접근 | 대표님이 직접 관리하는 계정 정보를 직원이 알지 못하는 경우 |

## 가격·재고·정산 관점의 주의점

### 가격 설정 시 필수 확인사항

<Steps>
  <Step title="온다 펜션플러스에서 가격 변경 범위 확인">
    가격 변경이 실시간 예약창에만 적용되는지, 온라인 판매 대행 채널에도 적용되는지 사전에 확인한다.
  </Step>
  <Step title="VCMS에서 변경 후 온다 사이트에서 실제 가격 확인">
    VCMS에서 가격을 변경한 후, 온다 펜션플러스 사이트에서 실제 노출되는 가격이 변경되었는지 직접 확인한다.
  </Step>
  <Step title="대행 판매 채널에서의 가격 확인">
    온다를 통해 대행 판매되는 채널(부킹닷컴 등)에서도 가격이 업데이트되었는지 별도 확인한다.
  </Step>
  <Step title="0원 표시 여부 확인">
    가격 설정 오류로 인해 0원으로 표시되어 판매될 위험이 없는지 확인한다.
  </Step>
</Steps>

### 재고 관련

- 온다 펜션플러스의 재고 묶음 방식이 VCMS와 다를 수 있으므로, 연동 시 재고 수량이 올바르게 동기화되는지 확인
- 직접 판매 상품과 대행 판매 상품의 재고가 별도로 관리될 수 있음
- 가격 변경 시 재고도 함께 영향을 받을 수 있으므로 주의

### 정산 관련

- 온다의 판매 대행 수수료 구조를 사전에 파악하고, 수익 계산에 반영
- 직접 판매와 대행 판매의 정산 방식이 다를 수 있음
- 이전 가격으로 판매된 건에 대한 정산 차이에 주의

## 당일특가 특성

온다 펜션플러스는 **당일특가** 기능을 고유하게 제공한다. 이 기능은 다른 판매 채널에는 없는 온다만의 특성이다.

<AccordionGroup>
<Accordion title="당일특가란?">
- 당일 판매되지 않은 객실을 특별 할인가로 판매하는 기능
- 온다 펜션플러스 플랫폼에서만 사용 가능
- 다른 채널(야놀자, 여기어때 등)에는 해당 기능이 없어 통합적인 가격 전략 수립에 제약
</Accordion>

<Accordion title="당일특가 가격 관리">
- 다른 플랫폼 대비 당일특가 가격이 낮을 수 있으므로, 가격을 상향 조정하여 수익 최적화 기회가 있을 수 있음
- 당일특가 가격이 다른 채널의 정상가보다 지나치게 낮으면 채널 간 가격 불일치 문제 발생
- 당일특가를 다른 채널에도 적용하고 싶은 수요가 있으나, 타 플랫폼에서는 지원하지 않음
</Accordion>
</AccordionGroup>

## 먼저 확인할 순서

<Steps>
  <Step title="판매 대행 구조 이해">
    온다 펜션플러스가 직접 판매와 대행 판매를 겸하는 채널임을 인지하고, 각각의 가격/재고 반영 방식을 파악한다.
  </Step>
  <Step title="가격 변경 반영 범위 테스트">
    소폭의 가격 변경을 시행한 후, 온다 직접 사이트와 대행 판매 채널 모두에서 반영 여부를 확인한다.
  </Step>
  <Step title="이전 가격 잔류 확인">
    최근 가격 변경 이력이 있다면, 온다 채널에서 이전 가격이 아직 노출되고 있지 않은지 확인한다.
  </Step>
  <Step title="서버 상태 및 계정 접근성 확인">
    온다 펜션플러스 서버 접속이 정상인지, 관리 계정에 접근 가능한지 확인한다.
  </Step>
  <Step title="당일특가 설정 확인">
    당일특가가 활성화되어 있는 경우, 가격이 다른 채널과 적절한 수준인지 확인한다.
  </Step>
  <Step title="재고 동기화 상태 확인">
    VCMS와 온다 펜션플러스 간의 재고 수량이 일치하는지 확인한다.
  </Step>
</Steps>

## 자주 오해하는 포인트

<AccordionGroup>
<Accordion title="온다 펜션플러스는 일반 OTA와 같은 방식으로 연동된다?">
**다르다.** 온다 펜션플러스는 판매 대행 구조를 가지고 있어 일반적인 OTA 채널 매니저 연동과 다른 방식으로 작동한다. 직접 판매와 대행 판매가 혼재되어 있으며, VCMS 연동에 구조적 어려움이 보고되고 있다.
</Accordion>

<Accordion title="가격을 변경하면 온다의 모든 판매 채널에 반영된다?">
**반드시 그렇지 않다.** 실시간 예약창에는 반영되더라도 온라인 판매 대행 채널(OTA)에는 별도 반영이 필요할 수 있다. 가격 변경 후 반드시 대행 채널에서도 확인해야 한다.
</Accordion>

<Accordion title="온다에서 당일특가를 설정하면 다른 채널에도 적용된다?">
**적용되지 않는다.** 당일특가는 온다 펜션플러스 고유 기능이며, 다른 판매 채널에는 해당 기능 자체가 없다.
</Accordion>

<Accordion title="온다를 통한 부킹닷컴 판매가 기존 직접 판매와 동일하다?">
**차이가 있을 수 있다.** 온다를 통한 판매 대행 시 기존에 쌓아둔 부킹닷컴 리뷰가 사라질 수 있으며, 수수료 구조도 다를 수 있다. 전환 전 반드시 확인이 필요하다.
</Accordion>
</AccordionGroup>

## 운영 팁

- **가격 변경 후 이중 확인 루틴**: 가격 변경 시 반드시 (1) 온다 직접 사이트, (2) 대행 판매 채널에서 각각 확인하는 이중 확인 루틴을 만든다. 온다에서만 이전 가격이 남아 있는 문제를 사전에 방지.
- **당일특가 가격 최적화**: 당일특가 가격이 다른 채널 대비 지나치게 낮지 않은지 정기적으로 확인하고, 필요시 상향 조정하여 수익을 최적화한다.
- **대행 판매 채널 리뷰 보존 확인**: 온다를 통한 대행 판매 전환 시, 기존 OTA(부킹닷컴 등)의 리뷰가 보존되는지 사전에 온다 담당자에게 확인한다.
- **0원 판매 방지**: 가격 설정 오류로 인한 0원 판매 위험이 보고된 바 있으므로, 가격 변경 후 즉시 노출 가격을 확인한다.
- **연동 가능성 확인**: VCMS와의 완전한 연동이 어려울 수 있는 채널이므로, 벤디트 개발팀에 연동 가능성과 해결 방안을 사전에 확인한다.
- **서버 불안정 대비**: 온다 서버가 불안정할 수 있으므로, 중요한 가격 변경이나 재고 조정은 서버 상태가 안정적인 시간대에 수행한다.
- **네이버 직계약 비교 검토**: 네이버의 자체 수수료(4%)가 온다 등 판매 대행사를 통한 판매보다 저렴할 수 있다. 채널별 수수료를 비교하여 최적의 채널 믹스를 구성한다.
