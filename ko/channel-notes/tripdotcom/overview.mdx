# 트립닷컴 채널 운영 노트

트립닷컴은 VCMS에서 공식 연동이 지원되는 해외 OTA 채널이다. 아고다와 유사한 입금가 기반 구조를 가지고 있으나, 시스템 안정성, 로그인 문제, '예약 불가' 상태 등 별도의 운영 이슈가 빈번하게 보고된다.

---

## 채널 특성

| 항목 | 내용 |
|------|------|
| 연동 방식 | 공식 API 연동 (재고 + 요금 + 예약) |
| 요금 기준 | **입금가(net rate)** 기준 — 아고다와 동일 |
| 수수료 구조 | 약 13%, 실질적으로는 프로모션 등으로 더 높을 수 있음 |
| 연동 활성화 | 시스템 설정 외에 **트립닷컴에 별도 전화 통보**가 필요 |
| 동기화 주기 | 자동 동기화 약 1시간에 1회. 강제 가져오기 기능 있음 |
| 과거 예약 | 연동 이전 예약 정보 미제공 — 기존 예약은 수동 관리 필요 |
| 취소 내역 | 기존 예약의 취소 내역을 반영하지 못함 — 직접 취소 관리 |
| 시간 정보 | 입퇴실 시간을 명시적으로 제공하지 않음 |
| 개발팀 위치 | 해외(중국)에 위치하여 직접 전화 소통 어려움, 이메일 소통 |

## 많이 막히는 지점

### 1. 로그인 문제

트립닷컴 채널 매니저 계정 로그인에 지속적인 문제가 보고된다.

<AccordionGroup>
<Accordion title="주요 로그인 이슈 유형">
- **비밀번호 불일치**: 트립닷컴 엑스트라넷에 저장된 비밀번호가 고객이 기록한 것과 다른 경우
- **계정 미연동**: "계정이 연동되어 있지 않거나 잘못된 계정"이라는 메시지 반복
- **인증 루프**: 프론트 PC에서 직접 로그인 시도 시 지속적인 인증 요구, 인증 성공 시 예약 정보가 사라지는 현상
- **대표님 정보 부재**: 업장 직원이 대표님의 이메일·비밀번호를 알지 못해 진행 불가
</Accordion>
</AccordionGroup>

<Warning>
트립닷컴에 직접 로그인하면 채널 매니저 연동에 영향을 줄 수 있다. 프론트 PC에서의 직접 로그인은 피하고, 예약 확인은 VCMS를 통해 수행하는 것을 권장한다.
</Warning>

### 2. '예약 불가' 상태의 지속

트립닷컴에서 가장 빈번하게 보고되는 문제다.

| 증상 | 가능한 원인 |
|------|------------|
| 모든 객실 '예약 불가' 표시 | 연동 설정 미완료 또는 트립닷컴 내부 문제 |
| '더 이상 예약을 할 수 없는 숙소입니다' | 트립닷컴 내부 설정 문제 또는 판매 정지 |
| 재고가 있는데 '매진' 표시 | 캘린더 블로킹 또는 요금제 미설정 |
| 특정 객실만 노출 안됨 | 해당 룸 타입에 대한 요금제 미설정 |

<Steps>
  <Step title="VCMS에서 연동 상태 확인">
    VCMS의 채널 연동 화면에서 트립닷컴 연동이 활성 상태인지 확인한다.
  </Step>
  <Step title="요금제 설정 확인">
    트립닷컴은 '타입'은 하나이지만 '요금제'가 여러 개인 구조다. 각 요금제에 요금이 설정되어 있는지 확인한다. 요금제가 미설정된 룸 타입은 노출되지 않는다.
  </Step>
  <Step title="캘린더 블로킹 여부 확인">
    과거에 주말 할인 적용 등을 위해 캘린더를 막았던 이력이 있는지 확인한다.
  </Step>
  <Step title="트립닷컴에 직접 문의">
    위 확인으로도 해결되지 않으면 트립닷컴 파트너 센터에 직접 전화하여 상태를 문의한다.
  </Step>
</Steps>

### 3. 오래된 계정과 최근 계정의 연동 방식 차이

<AccordionGroup>
<Accordion title="계정 시기별 차이">
- **오래된 계정**: 요금제별로 재고가 분리되어 있을 수 있음. 각 요금제에 대해 개별적으로 재고 설정이 필요
- **최근 개설 계정**: 재고가 합쳐져 있는 경우가 많아 연동 방식이 상대적으로 단순
- 이 차이로 인해 동일한 설정 방법을 적용해도 업장마다 결과가 달라질 수 있음
</Accordion>
</AccordionGroup>

### 4. 연동 활성화까지의 지연

- 트립닷컴 연동은 시스템 설정 외에 **전화 통보**가 필요하며, 활성화까지 1~2일 소요
- 이 기간 동안 일시적으로 재고가 막힐 수 있음
- CM 연동 처리 속도가 느리고, 완료 후에도 통보가 없어 판매 중단 문제 발생 가능

## 요금·재고 관점의 주의점

### 요금 기대치와 실제 노출 요금의 차이

<Warning>
트립닷컴에 설정한 요금(예: 11만 5천원, 15만원)과 실제 판매된 요금(예: 8만원대) 사이에 큰 차이가 발생하여 손해를 입었다는 보고가 있다. 이는 아고다와 마찬가지로 입금가 기반 연동, 트립닷컴 자체 프로모션, 환전 수수료 등이 복합적으로 작용한 결과다.
</Warning>

| 항목 | 상세 |
|------|------|
| 요금 설정 기준 | 입금가 (판매가 아님) |
| 판매가 통제 | 불가 — 트립닷컴이 최종 판매가 결정 |
| 프로모션 영향 | 7연박 할인(10~15%) 등이 자동 적용될 수 있음 |
| 수수료 표면 vs 실질 | 공식 13%, 실질적으로는 프로모션·세금으로 19%까지 차이 |

### 아고다와 연결된 요금 특성

트립닷컴은 아고다와 요금 구조가 연결되어 있는 경우가 있다.

<AccordionGroup>
<Accordion title="요금 링크 구조">
- 아고다와 트립닷컴은 일부 채널에서 요금이 서로 링크되어 있음
- 한 상품의 요금을 변경하면 연관된 다른 상품의 요금도 자동 변경되는 특성
- 아고다에서 요금을 수정하면 트립닷컴에도 영향을 줄 수 있고, 그 반대도 가능
- 이 링크 관계를 모르는 상태에서 한쪽만 요금을 변경하면 의도치 않은 요금 불일치 발생
</Accordion>

<Accordion title="인원별 요금 차이">
- 트립닷컴 연동 시 인원별 요금제가 아고다와 다른 방식으로 연동될 수 있음
- 아고다는 간이 침대를 통한 추가 인원 설정을 명시적으로 지원하여 최대 인원이 표시됨
- 트립닷컴은 간이 침대를 추가 서비스 형태로 처리하여 최대 인원 계산 방식이 다름
- 현재 2인까지만 판매가 제한되는 경우도 있음
</Accordion>
</AccordionGroup>

### 재고 관련 주의사항

| 주의사항 | 상세 |
|----------|------|
| 동기화 주기 | 자동 동기화 약 1시간에 1회. 즉각적이지 않음 |
| 미설정 = 판매 가능 | 미설정 상태가 곧 판매 설정으로 연결되는 로직 — VCMS에서 막아두지 않으면 열림 |
| 과거 예약 불러오기 불가 | 연동 이전 예약은 수동 관리 필수 |
| 캘린더 데이터 삭제 사례 | 연동 직후 호텔 판매 캘린더 데이터가 전부 삭제되는 문제 보고됨 |
| 재고 차단 후 잔여 | 마감 설정 후에도 트립닷컴 동기화 지연으로 예약이 들어올 수 있음 |

### 운영 확인 포인트

- 입금가만 연동되고 판매가는 정확하지 않음
- 판매가와 입금가 사이에 19%까지 차이가 발생한 사례
- 1박당 16만원 설정인데 8만 7천원으로 표기되는 수량/금액 표기 오류 보고
- 직원 인센티브를 벤디트 시스템 판매가 기준으로 지급하면 오차가 발생할 수 있음

## 먼저 확인할 순서

<Steps>
  <Step title="계정 정보 확인">
    트립닷컴 엑스트라넷의 정확한 ID와 비밀번호를 확보한다. 대표님이 직접 관리하는 경우가 많아 직원이 정보를 모르는 경우가 빈번.
  </Step>
  <Step title="연동 활성화 확인">
    VCMS에서 트립닷컴 연동 상태가 활성인지 확인한다. 트립닷컴에 별도 전화 통보가 이루어졌는지 확인.
  </Step>
  <Step title="자동 예약 확인 설정">
    트립닷컴의 자동 예약 확인(Auto-confirmation) 설정이 활성화되어 있는지 확인한다. 비활성화 시 예약 정보가 자동으로 연동되지 않음.
  </Step>
  <Step title="요금제 설정 확인">
    각 룸 타입에 대한 요금제가 모두 설정되어 있는지 확인한다. 미설정 룸 타입은 트립닷컴에서 노출되지 않음.
  </Step>
  <Step title="재고 180일 확인">
    연동 후 180일치 재고가 정상적으로 열려 있는지 확인한다.
  </Step>
  <Step title="실제 트립닷컴 사이트에서 노출 확인">
    트립닷컴 웹사이트에서 실제로 숙소가 검색되고, 예약 가능 상태로 표시되는지 확인한다.
  </Step>
</Steps>

## 자주 오해하는 포인트

<AccordionGroup>
<Accordion title="트립닷컴에 요금을 설정했으니 바로 판매가 시작된다?">
**아니다.** 시스템 설정 외에 트립닷컴에 별도 전화 통보가 필요하며, 활성화까지 1~2일 소요될 수 있다. 이 기간 동안 재고가 일시적으로 막힐 수 있다.
</Accordion>

<Accordion title="아고다와 같은 방식으로 설정하면 동일하게 작동한다?">
**다를 수 있다.** 계정 시기에 따라 재고 관리 방식(분리/통합)이 다르고, 인원별 요금 처리 방식도 아고다와 차이가 있다. 또한 요금이 서로 링크되어 있는 경우 한쪽 변경이 다른 쪽에 영향을 줄 수 있다.
</Accordion>

<Accordion title="연동하면 기존 예약도 자동으로 불러온다?">
**아니다.** 트립닷컴은 연동 이후 생성·변경된 예약 건만 연동 가능하다. 기존 예약은 수동으로 관리해야 한다.
</Accordion>

<Accordion title="트립닷컴에서 직접 요금을 변경해도 되나?">
**주의 필요.** VCMS와 연동된 상태에서 트립닷컴에서 직접 요금을 변경하면, VCMS 동기화 시 다시 원래 요금으로 되돌아갈 수 있다.
</Accordion>

<Accordion title="수수료가 13%이니 13%만 인상하면 된다?">
**실제로는 부족하다.** 트립닷컴도 프로모션, 환전 수수료, 세금 등이 추가되어 실질 차이가 19%까지 벌어진 사례가 보고되었다. 아고다와 유사하게 여유를 두고 인상 설정을 권장한다.
</Accordion>
</AccordionGroup>

## 운영 팁

- **연동 전 기존 예약 정리**: 연동 전에 트립닷컴에서 수기로 관리하던 예약을 VCMS에 미리 입력해 둔다. 연동 후에는 수기 예약과 연동 예약이 중복될 수 있으므로 중복 건을 정리해야 한다.
- **강제 가져오기 활용**: 자동 동기화(1시간 간격) 외에 VCMS에서 트립닷컴 예약을 강제로 가져오는 기능이 있다. 중요한 시점에는 수동으로 동기화를 실행한다.
- **퇴실 시간 주의**: 아고다와 마찬가지로 트립닷컴 예약의 퇴실 시간이 실제 퇴실일이 아닌 다음 날 14시로 설정되는 경우가 있다. 객실 재고 불일치의 원인이 될 수 있으므로 확인 필요.
- **교차 판매 모니터링**: 야놀자, 트립닷컴 등 다른 OTA가 고객 동의 없이 트립닷컴에 숙소를 교차 판매하는 경우가 보고되었다. 이중 수수료, 요금 통제 불가 문제가 발생할 수 있으므로 정기적으로 트립닷컴에서 자사 숙소 검색을 수행한다.
- **단일 채널 사용 시 비용 대비 효과 낮음**: 벤디트 직원도 트립닷컴 단일 채널에만 VCMS를 사용하는 것은 비용 대비 이점이 크지 않다고 안내한 사례가 있다. 아고다, 야놀자 등 다른 채널과 함께 사용할 때 효과적.
- **트립닷컴 개발팀 소통**: 문제 발생 시 트립닷컴 개발팀이 해외(중국)에 있어 이메일 소통이 주가 되며, 해결에 시간이 소요될 수 있다. 긴급한 경우 벤디트를 통해 에스컬레이션을 요청한다.
