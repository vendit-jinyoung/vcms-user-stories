# 네이버 — 로그인 및 캘린더

네이버 연동에서 가장 빈번하게 장애가 발생하는 영역은 **인증(2FA/앱비밀번호)**과 **캘린더 설정 초기화**입니다. 인증이 풀리면 연동이 조용히 끊기고, 동기화를 진행하면 기존에 수동으로 설정한 공휴일/딜 정보가 사라집니다.

---

## 2차 인증(2FA) 및 앱비밀번호

### 어떤 상황에서 많이 막히는지

- 네이버 2차 인증이 **해제되거나 변경**되면 VCMS 연동이 자동으로 끊김
- 2차 인증 해제 시 발급받았던 **애플리케이션 전용 암호가 무효화**되어 재발급 필요
- 네이버 계정에 **보호조치**가 걸려 시스템 접근 자체가 불가
- 비밀번호 변경 후 연동이 끊어지는 사례
- 과도한 로그인 시도 알림과 로그인 잠금
- 단체 계정(법인 계정)에서는 2차 인증 설정 자체가 어려움
- 2차 인증 번호가 대표(또는 가족)의 핸드폰으로만 수신되어, 해당 인물이 부재 시 연동 진행 불가

### 왜 이런 일이 생기는지

네이버의 보안 정책은 **자동화된 외부 로그인에 민감**합니다. VCMS가 네이버에 접속하는 방식이 네이버 보안 시스템에 의해 의심스러운 접근으로 감지될 수 있으며, 이로 인해:

- 보호조치가 자동 발동되어 계정이 잠김
- 로그인 시도 실패가 누적되면 계정 자체에 제한이 걸림
- 2차 인증 없이는 벤디트 시스템의 간편 로그인을 통한 접속이 불가능

벤디트 시스템은 **네이버 간편 로그인만 지원**하므로, 단체 계정처럼 간편 로그인이 불가능한 경우 연동 자체에 제약이 있습니다.

### 운영자가 직접 확인할 순서

<Steps>
  <Step title="네이버 2차 인증 상태 확인">
    네이버 계정 설정에서 2차 인증이 '사용' 상태인지 확인합니다. '사용 안 함'이면 연동이 끊길 수 있습니다.
  </Step>
  <Step title="앱비밀번호 발급 여부 확인">
    2차 인증이 활성 상태라면, VCMS용 애플리케이션 전용 암호가 정상 발급되어 있는지 확인합니다. 암호가 소실되었으면 재발급이 필요합니다.
  </Step>
  <Step title="보호조치 여부 확인">
    네이버 로그인을 시도하여 보호조치 안내가 표시되는지 확인합니다. 보호조치가 걸려 있으면 네이버 고객센터를 통해 해제해야 합니다.
  </Step>
  <Step title="2차 인증 수신 번호 확인">
    2차 인증 문자/앱 알림을 받는 핸드폰 번호가 현재 접근 가능한 번호인지 확인합니다. 대표의 핸드폰이 필요한 경우, 연동 일정을 대표의 일정에 맞춰 조율합니다.
  </Step>
  <Step title="IP 차단 설정 확인">
    네이버 계정에 해외 IP 또는 특정 지역 IP 차단 설정이 되어 있으면, 벤디트 서버에서 접속이 불가능합니다. 해당 설정을 해제합니다.
  </Step>
</Steps>

### 자주 생기는 오해

<AccordionGroup>
  <Accordion title="2차 인증을 끄면 더 편하지 않나요?">
    2차 인증을 끄면 앱비밀번호를 발급받을 수 없어 VCMS 연동이 **오히려 불가능**해질 수 있습니다. 2차 인증은 켜둔 상태에서 앱비밀번호를 사용하는 것이 권장됩니다.
  </Accordion>
  <Accordion title="네이버 비밀번호를 바꿨더니 연동이 끊겼어요">
    네이버 비밀번호 변경 시 기존에 발급받은 앱비밀번호도 영향을 받을 수 있습니다. 비밀번호 변경 후 VCMS 연동 상태를 반드시 확인하고, 필요 시 앱비밀번호를 재발급합니다.
  </Accordion>
  <Accordion title="단체 계정으로는 연동이 안 되나요?">
    벤디트 시스템은 네이버 간편 로그인만 지원합니다. 단체 계정(법인명의)은 간편 로그인이 불가능한 경우가 있어, 개인 네이버 계정으로 변경하거나 새 계정을 생성해야 할 수 있습니다.
  </Accordion>
  <Accordion title="보호조치가 걸렸는데 어떻게 해야 하나요?">
    네이버 고객센터에 직접 연락하여 보호조치 해제를 요청해야 합니다. 본인 인증이 필요하므로, 계정 명의자가 직접 처리해야 합니다.
  </Accordion>
</AccordionGroup>

## 스마트플레이스 계정 구조 문제

### 빈출 사례

- 네이버 스마트플레이스의 **주 관리자 계정이 통화 중인 담당자의 계정이 아닌** 경우 (매니저 계정으로 되어 있음)
- 사업자 ID가 **대표(어머니 등)의 명의**인데, 실제 운영은 다른 사람이 하는 경우
- 스마트플레이스 인증이 **다른 법인/개인 명의로 잘못 처리**되어 취득세 중과 문제 발생 가능성
- 네이버 스마트플레이스에 **중복된 채널이 등록**되어 있어 어떤 것을 삭제해야 할지 모르는 상황
- 야놀자가 프로모션으로 등록해준 네이버 플레이스를 업장이 직접 관리하려 할 때, 야놀자 측에서 계정 이관/삭제에 응답하지 않는 사례

<Warning>
  스마트플레이스 계정의 소유권과 관리 권한 구조를 연동 전에 반드시 파악해야 합니다. 계정 소유자, 관리자, 2차 인증 수신자가 모두 다를 수 있습니다.
</Warning>

## 공휴일 설정 초기화

### 어떤 상황에서 많이 막히는지

- VCMS와 재고/요금 동기화를 진행하면 **네이버에 설정된 공휴일 정보가 모두 삭제**됨
- 업장이 수개월에 걸쳐 수동으로 입력한 공휴일 요금이 한 번에 사라짐
- 공휴일 요금이 없어지면 해당 날짜에 평일/주말 요금이 적용되어 업장에 손실 발생
- 네이버에서 자동으로 관리되던 '공휴일이면 하루씩 열리는' 방식도 VCMS 연동 후 작동하지 않음
- VCMS에서만 공휴일 설정이 가능하며, 업장이 직접 네이버에서 설정할 수 없게 됨

### 왜 이런 일이 생기는지

VCMS의 동기화 방식은 **VCMS의 데이터로 네이버의 데이터를 덮어쓰는** 구조입니다. 네이버에만 존재하는 데이터(공휴일, 오늘의 딜 등)는 VCMS에 대응하는 필드가 없으므로, 동기화 과정에서 삭제됩니다.

이는 설계상의 제약이며, 현재로서는 VCMS에서 공휴일을 별도로 재설정하는 것이 유일한 대응 방법입니다.

### 운영자가 직접 확인할 순서

<Steps>
  <Step title="동기화 전 공휴일 백업">
    VCMS 동기화를 진행하기 전에, 네이버에 설정된 공휴일 정보를 **캡처하여 보관**합니다.
  </Step>
  <Step title="동기화 후 VCMS에서 재설정">
    동기화 완료 후 VCMS의 판매 관리 화면에서 공휴일 요금을 다시 입력합니다.
  </Step>
  <Step title="공휴일 반영 확인">
    네이버 스마트플레이스에서 공휴일 요금이 올바르게 표시되는지 확인합니다.
  </Step>
</Steps>

<Warning>
  재고 조절 기능만 단독 사용하는 경우에도, 네이버의 공휴일 및 오늘의 딜 기능은 초기화됩니다. 이 제약을 피하려면 요금 조절 기능과 함께 사용하는 것이 권장됩니다.
</Warning>

## '오늘의 딜' 비활성화

### 증상

VCMS와 재고/요금을 동기화하면, 네이버의 '오늘의 딜' 기능이 비활성화됩니다.

- VCMS 연동 후에는 **네이버에서 직접 '오늘의 딜'을 관리할 수 없음**
- '오늘의 딜' 요금 변경은 VCMS 판매 관리 화면에서만 가능
- 오늘의 딜을 3개월 정도만 사용하는 등 활용도가 낮은 업장도 있지만, 적극적으로 사용하는 업장에는 큰 단점

### 대응

- 연동 전에 업장의 '오늘의 딜' 사용 빈도와 의존도를 파악
- 자주 사용하는 업장에는 VCMS 내에서의 대체 운영 방법을 안내
- 오늘의 딜에 크게 의존하는 업장은 동기화 범위를 신중하게 결정

## 네이버 연동 끊김의 일반적 원인 정리

| 원인 | 증상 | 대응 |
|------|------|------|
| 2차 인증 해제 | 연동 조용히 끊김, 알림 없음 | 2차 인증 재설정 + 앱비밀번호 재발급 |
| 보호조치 발동 | 로그인 불가, 연동 불가 | 네이버 고객센터 통해 해제 |
| 비밀번호 변경 | 기존 연동 끊김 | 새 비밀번호로 재연동 + 앱비밀번호 재발급 |
| IP 차단 설정 | 벤디트 서버 접속 차단 | 네이버 보안 설정에서 IP 차단 해제 |
| 과도한 로그인 시도 | 계정 잠금 | 일정 시간 대기 후 재시도, 필요 시 네이버 고객센터 문의 |
| 단체 계정 사용 | 간편 로그인 불가 | 개인 네이버 계정으로 변경 |

## 실무 체크리스트

- [ ] 네이버 2차 인증이 활성 상태이고, 앱비밀번호가 정상 발급되어 있는가
- [ ] 2차 인증 문자를 받는 핸드폰에 현재 접근 가능한가
- [ ] 네이버 계정에 보호조치가 걸려있지 않은가
- [ ] IP 차단 설정이 벤디트 서버를 차단하지 않는가
- [ ] 동기화 전에 공휴일 설정을 캡처/백업했는가
- [ ] 동기화 후 VCMS에서 공휴일 요금을 재설정했는가
- [ ] '오늘의 딜' 사용 여부를 사전에 확인했는가
- [ ] 스마트플레이스 주 관리자 계정과 실제 운영자가 누구인지 파악했는가
- [ ] 야놀자 경유로 등록된 네이버 플레이스가 있는지 확인했는가
