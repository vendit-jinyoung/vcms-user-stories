# 아고다 채널 운영 노트

아고다는 VCMS에서 공식 연동이 지원되는 해외 OTA 채널이다. 재고 및 요금 동기화가 가능하지만, 국내 채널과 근본적으로 다른 요금 체계와 프로모션 구조를 가지고 있어 별도의 이해가 필요하다.

---

## 채널 특성

| 항목 | 내용 |
|------|------|
| 연동 방식 | 공식 API 연동 (재고 + 요금 + 예약) |
| 요금 기준 | **입금가(net rate)** 기준 — 판매가가 아님 |
| 수수료 구조 | 표면 수수료 외에 프로모션·세금·환전 등이 추가되어 실질 수수료가 높음 |
| 인원별 요금 | 지원됨 — 기준 인원 + 추가 인원 요금 구조 |
| 아동 요금 | VCMS에서 설정 불가, YCS에서 별도 설정 필요 |
| 시간 정보 | 명시적 입퇴실 시간을 제공하지 않음 — VCMS 문자 발송 시 별도 설정 필요 |
| 채널 매니저 제한 | 하나의 채널 매니저만 연결 가능 — 기존 CMS 연동 해제 필수 |

---

## 많이 막히는 지점

### 1. 입금가 vs 판매가 혼동

아고다 연동의 가장 빈번한 혼란 포인트다.

<Warning>
아고다 공식 연동 시 VCMS에 전달되는 금액은 **입금가(net rate)**이다. 업장이 VCMS에 입력한 '판매가'가 아니라, 아고다가 수수료를 제외하고 업장에 실제로 입금하는 금액만 연동된다. 고객에게 노출되는 최종 판매가는 아고다가 자체적으로 결정하며, VCMS에서 정확히 파악할 수 없다.
</Warning>

- VCMS에서 "10만원"을 설정해도, 아고다 사이트에서 고객에게 보이는 가격은 다를 수 있음
- 입금가 기준이므로 매출 부가세 신고 시 판매가 역산이 필요
- 벤디트 시스템에는 입금가만 정확하게 들어오며, 판매가는 보장되지 않음

### 2. 실질 수수료가 예상보다 높음

<AccordionGroup>
<Accordion title="수수료 구조 상세">
- 공식 수수료율: 약 15% (초기 계약 시 18%인 경우도 있음)
- 실질 부담: 프로모션 할인, 환전 수수료, 세금 등이 추가되어 **16.5% 이상**에 달하는 경우가 빈번
- 아고다가 자체적으로 적용하는 할인(10% 등)이 업장 동의 없이 적용되어 설정가보다 낮은 가격으로 판매되는 사례 다수 보고됨
- 결과적으로 "설정한 가격"과 "실제 입금되는 금액"의 차이가 크게 발생할 수 있음
</Accordion>
</AccordionGroup>

### 3. 가격이 변경되지 않거나 이전 가격으로 되돌아감

- VCMS에서 요금을 변경했으나 아고다에 반영되지 않는 사례가 보고됨
- 마감을 설정해도 예약이 계속 들어오는 경우 존재
- 캐시 문제로 인해 특정 월의 상품이 아고다에서 노출되지 않는 현상도 확인됨

### 4. 아고다 계정에 여러 숙박업소가 등록된 경우

- 벤디트와 연동 시 **하나의 숙박업소만** 연동 가능
- 잘못 등록된 중복 숙박업소가 있으면 수동 관리 필요

## 가격·재고·정산 관점의 주의점

### 가격 설정 시 필수 확인사항

| 확인 항목 | 상세 |
|-----------|------|
| 입금가 기준 이해 | VCMS에 입력하는 금액 = 업장이 받을 입금가. 판매가가 아님 |
| 인원별 요금 활성화 | 추가 상품 등록 시 '인원별 요금 설정'을 반드시 활성화해야 오류 방지 |
| 기준 인원 확인 | YCS에서 기준 인원이 잘못 설정되면 추가 인원 요금을 받지 못함 |
| 요일별 추가 요금 | 아고다는 요일별 추가 금액이 다를 수 있으나 VCMS 정률제와 맞지 않는 경우 발생 |
| 채널 전송가 설정 | 수수료를 반영한 별도 채널 전송가 설정 필요 |

### 재고 관련

- 재고 동기화는 정상 작동하지만, 마감 설정 후에도 아고다 자체 캐시로 인해 즉시 반영되지 않을 수 있음
- '도보특가' 등 특수 상품의 경우 VCMS에서 재고를 닫아도 YCS에서 열려 있는 현상이 새벽 시간대에 발생한 사례 있음

### 정산 관련

- 정산 시스템이 불투명하고 지연이 심하다는 피드백 다수
- 가상 카드 결제 프로세스가 복잡 — 하나카드 해외 키인 결제 시 서울 보증 보험 가입 등 여러 단계 필요
- 가상 카드 번호의 24시간 유효 기간 주의

## 먼저 확인할 순서

<Steps>
  <Step title="YCS 기준 인원 및 최대 인원 확인">
    YCS에서 각 객실의 기준 인원과 최대 인원이 올바르게 설정되어 있는지 확인한다. 2인실인데 4명으로 표시되는 등의 불일치가 빈번하게 보고됨.
  </Step>
  <Step title="인원별 요금 설정 활성화 여부">
    추가 상품 등록 시 '인원별 요금 설정'이 활성화되어 있는지 확인. 비활성화 상태에서는 오류가 발생한다.
  </Step>
  <Step title="VCMS 입금가 설정 확인">
    VCMS에 입력된 금액이 '판매가'가 아닌 '입금가'임을 업장에 재확인. 혼동으로 인한 가격 오류가 가장 빈번한 이슈.
  </Step>
  <Step title="아고다 사이트에서 실제 노출 가격 확인">
    설정 후 반드시 아고다 웹사이트에서 실제 노출되는 판매가를 직접 확인한다. 입금가 기준이므로 예상과 다를 수 있음.
  </Step>
  <Step title="자체 프로모션 적용 여부 확인">
    업장이 설정하지 않은 프로모션('Gorocal Tonight' 등)이 자동 활성화되어 있지 않은지 YCS에서 확인.
  </Step>
  <Step title="연동 상태 및 캐시 확인">
    요금 변경 후 아고다에 반영되지 않는 경우, 캐시 문제일 수 있으므로 시간을 두고 재확인하거나 아고다 담당자에게 문의.
  </Step>
</Steps>

## 자주 오해하는 포인트

<AccordionGroup>
<Accordion title="VCMS에 설정한 가격 = 고객이 보는 가격이다?">
**아니다.** VCMS에 설정한 금액은 입금가이며, 아고다가 자체 마진·프로모션·세금을 더해 최종 판매가를 결정한다. 따라서 VCMS 설정가보다 훨씬 높거나, 프로모션 적용 시 오히려 낮게 보일 수 있다.
</Accordion>

<Accordion title="아고다 수수료가 15%이니까 15%만 더 올리면 된다?">
**실제로는 부족하다.** 프로모션 할인, 환전 수수료, 세금 등이 추가되어 실질 부담은 16.5% 이상이 될 수 있다. 20~30% 인상을 설정해도 아고다에서 가장 저렴하게 보이는 경우가 보고됨.
</Accordion>

<Accordion title="아동 요금을 VCMS에서 설정할 수 있다?">
**불가하다.** 아동 요금은 YCS(아고다 엑스트라넷)에서 직접 설정해야 한다.
</Accordion>

<Accordion title="국내 채널과 동일한 방식으로 가격을 설정하면 된다?">
**다르다.** 아고다는 인원별 요금을 한 번에 모두 전송해야 하며, 패밀리룸의 경우 특정 인원(6명)까지 추가 요금이 자동 적용되지 않는 등 국내 채널과 구조가 다르다. 야놀자·여기어때의 인원 추가 요금은 현장 결제 방식이지만, 아고다는 시스템 내에서 처리된다.
</Accordion>

<Accordion title="마감 설정하면 바로 예약이 차단된다?">
**즉시 반영되지 않을 수 있다.** 아고다 캐시 및 동기화 지연으로 인해 마감 후에도 예약이 들어오는 사례가 보고됨. 특히 새벽 시간대에 도보특가 등 특수 상품에서 발생 빈도가 높다.
</Accordion>
</AccordionGroup>

## 운영 팁

- **채널 전송가를 활용하라**: 기준가에서 아고다 수수료율(15~18%)을 반영한 별도 채널 전송가를 설정하면 수익성을 보호할 수 있다. 단, 실질 수수료가 표면 수수료보다 높으므로 여유를 두고 설정할 것.
- **YCS 구 버전 캘린더 활용**: 일부 요금제 설정(룸온리, 늦은 입실 등)은 YCS 구 버전 캘린더에서만 가능한 경우가 있다.
- **아고다 가격 매칭 메일에 대비하라**: 아고다는 다른 플랫폼보다 비싸면 가격 매칭 요구 메일을 보내는 경우가 있다. 채널별 가격 전략을 사전에 수립해 두는 것이 좋다.
- **연동 해제 후 수동 관리 사례**: 아고다가 동의 없이 할인 판매하여 손해를 본 경험으로 연동을 수동 중단한 업장 사례가 있다. 프로모션 통제가 어려운 경우의 대안으로 참고.
- **예약 연동만 사용 옵션**: 재고/요금 동기화 없이 예약 연동만 사용하는 것도 가능하다. 가격 통제를 중시하는 업장에 적합.
- **퇴실 시간 주의**: 아고다 예약은 퇴실 시간이 실제 퇴실일이 아닌 다음 날 14시로 설정되는 경우가 있어 운영상 확인 필요.
