# 아고다 프로모션 및 정산 가이드

아고다는 업장이 설정하지 않은 프로모션을 자체적으로 적용하는 채널이다. 이로 인해 판매가 통제가 어렵고, 정산 금액과 설정 금액 사이의 괴리가 발생한다. 이 문서는 프로모션 관련 혼란과 정산 시 유의사항을 정리한다.

---

## 아고다 자체 프로모션의 구조

### 업장 동의 없이 적용되는 프로모션

아고다는 플랫폼 차원에서 다양한 프로모션을 자체적으로 적용할 수 있다. 업장이 YCS에서 설정하지 않은 할인이 적용되는 경우가 빈번하게 보고된다.

| 프로모션 유형 | 설명 | 영향 |
|--------------|------|------|
| Gorocal Tonight | 당일 특가 프로모션 | 업장이 설정하지 않아도 활성화될 수 있음 |
| 요금 매칭 | 타 채널 대비 요금 일치 요구 | 메일로 요금 인하 요청이 옴 |
| 회원 전용 할인 | 아고다 회원 로그인 시 추가 할인 | 비로그인 요금과 로그인 요금이 다름 |
| 모바일 할인 | 모바일 앱 결제 시 추가 할인 | 웹 요금과 앱 요금이 다를 수 있음 |
| 자동 10% 할인 | 업장 설정 요금에서 자동 차감 | 의도한 판매가보다 낮게 노출 |

<Warning>
아고다에서 업장이 설정하지 않은 자동 할인 및 요금 표시가 발생하여 실제 판매 요금에 대한 혼란과 잠재적 수익 손실이 우려된다. 특히 10% 할인이 자체적으로 적용되어 설정가보다 낮은 요금에 객실이 판매되는 사례가 다수 보고되었다.
</Warning>

### 프로모션 적용 시 요금 흐름

```
VCMS 설정가 (입금가): 85,000원
  ↓ 아고다 마진 추가
아고다 기본 판매가: ~100,000원
  ↓ 아고다 자체 프로모션 적용 (10% 할인)
실제 고객 노출가: ~90,000원
  ↓ 회원 할인 추가 적용
회원 결제가: ~85,000원 이하

→ 결과: 업장은 85,000원을 설정했지만, 고객은 더 낮은 요금으로 결제
→ 업장에 입금되는 금액은 수수료 차감 후 85,000원보다 적을 수 있음
```

## 요금 매칭 메일 대응

### 매칭 메일이란?

아고다는 자사 플랫폼에서의 요금이 다른 채널(야놀자, 여기어때, 네이버 등)보다 비싸다고 판단되면 업장에 요금 매칭을 요구하는 메일을 보낸다.

<AccordionGroup>
<Accordion title="요금 매칭 메일의 주요 내용">
- "다른 플랫폼보다 비싸다"는 비교 데이터 포함
- 요금 인하 요청 또는 자동 조정 제안
- 응하지 않을 경우 노출 순위에 영향을 줄 수 있음을 시사
</Accordion>

<Accordion title="대응 전략">
1. **채널별 요금 전략을 사전 수립**: 아고다는 수수료가 높으므로 다른 채널보다 요금이 높은 것이 정상. 이를 아고다 담당자에게 설명할 근거를 준비
2. **입금가 기준 설명**: "동일한 입금가를 설정했으나, 아고다의 마진이 크기 때문에 판매가가 높게 보이는 것"이라는 논리
3. **프로모션 활용 vs 직접 인하 판단**: 아고다 자체 프로모션을 활용하면 업장 부담 없이 요금이 조정되지만, 업장이 직접 인하하면 입금가가 감소
4. **무시 가능**: 법적 강제력이 없으므로 요금 매칭에 응할 의무는 없음
</Accordion>
</AccordionGroup>

## 정산 관련 주의사항

### 정산 구조의 불투명성

아고다의 정산 구조에 대해 업장으로부터 다음과 같은 피드백이 반복적으로 확인된다:

| 이슈 | 상세 |
|------|------|
| 불투명한 정산 시스템 | 수수료, 세금, 프로모션 비용이 복합적으로 적용되어 실제 입금 금액 예측이 어려움 |
| 정산 지연 | 다른 국내 OTA 대비 정산 주기가 길고 지연이 빈번 |
| 판매가 미제공 | 공식 연동 시 입금가만 제공하여 실제 판매가 파악이 어려움 |
| 자동 할인 차감 | 업장 동의 없는 프로모션으로 인한 할인분이 업장 부담이 될 수 있음 |
| 페널티 부과 | 특정 조건 위반 시 페널티가 부과되는 사례 |

### 매출 부가세 신고 시 주의

<Warning>
아고다 연동 시 VCMS에 들어오는 금액은 입금가이므로, 매출 부가세 신고를 위해서는 판매가를 별도로 역산해야 한다. 아고다의 'Sell Inclusive' 방식(수수료와 세금을 포함하여 표시)으로 인해 실제 판매가와 입금가의 관계가 단순하지 않다.
</Warning>

### 정산 금액 확인 체크리스트

<Steps>
  <Step title="VCMS 예약 내역에서 입금가 확인">
    VCMS에 표시되는 아고다 예약의 금액은 입금가이다. 이 금액이 실제 업장에 입금될 예정 금액과 일치하는지 확인한다.
  </Step>
  <Step title="YCS 정산 보고서와 대조">
    YCS에서 제공하는 정산 보고서와 VCMS의 입금가를 대조한다. 프로모션 적용 건의 경우 차이가 발생할 수 있다.
  </Step>
  <Step title="프로모션 적용 건 별도 표시">
    자체 프로모션이 적용된 예약 건은 별도로 표시하여 관리한다. 이 건들의 입금가가 일반 건보다 낮을 수 있다.
  </Step>
  <Step title="가상카드 결제 건 별도 관리">
    가상 카드로 결제된 건은 24시간 유효 기간이 있으므로 빠른 처리가 필요하다.
  </Step>
</Steps>

## 가상 카드 결제 프로세스

아고다의 가상 카드 결제는 국내 업장에게 복잡한 절차를 요구한다.

| 단계 | 내용 |
|------|------|
| 1 | 아고다에서 가상 카드 번호 발급 |
| 2 | 하나카드를 통한 해외 키인 결제 진행 |
| 3 | '서울 보증 보험' 가입 필요 (사전 요건) |
| 4 | 관련 서류 제출 |
| 5 | 24시간 이내에 결제 처리 (가상 카드 유효 기간) |

<Warning>
가상 카드 번호는 **24시간 유효**하다. 이 기간 내에 결제를 처리하지 못하면 새로운 가상 카드를 요청해야 하며, 이 과정에서 추가 지연이 발생할 수 있다.
</Warning>

## VCMS에서의 아고다 요금 인상 설정

### 수수료 반영을 위한 채널 전송가 설정

VCMS에서 아고다 채널에 대해 별도의 요금 인상을 설정하여 수수료를 반영할 수 있다.

<AccordionGroup>
<Accordion title="퍼센트 기반 인상 설정">
기준가에서 일정 퍼센트를 인상하여 아고다에 전송하는 방식.

- 예: 기준가 100,000원 + 15% = 115,000원 아고다 전송
- 장점: 기준가 변경 시 자동 반영
- 단점: 지저분한 뒷자리 금액 발생 가능, 실질 수수료가 표면 수수료보다 높아 부족할 수 있음
</Accordion>

<Accordion title="고정 금액 인상 설정">
기준가에서 고정 금액을 더하여 전송하는 방식.

- 예: 기준가 100,000원 + 15,000원 = 115,000원 아고다 전송
- 장점: 깔끔한 요금 설정
- 단점: 기준가 변경 시 비율이 달라짐
</Accordion>
</AccordionGroup>

### 인상률 설정 시 고려사항

| 항목 | 권장 |
|------|------|
| 표면 수수료 | 15% |
| 프로모션 할인 감안 | +3~5% |
| 환전·세금 감안 | +1~2% |
| **안전 권장 인상률** | **+18~22%** |

<Warning>
20% 인상을 설정하고, 아고다에서 추가로 30%, 60%를 설정해도 아고다에 노출되는 최종 요금과 호텔이 받는 정산 금액이 인상분을 제대로 반영하지 않는 사례가 보고되었다. 설정 후 반드시 실제 노출 요금과 정산 금액을 확인할 것.
</Warning>

## 자주 오해하는 포인트

<AccordionGroup>
<Accordion title="아고다에서 가장 저렴하게 보이면 예약이 많이 들어올 것이다?">
높은 마진을 붙여도 아고다에서 가장 저렴하게 보이는 문제가 있다. 이는 아고다의 'Sell Inclusive' 방식과 자체 프로모션 때문이며, 저렴하게 보인다고 해서 업장에 유리한 것은 아니다. 입금가 기준으로 수익성을 판단해야 한다.
</Accordion>

<Accordion title="아고다 프로모션 할인분은 아고다가 부담한다?">
프로모션 유형에 따라 다르다. 일부 프로모션은 아고다가 자체 마진에서 차감하지만, 다른 프로모션은 업장 입금가에 영향을 줄 수 있다. 정산 보고서에서 개별 건의 프로모션 적용 내역을 확인해야 한다.
</Accordion>

<Accordion title="정산 금액이 안 맞으면 VCMS 문제인가?">
대부분의 경우 VCMS 문제가 아니라 아고다의 정산 구조에 기인한다. VCMS는 입금가를 정확히 표시하지만, 아고다의 프로모션·수수료·세금 적용에 의해 실제 입금 금액이 달라질 수 있다.
</Accordion>
</AccordionGroup>

## 운영 팁

- **프로모션 모니터링 루틴을 만들어라**: 월 1회 이상 YCS에서 활성화된 프로모션 목록을 확인하고, 업장이 설정하지 않은 항목이 있으면 비활성화 또는 아고다 담당자에게 문의한다.
- **입금가 기반 수익 계산 습관**: 판매가가 아닌 입금가 기준으로 수익을 계산하는 습관을 들인다. 판매가는 아고다가 통제하는 영역이다.
- **요금 매칭 메일은 냉정하게 판단**: 요금 매칭에 응할 의무는 없다. 채널별 수수료 구조가 다르므로 동일 판매가가 아닌 동일 입금가 기준으로 요금을 설정하는 것이 합리적이다.
- **예약 연동만 사용하는 옵션 고려**: 아고다의 프로모션 통제가 너무 어려운 경우, 재고/요금 동기화 없이 예약 연동만 사용하고 요금은 YCS에서 직접 관리하는 것도 대안이다.
- **정산 보고서 정기 대조**: VCMS 입금가, YCS 정산 보고서, 실제 입금 내역을 월별로 대조하여 차이가 발생하는 건을 식별한다.
- **아고다 수수료에 대한 업장 교육**: 아고다의 수수료가 높은 만큼, 다른 채널과 동일한 판매가를 적용하면 안 된다는 점을 업장에 명확히 전달한다.
