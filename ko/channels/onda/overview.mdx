온다 플러스는 일반적인 OTA와 다른 판매 대행 구조를 가진 채널입니다. 직접 판매와 대행 판매가 혼재하며, 요금 변경이 일부 채널에만 반영되는 구조적 한계, 이전 요금 잔류 문제, 0원 표시 위험 등 고유한 운영 이슈가 있습니다.

## 관리자 바로가기

- [온다 펜션 플러스](https://pension.onda.me)
- [온다 호텔 플러스](https://tporthotel.extranet.onda.me)

## 채널 특성 요약

| 항목 | 내용 |
|------|------|
| 채널 유형 | **판매 대행** — 직접 판매 + 대행 판매 혼재 |
| 요금 반영 | 실시간 예약창에만 적용, **대행 채널에는 별도 반영** 필요 |
| 이전 요금 | 요금 변경 후에도 **이전 요금이 잔류**할 수 있음 |
| 0원 위험 | 설정 오류 시 **0원으로 표시·판매**될 위험 |
| 당일특가 | 온다 고유 기능 — 다른 채널에는 없음 |
| 재고 방식 | 자체 묶음 방식 — VCMS 기준과 다를 수 있음 |
| 서버 안정성 | 간헐적 접속 지연·오류 보고 |

## 반복되는 운영 이슈

### 1. 판매 대행 구조의 이중성

온다 플러스는 자체 사이트에서 직접 판매하는 상품과, 외부 OTA에 대행 판매하는 상품이 하나의 시스템에서 혼재됩니다. VCMS에서 설정한 요금이 직접 판매(실시간 예약창)에는 반영되지만, 대행 판매 채널에는 별도로 반영되어야 할 수 있습니다. 이 구조를 모르는 업장은 "요금을 바꿨는데 왜 특정 채널에서는 이전 요금으로 예약이 들어오냐"는 혼란을 겪습니다. 직접 판매와 대행 판매의 재고도 별도로 관리될 수 있어, 한쪽에서 매진되어도 다른 쪽에서 예약이 가능한 상황이 발생합니다.

### 2. 이전 요금 잔류 문제

온다 플러스에서 가장 빈번하게 보고되는 문제입니다. 업장이 VCMS에서 요금을 인상하면 야놀자·여기어때 등 다른 채널에는 정상 반영되지만, 온다에서는 이전(낮은) 요금이 그대로 유지되는 현상이 발생합니다. 고객이 온다를 통해 이전 요금으로 예약하면 업장은 의도하지 않은 저가 판매로 손실을 입습니다. 이 문제의 원인은 온다의 요금 반영 구조에 있습니다 — 실시간 예약창에만 적용되고 온라인 판매 대행 채널에는 별도 처리가 필요한 것으로 추정됩니다.

CS에서 확인한 사례: "가격 인상 후에도 온다 채널에서만 이전 가격으로 예약이 들어와 판매 전략에 혼란과 손실이 발생했다." 요금 변경 후에는 반드시 온다의 직접 사이트와 대행 판매 채널 양쪽에서 반영 여부를 확인해야 합니다.

### 3. 0원 표시 위험

온다 플러스에서 요금이 0원으로 표시되어 판매될 위험이 보고되었습니다. 요금 설정 오류, 연동 과정의 데이터 누락, 서버 불안정 등이 원인일 수 있습니다. 0원으로 표시된 상태에서 예약이 들어오면 업장에 즉각적인 재정 손실이 발생하므로, 요금 변경 후 즉시 노출 요금을 확인하는 루틴이 필수입니다. 다른 채널에서는 이런 수준의 0원 위험이 보고된 적이 거의 없어, 온다 특유의 리스크로 분류됩니다.

### 4. 당일특가 기능

온다 플러스는 당일 판매되지 않은 객실을 특별 할인가로 판매하는 당일특가 기능을 고유하게 제공합니다. 이 기능은 다른 판매 채널(야놀자, 여기어때 등)에는 없으므로, 당일특가 요금이 다른 채널의 정상가보다 지나치게 낮으면 채널 간 요금 불일치 문제가 발생합니다. 당일특가를 다른 채널에도 적용하고 싶다는 수요가 있으나 타 플랫폼에서는 지원하지 않습니다. 당일특가 요금을 적절히 상향 조정하면 수익을 최적화할 수 있는 여지가 있습니다.

### 5. 서버 불안정

온다 플러스의 서버가 간헐적으로 불안정하다는 보고가 있습니다. 접속 지연, 에러 발생, 계정 접근 불가 등이 보고되며, 중요한 요금 변경이나 재고 조정은 서버 상태가 안정적인 시간대에 수행하는 것이 권장됩니다. 대표님이 직접 관리하는 계정 정보를 직원이 알지 못하는 경우도 빈번하여, 긴급 대응이 지연되기도 합니다.

### 6. 등록 재고 기준의 채널상품 재고 제한

온다는 연결된 채널상품 재고를 무한정 풀어두는 구조로 보면 안 됩니다. 운영 현장에서는 **등록된 실제 재고만큼만 채널상품 재고를 제한**해야 안전하다는 패턴이 반복됩니다. 특히 온다 펜션 플러스에서 직접 판매 상품, 대행 판매 상품, VCMS 연결 상품을 함께 쓰는 업장은 채널상품 재고를 더 크게 잡으면 실제 객실 수보다 많이 팔릴 위험이 커집니다.

## 호텔 업장 추가 주의사항

온다 플러스는 펜션 외에도 호텔 업장을 지원합니다. 호텔 업장에서는 펜션과 다른 몇 가지 운영 특성이 있습니다.

### 호텔 vs 펜션 연동 차이

| 항목 | 펜션 | 호텔 |
|------|------|------|
| 상품 구조 | 독채/객실 단위 | 객실 타입 단위 |
| 재고 관리 | 호실별 | 타입별 |
| 대행 채널 | 펜션 전문 OTA 위주 | 호텔 전문 OTA 포함 |
| 요금 구조 | 인원별 요금 중심 | 객실 요금 중심 |

### 호텔 연동 시 주의사항

온다 플러스를 통해 호텔 상품이 외부 OTA에 대행 판매될 때, 어떤 채널에 어떤 요금으로 노출되는지 정기적으로 확인해야 합니다. 온다 플러스의 대행 판매 채널 목록과 각 채널의 수수료 구조를 파악해두면 예상치 못한 저가 판매를 방지할 수 있습니다.

## 운영 체크리스트

<Steps>
  <Step title="요금 변경 후 이중 확인">
    요금 변경 시 (1) 온다 직접 사이트(실시간 예약창), (2) 대행 판매 채널에서 각각 반영 여부를 확인합니다. 온다에서만 이전 요금이 남아 있는 문제를 사전에 방지합니다.
  </Step>
  <Step title="0원 표시 여부 즉시 확인">
    요금 설정 또는 변경 후, 0원으로 표시되어 판매될 위험이 없는지 즉시 확인합니다.
  </Step>
  <Step title="당일특가 요금 수준 점검">
    당일특가가 활성화되어 있다면, 다른 채널의 정상가 대비 적절한 수준인지 정기적으로 확인합니다. 지나치게 낮으면 상향 조정합니다.
  </Step>
  <Step title="직접 판매·대행 판매 구분 이해">
    온다의 직접 판매 상품과 대행 판매 상품이 각각 어떤 채널에 노출되는지, 재고·요금 반영 범위가 어디까지인지 파악합니다.
  </Step>
  <Step title="등록 재고 기준으로 채널상품 상한 설정">
    채널상품 재고는 실제 등록된 재고를 넘지 않도록 제한합니다. 특히 독채, 원룸, 소수 객실 업장은 온다 쪽 재고 상한을 넉넉하게 두지 말고 실제 운영 수량 기준으로 먼저 맞추세요.
  </Step>
  <Step title="서버 안정 시간대에 주요 작업 수행">
    중요한 요금 변경이나 재고 조정은 온다 서버가 안정적인 시간대에 수행하고, 작업 후 반영 상태를 확인합니다.
  </Step>
</Steps>
