# 떠나요 — 채널 특이사항

떠나요는 주로 펜션 업장에서 사용되는 채널로, 자체 운영 기능과 OTA 판매 대행을 겸하는 독특한 구조입니다. 요금 상한선, 타입 기반 요금 체계, IP/기기 인증, 야놀자와의 재고 충돌 등 다른 채널에서는 볼 수 없는 고유한 제약이 있습니다.

## 채널 특성 요약

| 항목 | 내용 |
|------|------|
| 채널 유형 | 자체 운영 + OTA **판매 대행** 겸용 |
| 연동 방식 | **리피터 프로그램** 기반 — PC 상시 가동 필요 |
| 인증 | **IP 인증 + 기기 인증** (2차 인증) |
| 요금 구조 | **타입 기반** (호실별 아님) |
| 요금 제한 | 채널 자체 **요금 상한선** 존재 |
| 재고 관리 | '열기'/'막기'만 지원 — 세부 수량 조절 불가 |
| 수수료 | 기본 플랫폼 수수료 + 판매 대행 수수료 (약 13%) |
| 다박 처리 | **1박 단위로 분할** |

---

## 반복되는 운영 이슈

### 1. 요금 상한선

떠나요에서 가장 독특한 제약입니다. 채널 자체에 요금 상한선이 있어, VCMS에서 설정한 요금이 이 상한선을 초과하면 해당 객실이 자동으로 막히거나 최대가로만 적용됩니다. 요금 인상이 자유롭지 않고, 무조건 할인 형태로만 요금이 설정되는 구조이므로 성수기 요금 전략에 특히 제약이 큽니다. 상한선을 초과하는 요금을 설정하면 해당 날짜에 판매 자체가 불가능해질 수 있어, 설정 전에 반드시 떠나요의 상한선을 확인해야 합니다.

### 2. 타입 기반 요금 구조의 혼란

떠나요는 호실별이 아닌 타입별로 요금을 설정합니다. 동일 타입 내의 모든 호실이 같은 요금이어야 하는데, 실제로는 VIP 원룸(402~405호)의 요금이 65,000원과 120,000원으로 다르게 표시되는 사례가 보고됩니다. 동일 타입 내 요금이 불일치하면 시스템이 요금을 변경하려는 현상이 반복 발생합니다. 떠나요에 금액을 입력할 때 수수료를 제외하고 입력해야 하는 번거로움도 있으며, 이 부분이 자동 처리되지 않아 실수가 발생할 수 있습니다.

### 3. IP 인증 및 기기 인증

떠나요 연동의 가장 빈번한 장애 요인입니다. 여러 컴퓨터에서 접속 시 매번 IP 인증을 해야 하며, 새로운 기기에서 접속하면 기존 인증이 해제됩니다. 기기 인증(2차 인증)은 업장 PC에서만 가능하여 원격 설정이 불가능합니다. 떠나요 리피터 프로그램이 PC에 상시 실행되어야 연동이 유지되므로, 노트북을 닫거나 PC를 끄면 연동이 즉시 중단됩니다. 오래된 컴퓨터에서는 성능 저하와 접속 불가 문제도 발생합니다.

### 4. 야놀자 동시 판매 리스크

야놀자와 떠나요가 동일한 객실에 대해 동시에 판매하는 경우, 재고 충돌과 오버부킹 위험이 있습니다. 서로 다른 사업자 등록으로 같은 객실에 경쟁적으로 예약이 들어오면, 한쪽 예약이 다른 쪽의 재고에 반영되지 않아 이중 예약이 발생합니다. 떠나요 예약은 호실이 고정되어 VCMS에서 변경이 불가능하므로, 다른 채널의 방 배정에도 제약이 생깁니다. 수수료 이중 부과 가능성도 있어, 양 채널의 재고 연동 상태를 면밀히 확인해야 합니다.

### 5. 다박 예약 분할 처리

떠나요는 다박 예약을 1박 단위로 분할하여 처리합니다. 15박 예약이 시스템에서 15개의 개별 1박 예약으로 나뉘어 표시되며, 예약 박수 계산 방식도 VCMS와 달라 혼란이 생깁니다. 수동 병합이 필요하고, 각 예약의 금액·날짜 변경도 개별 대응해야 합니다.

## 운영 체크리스트

<Steps>
  <Step title="요금 상한선 확인">
    떠나요의 요금 상한선을 확인하고, 설정하려는 요금이 상한선 이내인지 점검합니다. 성수기 요금 설정 시 특히 주의합니다.
  </Step>
  <Step title="리피터 프로그램 및 PC 상시 가동 확인">
    떠나요 리피터 프로그램이 업장 PC에서 정상 실행 중인지, 전용 PC가 상시 가동되고 있는지 확인합니다. 노트북 사용 시 덮개를 닫지 않도록 안내합니다.
  </Step>
  <Step title="IP/기기 인증 상태 점검">
    현재 사용 중인 PC에서 IP 인증과 기기 인증이 완료되어 있는지 확인합니다. 다른 기기에서 접속한 이력이 있으면 재인증이 필요합니다.
  </Step>
  <Step title="타입 내 요금 통일 확인">
    동일 타입 내 모든 호실의 요금이 통일되어 있는지 확인합니다. 불일치 시 시스템 오류가 반복됩니다.
  </Step>
  <Step title="야놀자 재고 연동 상태 확인">
    야놀자와 떠나요가 동시에 같은 객실을 판매하고 있지 않은지 확인합니다. 재고 충돌과 오버부킹을 방지합니다.
  </Step>
</Steps>
