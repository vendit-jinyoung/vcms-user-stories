네이버 채널에서 가장 빈번하게 장애가 발생하는 영역은 **인증**과 **캘린더 설정 초기화**입니다. 네이버의 보안 정책이 외부 시스템 연동에 민감하게 반응하며, 동기화를 진행하면 기존에 수동 설정한 공휴일·오늘의 딜 정보가 사라집니다. 네이버 도움말에서는 보통 **애플리케이션 전용 암호(앱비밀번호)** 라는 표현을 쓰고, 현장에서는 이를 애플리케이션 암호나 앱비밀번호라고 함께 부릅니다.

## 채널 특성 요약

| 항목 | 내용 |
|------|------|
| 인증 방식 | **2차 인증(2FA) + 애플리케이션 전용 암호(앱비밀번호)** 필수 |
| 보안 정책 | 타지역 IP 접속 자동 차단, 과도한 로그인 시도 잠금 |
| 동기화 범위 | 최대 **180일** |
| 공휴일 처리 | 동기화 시 **기존 공휴일 설정 전체 삭제** |
| 오늘의 딜 | 동기화 후 **비활성화** |
| 계정 구조 | 간편 로그인만 지원, 단체 계정 연동 불가 |

## 반복되는 운영 이슈

### 1. 2FA 및 앱비밀번호 장애

네이버 연동이 조용히 끊기는 가장 흔한 원인은 2차 인증 상태 변경입니다. 2차 인증이 해제되거나 비밀번호가 변경되면 기존에 발급받았던 애플리케이션 전용 암호(앱비밀번호)가 무효화되어 연동이 끊깁니다. 이때 별도의 알림이 없어 업장이 인지하기까지 시간이 걸리고, 그 사이 요금·재고 동기화가 중단됩니다. 2차 인증 수신 번호가 대표(또는 가족)의 핸드폰으로만 수신되어, 해당 인물이 부재 시 연동 재설정이 불가능한 상황도 자주 발생합니다.

역설적으로 2차 인증을 끄면 앱비밀번호를 발급받을 수 없어 연동이 오히려 불가능해집니다. CS에서는 이렇게 안내합니다: "2차 인증은 반드시 켜둔 상태에서 앱비밀번호를 사용하는 것이 권장됩니다. 끄면 연동이 불가능합니다."

### 2. 타지역 로그인 차단과 보호조치

네이버의 보안 시스템은 자동화된 외부 로그인에 민감합니다. VCMS가 네이버에 접속하는 방식이 의심스러운 접근으로 감지되어 보호조치가 자동 발동되는 경우가 있으며, 이때 계정 자체가 잠겨 로그인 자체가 불가능해집니다. 네이버 계정에 해외 IP 또는 특정 지역 IP 차단 설정이 되어 있으면 벤디트 서버에서의 접속이 원천 차단됩니다. 과도한 로그인 시도가 누적되면 계정에 제한이 걸리며, 단체 계정(법인 명의)에서는 간편 로그인이 불가능하여 연동 자체에 제약이 있습니다.

### 3. 공휴일 캘린더 덮어쓰기

VCMS와 재고/요금 동기화를 진행하면 네이버에 설정된 공휴일 정보가 전체 삭제됩니다. 업장이 수개월에 걸쳐 수동 입력한 공휴일 요금이 한 번에 사라지며, 해당 날짜에 평일/주말 요금이 대신 적용되어 손실이 발생합니다. 네이버에서 자동으로 관리되던 "공휴일이면 하루씩 열리는" 방식도 연동 후 작동하지 않습니다. 이는 VCMS의 동기화가 VCMS 데이터로 네이버 데이터를 덮어쓰는 구조이기 때문이며, 네이버에만 존재하는 필드(공휴일, 오늘의 딜 등)는 대응하는 데이터가 없어 삭제되는 것입니다.

CS에서는 동기화 전에 네이버 공휴일 설정을 캡처하여 보관하고, 동기화 후 VCMS에서 재설정하라고 안내합니다.

### 4. 180일 동기화 제한과 오늘의 딜

네이버 채널의 동기화 범위는 최대 180일로 제한됩니다. 이 기간을 넘는 예약에 대해서는 요금이 전달되지 않습니다. 또한 동기화 후에는 네이버의 '오늘의 딜' 기능이 비활성화되어, 네이버에서 직접 관리할 수 없게 됩니다. 오늘의 딜을 적극적으로 사용하는 업장에는 이것이 상당한 운영 제약으로 작용합니다. 연동 전에 오늘의 딜 사용 빈도와 의존도를 반드시 파악해야 합니다.


### 5. 계정 명의 불일치 문제

네이버 스마트플레이스 인증이 다른 법인/개인 명의로 잘못 등록된 경우, 계정 이전 및 재설정 작업이 필요합니다. 주 관리자 계정이 현재 담당자가 아닌 매니저나 전 직원 계정으로 되어 있어 설정에 어려움이 발생하는 사례도 빈번합니다. 연동 전에 반드시 네이버 스마트플레이스의 주 관리자 계정이 누구인지 확인하세요.

## 자주 오해하는 포인트

<AccordionGroup>
  <Accordion title="2차 인증을 끄면 연동이 더 쉬워지지 않나요?">
    반대입니다. 2차 인증을 끄면 애플리케이션 전용 암호(앱비밀번호)를 발급받을 수 없어 VCMS 연동 자체가 불가능해집니다. 2차 인증을 켠 상태에서 앱비밀번호를 발급받아 사용하는 것이 올바른 설정입니다.
  </Accordion>
  <Accordion title="네이버 비밀번호를 바꿨더니 연동이 끊겼어요">
    네이버 비밀번호 변경 시 기존 앱비밀번호가 무효화됩니다. 비밀번호 변경 후에는 VCMS 연동 상태를 반드시 확인하고, 필요 시 앱비밀번호를 재발급합니다.
  </Accordion>
  <Accordion title="단체 계정(법인 명의)으로 연동할 수 있나요?">
    네이버 간편 로그인만 지원되므로, 단체 계정은 간편 로그인이 불가능하여 연동이 안 될 수 있습니다. 개인 네이버 계정으로 변경하거나 새 계정 생성이 필요할 수 있습니다.
  </Accordion>
  <Accordion title="공휴일 요금을 네이버에서 직접 설정하면 유지되나요?">
    VCMS 연동 상태에서 네이버에 직접 설정한 공휴일 요금은 다음 동기화 시 삭제됩니다. 공휴일 요금은 VCMS에서 설정해야 유지됩니다.
  </Accordion>
  <Accordion title="오늘의 딜을 계속 쓸 수 있나요?">
    VCMS 동기화 후에는 네이버에서 직접 오늘의 딜을 관리할 수 없습니다. VCMS 판매 관리 화면에서 대체 운영해야 합니다. 오늘의 딜에 크게 의존하는 업장은 동기화 범위를 신중하게 결정해야 합니다.
  </Accordion>
  <Accordion title="주 관리자 계정이 전 직원 명의로 되어 있어요">
    네이버 스마트플레이스의 주 관리자 계정이 전 직원이나 다른 명의로 되어 있으면, 해당 계정으로 로그인하거나 계정 이전 절차를 진행해야 합니다. 네이버 고객센터에 계정 이전을 요청하거나, 새 계정으로 스마트플레이스를 재등록하는 방법이 있습니다. 재등록 시 기존 리뷰와 데이터가 유지되는지 사전에 확인하세요.
  </Accordion>
</AccordionGroup>

## 운영 체크리스트

<Steps>
  <Step title="2차 인증 및 앱비밀번호 상태 확인">
    네이버 계정 설정에서 2차 인증이 '사용' 상태인지, 앱비밀번호가 정상 발급되어 있는지 확인합니다. 2차 인증 수신 번호에 현재 접근 가능한지도 확인합니다.
  </Step>
  <Step title="IP 차단·보호조치 점검">
    네이버 계정에 해외 IP 또는 특정 지역 IP 차단 설정이 되어 있지 않은지, 보호조치가 걸려 있지 않은지 확인합니다. 보호조치가 걸려 있으면 네이버 고객센터를 통해 해제합니다.
  </Step>
  <Step title="동기화 전 공휴일·오늘의 딜 백업">
    VCMS 동기화 전에 네이버에 설정된 공휴일 정보와 오늘의 딜 설정을 캡처하여 보관합니다.
  </Step>
  <Step title="동기화 후 공휴일 재설정">
    동기화 완료 후 VCMS 판매 관리 화면에서 공휴일 요금을 다시 입력합니다. 네이버 스마트플레이스에서 반영 여부를 확인합니다.
  </Step>
  <Step title="스마트플레이스 계정 구조 파악">
    주 관리자 계정, 실제 운영자, 2차 인증 수신자가 누구인지 파악합니다. 야놀자 경유로 등록된 네이버 플레이스가 있는지도 확인합니다.
  </Step>
</Steps>
