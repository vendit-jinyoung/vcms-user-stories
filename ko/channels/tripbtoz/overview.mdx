트립비토즈는 현재 이 저장소의 VOC 근거가 다른 주요 채널만큼 두껍지는 않지만, `docs.vcms.io`의 연동 가능 채널 안내에는 포함되어 있습니다. 그래서 이 페이지는 과장된 기능 설명 대신, **실제 파트너센터 진입 경로와 초기 연동 전에 반드시 확인할 체크포인트**를 정리합니다.

## 관리자 바로가기

- [트립비토즈 파트너센터](https://t-inven.tripbtoz.com)

## 먼저 확인할 점

| 항목 | 확인 포인트 |
|------|-------------|
| 관리자 경로 | 공개 소비자 사이트가 아니라 파트너센터(`t-inven`) 계정인지 확인 |
| 계정 관리 | 대표 계정/실무 계정 중 실제 운영자가 접근 가능한 계정인지 확인 |
| 샘플 예약 검증 | 첫 연동 직후 샘플 예약 1건으로 예약 소스 표기와 재고 차감 확인 |
| 상품 매핑 | 객실 타입 이름이 VCMS와 다르면 매핑 누락 여부 점검 |
| 운영 기준 | 가격/재고를 VCMS 기준으로 볼지, 트립비토즈 파트너센터 기준으로 볼지 먼저 결정 |

## 운영 메모

트립비토즈는 공개 고객 사이트와 파트너센터가 분리되어 있으므로, 실무자는 소비자 화면이 아니라 파트너센터 기준으로 상태를 확인해야 합니다. 채널 수가 많아질수록 "로그인은 됐는데 어느 화면이 기준인지"부터 헷갈리는 경우가 많기 때문에, 첫 세팅 때 기준 화면을 통일하는 것이 중요합니다.

또한 현재 이 저장소의 근거 기준에서는 트립비토즈만의 반복 장애 패턴이 충분히 누적되어 있지 않습니다. 그래서 첫 도입 업장은 다른 채널처럼 "이미 알려진 문제"를 가정하기보다, **샘플 예약/취소/마감 3가지를 직접 검증**하면서 운영 규칙을 잡는 편이 안전합니다.
