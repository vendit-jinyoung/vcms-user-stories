야놀자의 요금 관련 동작에는 다른 채널에서는 볼 수 없는 고유한 특성이 여러 가지 있습니다. 프로모션이 별도 상품으로 생성되는 구조, 도보특가·오픈특가가 의도치 않게 열리는 현상, 아고다로의 교차 판매 등을 이해하지 않으면 예상 밖의 저가 판매가 발생합니다.

## 채널 특성 요약

| 항목 | 내용 |
|------|------|
| 요금 되돌림 | VCMS → 야놀자 덮어쓰기로 인한 수동 변경 무효화 |
| 프로모션 구조 | 기존 상품 할인이 아닌 **별도 상품 생성** 방식 |
| 도보특가·오픈특가 | 별도 설정 없이도 **자동으로 열릴 수 있음** |
| 교차 판매 | 야놀자가 아고다에 자체 상품으로 **교차 판매** |
| 쿠폰 적용 | 야놀자 자체 쿠폰으로 판매가 하락 |
| 요금 단위 | 퍼센트 기반 설정 시 뒷자리 금액 발생 |

## 반복되는 운영 이슈

### 1. 프로모션이 별도 상품으로 생성되는 구조

야놀자의 프로모션 시스템(핫딜, 오픈특가, 당일 할인 등)은 기존 상품에 할인을 적용하는 방식이 아니라 별도의 프로모션 상품을 새로 만드는 방식으로 작동합니다. 이 때문에 프로모션을 생성할 때마다 VCMS에서 새 상품을 인식하고 매핑하는 작업이 필요합니다. 프로모션 상품의 요금이 VCMS 기준가와 다르게 적용될 수 있으며, 프로모션이 종료되어도 매핑이 자동 정리되지 않습니다. 의도한 것보다 훨씬 낮은 요금으로 객실이 판매되는 사례가 프로모션 매핑 누락에서 비롯되는 경우가 많습니다.

### 2. 도보특가·오픈특가의 자동 열림

야놀자의 도보특가(주변 위치 기반 특가)와 오픈특가는 업장이 명시적으로 설정하지 않아도 특정 조건에서 자동으로 활성화될 수 있습니다. 모든 객실을 마감 처리했는데도 특정 오픈특가 요금제만 활성 상태로 남아 있어, 의도치 않은 저가 예약이 발생하는 사례가 보고됩니다. 또한 시스템이 자동으로 90일치 요금제를 열어버려 주말/주중 요금 구분 없이 평일 요금으로 일괄 적용되는 현상도 있습니다. 도보 전용 상품의 요금이 일반 상품과 동일하게 설정되어 할인이 적용되지 않는 경우도 확인됩니다.

### 3. 아고다로의 교차 판매

야놀자는 채널링 계약을 통해 자사 상품을 아고다에서 판매할 수 있습니다. 업장이 인지하지 못한 채 야놀자 상품이 아고다에 노출되고, 아고다 예약이 야놀자 예약으로 표시되는 혼란이 발생합니다. 이 구조로 인해 이중 수수료 문제가 생길 수 있으며, 업장이 아고다에 직접 설정한 요금과 야놀자를 통해 들어간 요금이 충돌하여 같은 숙소가 아고다에서 서로 다른 가격으로 노출되는 상황도 발생합니다.

CS에서는 이렇게 안내합니다: "아고다 예약인데 야놀자로 표시되는 것은 야놀자의 교차 판매 구조 때문입니다. 이는 오류가 아닌 정상 동작이지만, 요금 통제와 수수료 측면에서 주의가 필요합니다."

### 4. 요금 되돌림의 세부 패턴

요금 되돌림은 단순히 "수동 변경이 복원된다"를 넘어 여러 세부 패턴이 있습니다. 성수기/준성수기 요금 전환 시점에 시스템 설정과 업장 기대가 불일치하여 잘못된 요금이 적용된 사례(7월 19일 준성수기 요금이 평수기로 적용), 삼일절 같은 공휴일에 주말 요금을 적용하고 싶었지만 야놀자에서 해당 날짜의 요금 변경 섹션이 잠겨 변경 불가능한 사례 등이 보고됩니다. 퍼센트 기반 채널 전송가를 사용하면 121,000원 같은 이상한 금액이 노출되어 업장이 혼란을 겪는 경우도 있습니다.

### 5. 야놀자 자체 할인 적용

야놀자도 아고다처럼 자체 쿠폰이나 공격적인 가격 정책을 적용합니다. 야놀자의 공격적인 가격 정책으로 호텔 자체 가격보다 낮은 가격으로 판매되는 사례, 의도치 않은 과도한 할인이 적용되어 객실이 저렴하게 판매되는 문제가 확인됩니다. VVIP 객실의 야놀자 판매가(85,000원)와 VCMS 설정가(80,000원)가 다른 것도 야놀자 측 마진이나 쿠폰 적용에 따른 차이일 수 있습니다.


### 6. 판매 대행 구조와 수수료 이중 부담

야놀자가 판매 대행 업체로 기능하는 경우, 기본 플랫폼 수수료에 추가 수수료가 부과되어 업장의 총 수수료 부담이 높아집니다. 채널 연동 방식 변경 시 야놀자에 쌓인 리뷰 데이터가 손실될 수 있다는 점도 업장이 연동 방식을 쉽게 바꾸지 못하는 이유 중 하나입니다.

## 운영 체크리스트

<Steps>
  <Step title="활성 프로모션 매핑 확인">
    야놀자 파트너센터에서 현재 활성화된 프로모션(핫딜, 오픈특가, 도보특가 등) 목록을 확인하고, 모든 프로모션 상품이 VCMS에 매핑되어 있는지 점검합니다.
  </Step>
  <Step title="오픈특가·도보특가 상태 점검">
    의도하지 않게 열려 있는 특가 요금제가 없는지 확인합니다. 전체 마감 시에도 개별 요금제가 활성 상태인지 별도로 확인합니다.
  </Step>
  <Step title="교차 판매 모니터링">
    아고다에서 자사 숙소를 검색하여, 야놀자 채널링을 통한 교차 판매 상품이 노출되고 있는지 확인합니다. 직접 등록 상품과의 요금 충돌 여부를 점검합니다.
  </Step>
  <Step title="프로모션 변경 시 재매핑 실행">
    프로모션을 새로 만들거나 수정하면 VCMS 매핑도 다시 진행합니다. 종료된 프로모션의 잔여 매핑도 정리합니다.
  </Step>
  <Step title="실제 판매 요금 확인">
    야놀자 앱에서 실제 고객에게 노출되는 요금을 주기적으로 확인합니다. VCMS 설정 요금과 실제 판매 요금의 차이가 있다면 야놀자 자체 쿠폰이나 프로모션 적용 여부를 점검합니다.
  </Step>
</Steps>
