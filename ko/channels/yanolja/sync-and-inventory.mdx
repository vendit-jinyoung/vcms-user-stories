야놀자는 국내 숙박 OTA 중 가장 많은 재고·요금 관련 이슈가 보고되는 채널입니다. 동기화 방향의 특수성, 연동 차단 이력, 재고 불일치, 판매 기간 제한 등이 복합적으로 얽혀 있어 채널 자체의 동작 방식을 먼저 이해하는 것이 중요합니다.

## 채널 특성 요약

| 항목 | 내용 |
|------|------|
| 동기화 방향 | VCMS → 야놀자 **일방향 덮어쓰기** |
| 판매 기간 | 기본 **30일** 제한 (90~120일 확장 권장) |
| 연동 끊김 원인 | 비밀번호 변경, IP 차단, 파트너센터 직접 수정 |
| 재고 자동 복구 | 일부 카테고리에서 미지원 |
| 동기화 주기 | 약 5분 간격, 성수기·당일 시간대에 느려질 수 있음 |
| 호텔 카테고리 | 재고/요금 연동 불완전 |

## 반복되는 운영 이슈

### 1. 일방향 덮어쓰기와 요금 되돌림

야놀자 파트너센터에서 직접 요금을 변경해도, 다음 동기화 시점에 VCMS 기준 요금으로 복원됩니다. 이 구조를 모르는 업장에서는 "요금을 바꿨는데 자꾸 원래대로 돌아간다"는 문의가 반복됩니다. 특히 일요일 요금을 주말 요금으로 수동 변경했을 때, VCMS에서 평일 요금으로 리셋되는 패턴이 빈번합니다. 야놀자 파트너센터에 기존에 설정해둔 기간별 자동 요금 변경 기능도 VCMS와 충돌할 수 있으므로, 연동 전에 반드시 정리가 필요합니다.

CS에서는 이렇게 안내합니다: "VCMS와 연동된 상태에서는 야놀자 파트너센터에서의 요금 변경이 유지되지 않습니다. 모든 요금 변경은 VCMS에서 진행해야 합니다."

### 2. 연동 끊김과 IP 차단

야놀자는 외부 CMS의 연동을 정책적으로 제한하거나 차단한 이력이 있습니다. 벤디트 서버 IP를 차단하여 연동을 중단시킨 사례, 모텔에 대한 정식 계약 부재를 이유로 시스템을 차단한 사례, 특정 요일에만 차단이 발생하는 패턴 등이 보고되었습니다. 연동이 끊기면 재고 차감이 이루어지지 않아 이전에 없던 오버부킹이 처음으로 발생하는 경우가 있습니다.

끊김 감지 방법으로는 VCMS에서 야놀자 채널 상태가 비정상(빨간색 표시)으로 나타나는지 확인하거나, 야놀자 예약이 일정 시간 이상 들어오지 않는 경우를 모니터링합니다.

### 3. 재고 차단 후 자동 복구 실패

야놀자에서 객실 재고가 0이 된 후 자동으로 1로 복구되지 않는 현상이 펜션 카테고리에서 특히 빈번합니다. 야놀자 시스템이 특정 시점에 판매 채널을 자동으로 차단하면서 수동 해제를 시도해도 야놀자가 계속 마감시키는 현상도 확인됩니다. VCMS에서 오픈한 객실이 야놀자에서는 마감되어 있거나, 그 반대의 불일치가 간헐적으로 발생합니다.

### 4. 동기화 지연

VCMS에서 변경한 요금·재고가 야놀자에 즉시 반영되지 않는 경우가 있습니다. 최초 동기화에 상당 시간이 소요되며, 이후에도 불규칙하게 작동하는 사례가 보고됩니다. 당일 예약이 활발한 시간대에는 동기화 속도가 느려져, VCMS에서 마감 처리를 해도 야놀자에서 동기화가 늦어 추가 예약이 들어오는 오버부킹 위험이 있습니다.

### 5. 판매 기간 30일 제한

야놀자 채널은 판매 기간을 기본 30일로 제한합니다. 30일 이후에는 상품 판매가 자동으로 막히며, 90일 또는 120일로 확장해야 장기 예약을 받을 수 있습니다. 다만 야놀자 측 정책에 따라 확장에 제약이 있을 수 있고, 여기어때의 연동 기간(30일/180일)과도 맞추기 어려운 구간이 발생합니다.


### 6. 연동 알림 미수신 문제

중요한 시스템 알림(야놀자 연동 지연, 채널 끊김 등)이 대표번호로 발송되어 아무도 확인하지 못하는 경우가 있습니다. 알림 수신 번호를 실제 담당자 번호로 설정하고, 연동 상태를 주기적으로 직접 확인하는 루틴이 필요합니다.

### 7. 기존 CMS에서 VCMS로 전환 시 주의사항

기존 채널 매니저(떠나요 등)에서 VCMS로 전환할 때, 야놀자 연동을 끊으면 야놀자에 등록된 상품 정보와 예약 데이터가 소실될 위험이 있습니다. 전환 전에 야놀자 파트너센터에서 현재 상품 설정을 백업하고, 기존 예약 데이터를 별도로 저장해두세요.

    VCMS 연동 상태에서는 파트너센터의 요금 변경이 다음 동기화 시 VCMS 기준으로 덮어쓰기됩니다. 야놀자에 설정해둔 기간별 자동 요금 변경도 마찬가지입니다. 모든 요금은 VCMS에서 관리해야 합니다.
  </Accordion>
  <Accordion title="연동이 끊어졌는데 자동으로 복구되나요?">

  </Accordion>

  </Accordion>
  <Accordion title="30일 넘는 예약은 어떻게 받나요?">

  </Accordion>

  </Accordion>

## 운영 체크리스트

<Steps>
  <Step title="파트너센터 직접 수정 금지 안내">
    야놀자 파트너센터에서 요금이나 재고를 직접 변경하지 않도록 업장에 안내합니다. VCMS 연동 중에는 모든 변경이 VCMS를 통해 이루어져야 합니다.
  </Step>
  <Step title="기존 자동 요금 변경 설정 정리">
    야놀자에 기설정된 기간별 자동 요금 변경 기능이 있다면, 연동 전에 반드시 비활성화합니다. 양쪽에서 동시에 요금을 제어하면 예측 불가능한 요금이 적용됩니다.
  </Step>
  <Step title="연동 상태 주기적 확인">
    VCMS 채널 연동 화면에서 야놀자 상태를 주기적으로 확인합니다. 비정상 표시가 있거나 야놀자 예약이 일정 시간 이상 들어오지 않으면 연동 끊김을 의심합니다.
  </Step>
  <Step title="판매 기간 확장 설정">
    야놀자 파트너센터에서 판매 기간을 30일에서 90~120일로 변경합니다. 장기 예약이 필요한 성수기 전에 미리 설정합니다.
  </Step>
  <Step title="마감 처리 후 이중 확인">
    VCMS에서 마감 처리 후, 야놀자 파트너센터에서도 해당 객실이 실제로 마감되었는지 확인합니다. 동기화 지연으로 인한 오버부킹을 방지합니다.
  </Step>
</Steps>
