## 꿀스테이 업종별 구분

꿀스테이는 **모텔, 호텔, 펜션** 세 가지 업종으로 나뉘어 있으며, 각각 파트너센터 접속 경로와 관리 방식이 다릅니다.

| 업종 | 대상 | 특징 |
|------|------|------|
| **모텔** | 모텔, 여관 | 대실/숙박 분리 관리, 모텔 전용 UI |
| **호텔** | 호텔, 리조트 | 객실 타입 기반 관리, 호텔 전용 UI |
| **펜션** | 펜션, 풀빌라, 글램핑 | 독채/호실 기반 관리, 펜션 전용 UI |

<Info>
업종에 따라 요금 설정 방식, 대실 지원 여부, 상품 구조가 달라요. 잘못된 업종으로 등록하면 운영에 불편이 생길 수 있어요.
</Info>

## 파트너센터 접속 경로

각 업종별로 파트너센터 접속 URL이 다릅니다.

| 업종 | 파트너센터 URL |
|------|-------------|
| **모텔** | `https://motel.partner.kkulstay.com` |
| **호텔** | `https://hotel.partner.kkulstay.com` |
| **펜션** | `https://pension.partner.kkulstay.com` |

<Warning>
위 URL은 참고용이에요. 정확한 접속 경로는 꿀스테이 가입 시 안내받은 주소를 사용하세요. 파트너센터 URL이 변경될 수 있으므로, 접속이 안 되면 꿀스테이 고객센터에 문의하세요.
</Warning>

## VCMS와 병행 운영 시 주의사항

꿀스테이를 VCMS 연동 채널(야놀자, 여기어때 등)과 함께 사용하는 경우, 재고 관리에 특별한 주의가 필요해요.

### 1. 꿀스테이 예약 → VCMS 수기 등록 필수

꿀스테이에서 예약이 들어오면 **즉시 VCMS에 수기 예약으로 등록**하세요. 그렇지 않으면 VCMS 연동 채널에서 해당 객실이 계속 열려 있어 오버부킹이 발생해요.

### 2. VCMS 노출 재고에서 꿀스테이 분량 차감

꿀스테이에 일정 수의 객실을 상시 열어놓고 있다면, 그 수량만큼 VCMS의 채널 노출 재고에서 미리 빼세요. 예를 들어 전체 10실 중 꿀스테이에 2실을 열어놓았다면, VCMS 노출 재고를 8실로 설정하세요.

### 3. 꿀스테이와 VCMS 입퇴실 시간 연동

꿀스테이(모텔, 호텔, 펜션)는 VCMS와 **입퇴실 시간 연동이 지원**돼요. VCMS에서 설정한 입퇴실 시간이 꿀스테이에 자동 반영됩니다. 단, 꿀스테이 파트너센터에서 직접 변경한 시간은 VCMS 동기화 시 덮어씌워질 수 있어요.

### 4. 요금은 별도 관리

꿀스테이의 요금은 VCMS에서 관리되지 않아요. 꿀스테이 파트너센터에서 직접 요금을 설정하고, 다른 채널 요금과의 일관성을 수동으로 관리해야 해요.

## 채널 사용 현황 (VOC 기반)

VOC 데이터 분석 결과, 꿀스테이는 야놀자·여기어때·네이버·아고다와 함께 **4~5번째 채널**로 사용하는 업장이 많습니다.

- "야놀자, 여기어때, 꿀스테이, 아고다 4개 채널을 사용하고 있어요"
- "야놀자, 여기어때, 꿀스테이, 네이버 채널 연동을 원해요"
- "야놀자, 아고다, 꿀스테이 등 추후 연동 계획이 있어요"

꿀스테이의 VCMS 공식 연동은 현재 지원되지 않지만, 위와 같이 꿀스테이를 활발히 사용하는 업장이 적지 않으므로 병행 운영 방법을 숙지해두는 것이 중요합니다.
